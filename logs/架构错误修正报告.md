### 2️⃣ require_api_key 未定义 (严重)

```python
# 错误日志: name 'require_api_key' is not defined
# 位置: container_dashboard.py:280
```

**修复**: 在 `container_dashboard.py` 顶部添加:
```python
from functools import wraps
from flask import request, jsonify

def require_api_key(f):
    """API Key 验证装饰器"""
    @wraps(f)
    def decorated(*args, **kwargs):
        api_key = request.headers.get('X-API-Key')
        if not api_key:
            return jsonify({'error': 'API key required'}), 401
        return f(*args, **kwargs)
    return decorated
```

### 3️⃣ 云端直连错误 (架构问题)

**错误**: `cloud_poller.py` 直连 `124.223.57.82:5006`

**代码位置**: `standalone_dispatch_server.py:825`
```python
CLOUD_RELAY_URL = os.getenv('CLOUD_RELAY_URL', '') or \
                  os.getenv('WECHAT_CLOUD_HOST', '') or \
                  'http://124.223.57.82:5006'
```

**正确架构**:
```
5008 (cloud_poller.py)
    ↓
5003 (standalone_dispatch_server.py) 的 _CloudPoller
    ↓
云端 5006
```

**问题分析**:
- 5008 的 cloud_poller.py 和 5003 的 _CloudPoller 都在轮询云端
- 造成重复轮询和资源浪费
- 应该统一由 5003 处理云端通信

**修复方案**:

方案 A: 禁用 5008 的 cloud_poller
```python
# 在 5008 的 app.py 中
# 注释掉 cloud_poller 初始化
# from cloud_poller import CloudPoller
# poller = CloudPoller()
```

方案 B: 让 5008 通过 5003 转发
```python
# 修改 cloud_poller.py，改为调用 5003 API
CLOUD_PROXY_URL = 'http://localhost:5003/api/dispatch-center/forward-to-cloud'

def poll_cloud():
    """通过5003转发到云端"""
    response = requests.post(
        CLOUD_PROXY_URL,
        json={'action': 'poll'},
        timeout=10
    )
    return response.json()
```

方案 C: 统一由 5003 处理，5008 不启动轮询
```python
# 在 5008 app.py 启动时
# 只启动消息处理，不启动轮询
from cloud_poller import CloudPoller

# 只初始化，不启动轮询线程
poller = CloudPoller(message_handler=handle_message)
# 不要调用 poller.start()
```
