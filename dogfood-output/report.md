### ✅ ISSUE-002: 5002库存管理UI蓝图路由路径错误 - **已修复**

| 属性 | 值 |
|------|-----|
| **严重程度** | 🔴 Critical → ✅ 已修复 |
| **服务** | 5002 容器中心 (库存管理蓝图) |
| **发现时间** | 2026-06-24 |
| **修复时间** | 2026-06-24 |

**问题描述**:
5002容器中心服务的库存管理蓝图 (`/inventory/*`) 无法访问，返回404错误。

**根本原因**:
蓝图内路由定义了完整路径 `/inventory/dashboard`，但蓝图注册时已添加 `url_prefix='/inventory'`，导致最终路径变成 `/inventory/inventory/dashboard`。

**修复方案**:
修改 `inventory_web/routes_core.py` 和相关蓝图文件中的所有路由定义，去掉 `/inventory` 前缀：
- `@bp.route('/inventory/dashboard')` → `@bp.route('/dashboard')`
- 其他路由同理

**修复后验证**:
```
GET /inventory/dashboard → 200 OK (11624 bytes)
```

**修复文件**:
- [routes_core.py](file:///d:/yuan/不锈钢网带跟单3.0/mobile_api_ai/inventory_web/routes_core.py)
- [routes.py](file:///d:/yuan/不锈钢网带跟单3.0/mobile_api_ai/inventory_web/routes.py)
- [container_center_api.py](file:///d:/yuan/不锈钢网带跟单3.0/mobile_api_ai/container_center_api.py#L244-L252) (白名单)
