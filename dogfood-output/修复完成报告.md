### P0-2 5001 登录 API 修复

**问题**: 5001代理到5003的`/api/login`端点不存在(405错误)

**修复**:
1. 修改 `desktop_web/server.py:657`:
   ```python
   # 原: requests.post(f'{DISPATCH_BASE}/api/login', json=data, timeout=5)
   # 改: requests.post(f'{DISPATCH_BASE}/api/auth/login', json=data, timeout=5)
   ```

2. 添加自动转换 name→operator_id 逻辑:
   ```python
   if 'name' in data and 'operator_id' not in data:
       try:
           r0 = requests.get(f'{DISPATCH_BASE}/api/dispatch-center/operators', timeout=3)
           if r0.status_code == 200:
               ops = r0.json().get('data') or []
               for op in ops:
                   if op.get('name') == data['name']:
                       data['operator_id'] = op.get('id') or op.get('enterprise_id')
                       break
       except Exception:
           pass
   ```

3. 修改前端 `desktop_web/templates/login.html:46`:

   ```javascript
   // 改: body: JSON.stringify({name: u, username: u, operator_id: u})
   ```

### 5003 _load_operators 缓存 BUG

**问题**: 5003 的 _load_operators 函数从 workers 表加载操作员时，name 字段加载异常（用 name 查询时找不到，但用 operator_id 可以找到）

**影响**: 5001 前端登录需要直接传 operator_id="YuanGangBiao" 才能成功

**优先级**: 🟠 P1（需后续排查）

**可能原因**:
- MySQLStorage.fetch_all 返回字典时，name 字段编码问题
- _load_operators 缓存逻辑 BUG
- workers 表 name 列的 charset 与读取不一致

**建议排查**:
- 5003 logs 中搜索 "load_operators" 或 "operators_cache"
- 直接查询 workers 表 name 列的内容
- 单元测试 _load_operators 返回值
