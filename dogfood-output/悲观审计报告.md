## 一、审计方法说明

每个问题都从以下6个维度进行悲观审计：
1. **真实性 (T)**: 问题数据是否真实存在
2. **严重性 (S)**: 影响范围和紧急程度
3. **根因 (R)**: 根本原因分析
4. **影响 (I)**: 业务影响
5. **修复可行性 (F)**: 修复方案可行性
6. **回归风险 (G)**: 修复后可能引入的新问题

### P0-2: 订单产品名为空

| 维度 | 评估 |
|------|------|
| **真实性** | ✅ 真实（61个订单）|
| **严重性** | 🟢 **被高估** |
| **根因** | 这些订单**有 product_type**（产品类型）字段，只是 product_name 冗余字段为空 |
| **影响** | UI显示问题，不影响业务逻辑 |
| **修复可行性** | ✅ 可从 product_type 自动生成 |
| **回归风险** | 🟢 低 |

**审计结论**:
- ❌ **问题严重性被高估**
- 实际情况：所有61个订单都有 product_type（如"普通网带"、"冷冻螺旋网"等）
- product_name 是冗余字段，可从 product_type 自动生成
- 这是数据冗余设计问题，不是数据缺失

**修复方案**:
```sql
-- 同步 product_name 从 product_type
UPDATE steel_belt.orders o
JOIN steel_belt.product_types pt ON o.product_type = pt.name
SET o.product_name = pt.name
WHERE o.is_deleted=0 AND (o.product_name IS NULL OR o.product_name='');
```
