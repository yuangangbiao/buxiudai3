## 一、问题描述

5001桌面端的登录功能无法正常使用，用户输入账号"苑岗彪"后点击登录，仍然停留在登录页。

---

## 二、问题诊断过程

### 2.1 网络请求分析

| 请求 | 状态 | 说明 |
|------|:----:|------|
| `GET /login` | 200 | 登录页加载成功 |
| `GET /static/js/shared.js` | 200 | JS加载成功 |
| `POST /api/login` | **500** | 登录API失败 |
| `POST /api/dispatch-center/login` | **405** | 端点不存在 |
| `POST /api/login (5003)` | **405** | 端点不存在 |

### 2.2 错误信息

```
Expecting value: line 1 column 1 (char 0)
```

这是JSON解析错误，说明5003返回的不是有效JSON。

---

## 三、根因分析

### 3.1 5001登录API代码

文件: `desktop_web/server.py:653`

```python
@app.route('/api/login', methods=['POST'])
def api_login():
    data = request.get_json(silent=True) or {}
    try:
        r = requests.post(f'{DISPATCH_BASE}/api/login', json=data, timeout=5)
        body = r.json()  # ← 这里报错：5003返回的不是JSON
        ...
    except Exception as e:
        return jsonify({'code': -1, 'message': str(e)}), 500
```

### 3.2 问题点

- 5001调用 `http://127.0.0.1:5003/api/login`
- **但5003没有 `/api/login` 端点**
- 5003的登录端点可能是其他路径

---

## 四、5003实际登录端点

需要确认5003的真实登录API路径。从代码中看到：
- `/api/dispatch-center/login` - 405 Method Not Allowed
- `/api/login` - 405 Method Not Allowed

可能需要：
1. 确认正确的登录API路径
2. 或者5001直接调用5003的API

---
