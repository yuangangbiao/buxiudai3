# archive/ 目录规范

## 用途

存放**归档脚本**——旧版本、废弃脚本、历史工具等。

作为参考保留，不参与日常开发。

## 目录结构

```
archive/
├── 2025/          ← 2025年归档
├── 2026/          ← 2026年归档（当前）
└── README.md
```

## 归档规则

### 什么情况下归档

| 情况 | 操作 |
|------|------|
| 脚本已更新到新版本，旧版不再使用 | 归档旧版 |
| 功能已废弃，相关脚本不再使用 | 归档废弃脚本 |
| 一次性工具完成后想保留参考 | 归档 |
| 版本迭代（v2/v3/v4...）完成后 | 只保留最终版，其余归档 |

### 什么情况下删除

| 情况 | 操作 |
|------|------|
| 超过 1 年的归档脚本 | 删除 |
| 归档超过 3 个版本 | 只保留最新 2 个 |
| 归档脚本文件名已无参考价值 | 删除 |

## 禁止事项

- ❌ 禁止将归档脚本复制回 `scripts/` 或 `tools/`
- ❌ 禁止在归档目录创建新脚本（只能移入，不能新建）
- ❌ 禁止将归档目录作为"第二工作区"

## 定期清理

建议每半年清理一次归档目录，删除超过 1 年的脚本。

## 归档命名

归档文件建议加上日期：

```
fix_dispatch_v2.py         →  archive/2026/fix_dispatch_v2_20260620.py
verify_order_status.py     →  archive/2026/verify_order_status_20260620.py
```
