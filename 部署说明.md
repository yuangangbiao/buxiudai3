# 库存管理系统部署说明

## 系统架构

本系统采用**服务器-客户端**架构：

- **服务器端**：连接MySQL数据库，提供RESTful API服务
- **客户端**：通过API连接服务器，进行库存管理操作

---

## 快速开始

### 1. 服务器端部署

#### 1.1 环境要求

- Python 3.8+
- MySQL 5.7+ 或 MariaDB 10.2+
- 局域网访问权限（如需多台电脑连接）

#### 1.2 安装步骤

1. **安装Python依赖**
   ```bash
   pip install flask requests pymysql
   ```

2. **配置数据库**
   
   确保MySQL服务已启动，创建库存数据库：
   ```sql
   CREATE DATABASE inventory_system CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```

3. **运行配置器**
   
   双击 `启动配置器.bat`，配置：
   - 数据库连接信息
   - 服务器监听地址（建议设为 `0.0.0.0`）
   - 监听端口（默认 `8080`）
   - API密钥（请妥善保管）

4. **初始化数据库**
   
   确保数据库表结构已创建，参考 `reset_inventory_db.py`

5. **启动服务器**
   
   双击 `启动库存服务器.bat`

#### 1.3 防火墙设置

确保Windows防火墙允许指定端口（如8080）的入站连接：

1. 打开「Windows Defender 防火墙」
2. 点击「高级设置」
3. 添加入站规则，允许TCP协议的8080端口

---

### 2. 客户端部署

#### 2.1 环境要求

- Python 3.8+
- 能够连接到服务器的网络

#### 2.2 安装步骤

**方法一：全新安装**

1. 复制以下文件到客户端电脑：
   - `inventory_client.py`
   - `启动库存客户端.bat`
   - `inventory_client_config.json`（从配置器导出）

2. 双击 `启动库存客户端.bat`

**方法二：使用配置器**

1. 先在服务器上配置好
2. 从配置器导出客户端配置
3. 将配置文件和客户端程序一起复制
4. 启动客户端即可

#### 2.3 配置说明

在客户端配置文件（`inventory_client_config.json`）中：

```json
{
    "server_url": "http://192.168.1.100:8080",
    "api_key": "your_api_key_here",
    "auto_refresh": true,
    "refresh_interval": 60
}
```

- `server_url`：服务器的局域网IP地址和端口
- `api_key`：必须与服务器配置一致
- `auto_refresh`：是否自动刷新数据
- `refresh_interval`：刷新间隔（秒）

## API接口说明

### 基础信息

- **Base URL**: `http://服务器IP:端口`
- **认证方式**: Header中添加 `X-API-Key`

### 接口列表

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/health` | GET | 健康检查 |
| `/api/inventory/list` | GET | 获取库存列表 |
| `/api/inventory/statistics` | GET | 获取统计数据 |
| `/api/inventory/low-stock` | GET | 获取低库存预警 |
| `/api/inventory/inbound` | POST | 入库操作 |
| `/api/inventory/outbound` | POST | 出库操作 |
| `/api/inventory/transactions` | GET | 获取出入库流水 |
| `/api/products` | GET | 获取商品列表 |
| `/api/warehouses` | GET | 获取仓库列表 |
| `/api/suppliers` | GET | 获取供应商列表 |
| `/api/notifications` | GET | 获取通知列表 |
| `/api/notifications/:id/respond` | POST | 响应通知 |

## 常见问题

### Q1: 客户端无法连接服务器

**排查步骤：**
1. 确认服务器已启动
2. 检查服务器IP地址是否正确
3. 确认防火墙已开放对应端口
4. 检查API密钥是否一致
5. 在客户端电脑上ping服务器IP

### Q2: 数据库连接失败

**排查步骤：**
1. 确认MySQL服务已启动
2. 检查数据库配置信息（主机、端口、用户名、密码）
3. 确认数据库已创建
4. 检查用户权限

### Q3: 局域网其他电脑无法访问

**解决方法：**
1. 服务器监听地址设为 `0.0.0.0`
2. 配置Windows防火墙允许对应端口
3. 确认客户端使用的是服务器的局域网IP

### Q4: 如何修改API密钥

**方法：**
1. 运行配置器
2. 在「服务器配置」标签页修改API密钥
3. 在「客户端配置」标签页使用相同的密钥
4. 保存并重启服务

## 安全建议

1. **API密钥安全**
   - 不要使用默认密钥
   - 定期更换API密钥
   - 不要将配置文件提交到公开仓库

2. **网络安全**
   - 建议在局域网内使用
   - 如需公网访问，建议使用VPN
   - 考虑使用HTTPS

3. **数据库安全**
   - 设置强密码
   - 限制数据库用户权限
   - 定期备份数据
