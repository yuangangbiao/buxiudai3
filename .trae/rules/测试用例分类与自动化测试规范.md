## 一、测试用例分类体系

### 1.1 case_type 功能类型

| 类型 | 说明 | 推断规则 |
|------|------|---------|
| `单元` | 单元测试（默认） | 普通测试文件 |
| `边界` | 边界值测试 | 文件名含 `boundary`、`edge` |
| `并发` | 并发测试 | 文件名含 `concurrent`、`parallel`、`thread` |
| `集成` | 集成测试 | 文件名含 `integration`、`e2e`、`workflow` |

### 1.2 boundary_cat 边界分类

| 分类 | 说明 | 推断规则 |
|------|------|---------|
| `空` | 空值边界 | 函数名含 `empty`、`null`、`none`、`blank` |
| `单条` | 单条数据 | 函数名含 `single`、`only`、`单` |
| `阈值` | 阈值边界 | 函数名含 `max`、`min`、`threshold`、`limit` |
| `上溢` | 溢出边界 | 函数名含 `overflow`、`exceed`、`超`、`溢` |
| `并发` | 并发场景 | 函数名含 `concurrent`、`parallel`、`race` |
| `参数化` | 参数化变体 | @pytest.mark.parametrize 装饰的测试 |

## 二、数据库表结构

```sql
CREATE TABLE test_cases (
    id INTEGER PRIMARY KEY,
    test_path TEXT NOT NULL,           -- 测试文件路径
    params TEXT DEFAULT '',             -- 参数化参数
    module TEXT NOT NULL,              -- 模块名
    case_type TEXT NOT NULL,           -- 功能类型
    boundary_cat TEXT,                 -- 边界分类
    last_run_at TIMESTAMP,             -- 最后运行时间
    last_status TEXT,                  -- 最后状态
    owner_func TEXT,                   -- 所属功能
    UNIQUE(test_path, params)
);
```

## 四、触发条件与使用流程

### 4.1 新增测试文件时

```
1. 运行 pytest 测试新文件
       ↓
2. 收集测试用例入库
   python .workbuddy/tools/regression_db.py import-tests
       ↓
3. 自动分析新增用例的分类（已集成）
       ↓
4. 分类已更新到数据库
```

### 4.2 手动触发

```bash
# 收集测试用例（自动更新分类）
python .workbuddy/tools/regression_db.py import-tests

# 仅更新分类
python .workbuddy/tools/auto_update_test_categories.py

# 全量重新分析所有测试
python .workbuddy/tools/auto_update_test_categories.py --all

# 监听模式（持续监控）
python .workbuddy/tools/auto_update_test_categories.py --watch
```

## 六、分类推断规则

### 6.1 自动推断逻辑

```python
# case_type 推断
def infer_case_type(test_path):
    if 'concurrent' in path: return '并发'
    if 'integration' in path: return '集成'
    if 'boundary' in path: return '边界'
    return '单元'  # 默认

# boundary_cat 推断
def infer_boundary_cat(test_path, params):
    if params: return '参数化'
    if 'empty' in name: return '空'
    if 'single' in name: return '单条'
    if 'max' in name: return '阈值'
    if 'overflow' in name: return '上溢'
    if 'concurrent' in name: return '并发'
    return None
```

### 6.2 手动指定分类

如需手动指定分类，可在测试文件的 docstring 中添加标记：

```python
def test_boundary_case():
    """
    case_type: 边界
    boundary_cat: 阈值
    """
    # 测试代码
```

## 八、常见问题

### 8.1 分类不准确

**原因**: 自动推断基于文件名/函数名，可能不准确

**解决**:
1. 重命名测试文件/函数，使其更符合推断规则
2. 或在 docstring 中手动指定分类

### 8.2 缓存未更新

**原因**: pytest 缓存导致测试结果不更新

**解决**:
```bash
# 删除缓存
del .workbuddy/.pytest_collect_cache.json

# 重新收集
python .workbuddy/tools/regression_db.py import-tests
```

### 8.3 数据库损坏

**原因**: 缓存文件版本不匹配

**解决**:
```bash
# 重建数据库
del .workbuddy/regression/regression.db
python .workbuddy/tools/regression_db.py init
python .workbuddy/tools/regression_db.py import-tests
```
