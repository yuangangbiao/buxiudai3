## 🔴 核心架构规则

### 规则 1: 云端通信统一路径

**强制要求**: 所有云端通信必须通过 `5003 调度中心` 统一转发到云端 `5006`

```
❌ 禁止: 各模块直连云端
✅ 必须: 统一通过 5003 调度中心转发
```

**正确架构**:
```
┌─────────────────┐
│  本地服务 (5008)  │
│  手机报工/库存   │
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│  调度中心 (5003) │  ← 统一转发入口
│  容器中心       │
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│  云端 (5006)    │
│  微信消息       │
└─────────────────┘
```

## 🛠️ 编码规范

### 禁止的写法

```python
# ❌ 错误: 直接连接云端
CLOUD_API_URL = 'http://124.223.57.82:5006/api/poll'
response = requests.get('http://124.223.57.82:5006/api/queue/poll')

# ❌ 错误: 在 5008 中启动独立的云端轮询
poller = CloudPoller(cloud_host='124.223.57.82', port=5006)
poller.start()

# ❌ 错误: 多处配置云端地址
WECHAT_CLOUD_HOST = 'http://124.223.57.82:5006'
```

### 正确的写法

```python
# ✅ 正确: 通过 5003 调度中心转发
DISPATCH_CENTER_URL = 'http://localhost:5003'
CLOUD_PROXY_ENDPOINT = '/api/dispatch-center/forward-to-cloud'

def poll_cloud():
    """云端轮询 - 通过 5003 统一转发"""
    response = requests.post(
        f'{DISPATCH_CENTER_URL}{CLOUD_PROXY_ENDPOINT}',
        json={'action': 'poll'},
        timeout=10
    )
    return response.json()

def send_to_cloud(message):
    """发送消息到云端 - 通过 5003 统一转发"""
    response = requests.post(
        f'{DISPATCH_CENTER_URL}{CLOUD_PROXY_ENDPOINT}',
        json={'action': 'send', 'message': message},
        timeout=10
    )
    return response.json()

# ✅ 正确: 只配置 5003 地址
DISPATCH_CENTER_HOST = 'localhost'
DISPATCH_CENTER_PORT = 5003
```

## 📝 5003 调度中心必须实现的 API

### 云端转发 API

```python
# standalone_dispatch_server.py 必须包含以下端点

@app.route('/api/dispatch-center/forward-to-cloud', methods=['POST'])
def forward_to_cloud():
    """统一云端转发入口"""
    data = request.json
    action = data.get('action')

    if action == 'poll':
        # 调用云端轮询
        return cloud_poller.poll()
    elif action == 'send':
        # 调用云端发送
        return cloud_poller.send(data.get('message'))
    elif action == 'ack':
        # 调用 ACK 确认
        return cloud_poller.ack(data.get('message_ids'))
    else:
        return jsonify({'error': 'Unknown action'}), 400
```

## ⚠️ 违反后果

| 违反场景 | 后果 |
|---------|------|
| 直连云端 | 架构混乱、重复轮询、资源浪费 |
| 多处配置云端地址 | 配置不一致、维护困难 |
| 绕过 5003 转发 | 云端消息丢失、功能异常 |
