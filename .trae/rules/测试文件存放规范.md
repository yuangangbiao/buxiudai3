## 一、测试文件目录结构

```
项目根目录/
├── tests/
│   ├── unit/                    # 单元测试
│   │   ├── core/               # core层测试
│   │   ├── models/             # models层测试
│   │   ├── services/           # services层测试
│   │   └── utils/              # utils层测试
│   ├── integration/             # 集成测试
│   ├── e2e/                    # 端到端测试
│   └── conftest.py             # 共享fixture
├── pytest.ini                  # pytest配置
└── reports/                    # 测试报告目录
```

## 三、文件命名规范

### 3.1 标准测试文件
```
test_{模块名}.py
```
- 示例: `test_order.py`, `test_validators.py`, `test_db.py`

### 3.2 完整覆盖测试文件
```
test_{模块名}_complete.py
```
- 示例: `test_order_complete.py`, `test_helpers_complete.py`

### 3.3 边界/深度测试文件
```
test_{模块名}_depth.py     # 深度测试
test_{模块名}_gaps.py      # 间隙测试
test_{模块名}_boundary.py  # 边界测试
```
- 示例: `test_order_depth.py`, `test_order_gaps.py`

### 3.4 DAO层测试文件
```
test_{模块名}_dao_complete.py
```
- 示例: `test_order_dao_complete.py`, `test_process_dao_complete.py`

### 3.5 集成测试文件
```
test_{模块名}_integration.py
test_{模块名}_mysql.py
```
- 示例: `test_process_code_integration.py`

## 七、Mock使用规范

### 7.1 数据库Mock

#### 方式1：类级fixture（推荐）
```python
class TestOrderDAO:
    @pytest.fixture(autouse=True)
    def setup_mocks(self):
        """自动Mock数据库"""
        self.mock_cursor = MagicMock()
        self.mock_conn = MagicMock()
        self.mock_conn.cursor.return_value.__enter__.return_value = self.mock_cursor
        self.mock_conn.cursor.return_value.__exit__.return_value = None

        self.get_conn_patch = patch('models.order.get_connection', return_value=self.mock_conn)
        self.get_conn_patch.start()
        yield
        self.get_conn_patch.stop()
```

#### 方式2：@patch装饰器
```python
@patch('models.order.get_connection')
def test_create_order(self, mock_get_conn):
    """测试创建订单"""
    mock_conn = MagicMock()
    mock_get_conn.return_value = mock_conn
    # 测试代码
```

#### 方式3：fixture参数
```python
def test_something(mock_get_connection):
    """使用mock_get_connection fixture"""
    # 测试代码
```

### 7.2 外部API Mock
```python
@patch('urllib.request.urlopen')
def test_external_api(self, mock_urlopen):
    """测试外部API调用"""
    mock_response = MagicMock()
    mock_response.read.return_value = b'{"status":"ok"}'
    mock_urlopen.return_value = mock_response
    # 测试代码
```

### 7.3 Mock注意事项
| 场景 | 规则 |
|------|------|
| 数据库操作 | 必须Mock |
| 外部API | 必须Mock |
| 时间相关 | 使用 freezegun 或 patch('time.time') |
| 文件系统 | 使用 tmp_path fixture |
| 随机数 | patch random 函数 |

## 八、集成测试标记

### 8.1 标记装饰器
```python
import pytest

# 集成测试（需要真实数据库）
pytestmark = pytest.mark.integration

# 或单个测试标记
@pytest.mark.integration
def test_mysql_connection():
    """需要真实MySQL连接"""
    pass
```

### 8.2 跳过集成测试
```bash
# 运行时不包含集成测试
pytest tests/unit -m "not integration"

# 只运行集成测试
pytest tests/unit -m integration
```

## 九、测试质量标准

### 9.1 必须包含
- ✅ 有断言（assert语句）
- ✅ 有Mock（数据库操作必须Mock）
- ✅ 有注释（说明测试目的）

### 9.2 禁止出现
- ❌ 空测试（只有pass）
- ❌ 污染主数据库（未Mock的数据库操作）
- ❌ 硬编码敏感信息

### 9.3 质量检查
```bash
# 运行质量检查
python _check_test_quality.py

# 输出指标
# - 有效测试率 >= 80%
# - 无空测试
# - 无未Mock的数据库操作
```

## 十、运行测试命令

### 10.1 基本命令
```bash
# 运行所有单元测试
pytest tests/unit --tb=short

# 运行特定层
```

## 十一、常见问题处理

### 问题1: sys.path路径错误
**错误**: `SyntaxError: '(' was never closed`
**原因**: `sys.path.insert`的括号数量不对
**解决**:
```python
# 正确（4个右括号）
sys.path.insert(0, os.path.dirname(os.path.dirname(os.path.dirname(__file__))))
```

### 问题2: 测试污染主数据
**原因**: 数据库操作未Mock
**解决**: 使用类级fixture或@patch装饰器Mock数据库

### 问题3: 集成测试失败
**原因**: 需要真实数据库
**解决**: 使用`pytestmark = pytest.mark.integration`标记，CI中跳过
