# 对话框使用规范

## 目录结构

```
views/dialogs/
├── __init__.py        # 统一导出
├── base.py            # BaseDialog + popup_form + alert/confirm
├── widgets.py         # 自定义控件
├── quality_dialogs.py # 质检模块
├── material_dialogs.py# 备料模块
├── rule_dialogs.py    # 规则模块
└── 新模块_dialogs.py  # 新模块按需创建
```

## 三种创建方式（优先级排序）

### 方式一：继承 BaseDialog（复杂对话框）
- `BaseDialog(parent, title, width, height, resizable, topmost, window_key)`
- `resizable` 默认 `True`（可调整大小 + 尺寸记忆）
- `window_key` 默认使用 `类名`，用于持久化窗口尺寸到 `data/window_config.json`
- 覆写模板方法：

| 方法 | 说明 | 必须 |
|------|------|:----:|
| `_build_ui()` | 构建界面 | ✅ |
| `_validate()` | 返回 `(bool, str)` | ❌ |
| `_on_confirm()` | Enter/按钮确认 | ❌ |
| `_on_cancel()` | Escape 取消 | ❌ |
| `_on_close()` | 点 X 关闭 | ❌ |

- 子类 `__init__` **先设实例属性，再调 `super().__init__()`**
- 不要覆写 `_handle_confirm/_handle_cancel/_handle_close`
- 子类 **禁止**在 `_build_ui()` 中调用 `setup_resizable_window()`（BaseDialog 已自动处理）
- 需要自定义 `window_key` 时在 `super().__init__(window_key="xxx")` 传入

### 方式二：popup_form（简单表单，≤5字段）
- `popup_form(title, fields, callback, width, height, resizable, on_confirm, window_key)`
- `resizable` 默认 `True`，传入 `window_key` 启用尺寸记忆
- 字段 type 支持：`entry` / `combo` / `grid_combo` / `checkgroup` / `attachment` / `label` / `spinbox` / `date` / `text` / `password` / `color`

### 方式三：内联 Toplevel（仅容器场景）
- 禁止直接写 `tk.Toplevel()` 创建对话框
- 仅允许**嵌入子视图的容器窗口**（如嵌入 QualityRuleView）
- 需要可调大小时手动调用 `setup_resizable_window(win, "唯一key", "800x600")`

## 导入规范
- 统一从 `views.dialogs` 导入（走 `__init__.py`）
- 避免循环引用时用局部导入 `from views.dialogs import XXX`
- 禁止直接导入实现文件 `from views.dialogs.xxx_dialogs import ...`

## 约束

| 项目 | 规范 |
|------|------|
| 窗口标题 | 中文，简洁 |
| 默认尺寸 | 信息 400×300，表单 500×400，大表单 650×auto |
| resizable | **默认 True**（可调大小 + 尺寸记忆），不可调大小时显式传 `resizable=False` |
| topmost | 仅强制注意场景设为 True |
| window_key | 默认类名，需自定义时传参（如 `window_key="my_dialog"`） |
| 单位选择 | 从 `utils.custom_types` 导入 `get_unit_options()` |
| 硬编码禁止 | 尺寸/颜色/字体从配置读取 |
