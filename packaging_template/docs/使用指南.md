## 配置说明

### `packaging_config.yaml`

| 字段 | 含义 | 默认值 |
|------|------|--------|
| `project.name` | 项目中文名 | 不锈钢网带跟单 3.0 |
| `project.name_exe` | 生成的 exe 名 | 不锈钢网带跟单系统v3.0.exe |
| `paths.main_entry` | Python 主入口 | main.py |
| `paths.dist_dir` | 输出目录 | dist |
| `build.console` | 是否显示控制台 | false (桌面应用) |
| `build.upx` | 启用 UPX 压缩 | true |
| `build.icon` | 图标路径 | null (默认) |
| `data_files.extensions` | 数据文件扩展名 | .json / .db / .txt |
| `excludes.modules` | 排除的大依赖 | numpy / pandas / cv2 等 |
| `ignore_patterns.dirs` | 打包时跳过的目录 | tests / docs / venv 等 |

## 流程

1. **Step 1**: 扫描 `D:\yuan\gbd3.0\` 下所有 .py 模块
2. **Step 2**: 解析所有 import 语句,构建 hidden_imports 列表
3. **Step 3**: 生成 `D:\yuan\gbd3.0\specs\gbd3.0.spec`
4. **Step 4**: 调 `pyinstaller` 打包,失败自动修复(最多 3 次)

---

## 自动修复机制

打包失败时,脚本自动:
1. 从 stderr 中提取 `ModuleNotFoundError` / `ImportError` 的模块名
2. 把缺失模块追加到 `packaging_config.yaml` 的 `auto_discovered` 字段
3. 自动重试打包(最多 3 次)

**禁用**: 加 `--no-auto-fix` 参数

## 故障排查

| 问题 | 解决 |
|------|------|
| `ModuleNotFoundError: No module named 'xxx'` | 脚本自动修复(最多 3 次)或手动加到 hidden_imports.base_modules |
| `UPX not found` | 下载 UPX 并加到 PATH,或设置 `build.upx: false` |
| exe 启动报错 | 用 `--console` 重新打包看错误输出 |
| 打包后路径错乱 | 检查 `data_files.extensions` 是否包含你的数据文件 |
| 打包超时 | 排除目录(`ignore_patterns.dirs`)可缩短扫描时间 |
