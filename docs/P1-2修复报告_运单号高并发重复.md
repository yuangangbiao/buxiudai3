## Bug 描述

**问题**: 发货单创建接口 `POST /api/shipment/add` 使用秒级时间戳生成运单号/发货单号，在高并发场景下同一秒内多个请求会生成相同的单号。

**影响**:
1. 数据库 UNIQUE 约束冲突（`shipment_no` 有 `UNIQUE` 约束）→ 插入失败
2. 物流追踪失败 → 无法查询物流信息
3. 客户投诉 → 运单号重复无法区分

---

## 修复内容

### 1. 修复 shipment_no 生成逻辑（第3068行）

| 项目 | 修复前 | 修复后 |
|------|--------|--------|
| **格式** | `SH{YYYYMMDDHHMMSS}` | `SH{10位毫秒时间戳}{3位随机数}` |
| **示例** | `SH20260623143025` | `SH7845123690128` |
| **并发容量** | 1秒1个 | 1秒900个 |
| **唯一性** | 秒级冲突 | 毫秒级+随机数双重保障 |

### 2. 新增 tracking_no 自动生成（第3070-3074行）

| 项目 | 修复前 | 修复后 |
|------|--------|--------|
| **来源** | 前端传入，可能为空 | 前端未传时自动生成 |
| **格式** | `WB{10位毫秒时间戳}{2位随机数}` | 同左 |
| **示例** | 可能为空或 `SF1234567890` | `WB784512369012` |

### 3. 代码对比

**修复前**:
```python
shipment_no = f"SH{datetime.now().strftime('%Y%m%d%H%M%S')}"
# ...
data.get('tracking_no', ''),  # 可能为空
```

**修复后**:
```python
import time
ts_ms = int(time.time() * 1000) % 10000000000
shipment_no = f"SH{ts_ms}{random.randint(100, 999)}"
# ...
tracking_no = data.get('tracking_no', '').strip()
if not tracking_no:
    ts_ms_track = int(time.time() * 1000) % 10000000000
    tracking_no = f"WB{ts_ms_track}{random.randint(10, 99)}"
```
