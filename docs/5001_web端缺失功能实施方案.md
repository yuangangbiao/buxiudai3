## 八、技术要点

### 共用层
- 所有新增 API 复用现有 `models/` 数据访问层，不重复造轮子
- 服务层（`services/`）直接调用，如 `ScheduleDispatchService`
- 权限控制：复用现有 `@require_auth` 装饰器

### UI 规范
- 风格与现有 `production.html` / `orders.html` 保持一致
- 使用共享 CSS：`/static/css/shared.css`
- Modal 弹窗复用现有 `showModal()` 模式

### 数据库
- 读写同一个 MySQL，无需新建表
- 事务处理：参考阶段五已修复的 `try/finally` 模式
