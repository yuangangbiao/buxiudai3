### 部署模式

```
生产部署（双进程）:
  container_center_api.py → 独立进程 :5002（数据+告警+消息）
  wechat_server.py        → 独立进程 :5003（仅调度流程）
  通过 CONTAINER_CENTER_URL=http://localhost:5002 连接

开发/单机部署:
  app.py → 整合启动（容器中心 + 调度中心在同一个进程，通过 SDK 内部路由）
```

### 环境变量清单

```bash
# 容器中心配置
CC_DATA_DIR=data                    # 分库数据目录
CC_PORT=5002                        # 容器中心端口
SHARED_SECRET=your-secret-here      # API 鉴权密钥

# 调度中心配置
CONTAINER_CENTER_URL=http://localhost:5002  # 容器中心 URL
DISPATCH_PORT=5003                  # 调度中心端口

# 云端配置
WECHAT_CLOUD_URL=http://wechat_cloud:5006   # 企业微信云端地址
```

### 验收标准

1. **生产部署验收**：
   - [ ] 容器中心启动后监听 :5002，全部 API 端点正常
   - [ ] 调度中心启动后监听 :5003，通过 SDK 连接容器中心
   - [ ] 两个进程独立运行，互不影响
   - [ ] 容器中心停止不影响调度中心进程（仅功能受限）

2. **开发部署验收**：
   - [ ] `app.py` 整合启动后两个服务均正常运行
   - [ ] 开发模式下自动使用 SDK 内部路由（无需网络）

3. **配置验收**：
   - [ ] `.env.example` 包含全部必需环境变量
   - [ ] 所有路径通过环境变量配置，无硬编码
   - [ ] 缺失环境变量时给出友好错误提示

4. **测试验收**：
   - [ ] 生产模式启动测试
   - [ ] 开发模式启动测试
   - [ ] 环境变量缺失测试

## 实现约束

### 技术栈
- Flask 启动配置
- `.env` 文件管理（python-dotenv 如果项目中已使用）
- 环境变量读取使用 `os.environ.get()` 或 `os.getenv()`

### 接口规范
- 双进程间通过 HTTP API 通信，无共享内存
- 共享密钥通过 `SHARED_SECRET` 环境变量传递

### 质量要求
- 启动时校验必需环境变量，缺失则打印清晰错误并退出
- 禁止将密钥、密码 etc. 硬编码在代码中
- 禁止将路径硬编码
