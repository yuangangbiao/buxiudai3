## 输出契约（Output Contract）

### 交付物
- 前端告警规则配置页面（1 个或 2 个页面/组件）

### 页面内容

```
告警规则配置页面
├── 超时告警
│   ├── 启用/禁用开关（调用 PUT /api/container/config/alert_rules）
│   ├── 超时时间阈值输入（分钟）
│   ├── 提醒间隔输入（分钟）
│   └── 最大提醒次数输入
├── 外协催单
│   ├── 启用/禁用开关
│   ├── 提醒天数输入（逗号分隔）
│   └── 提醒时间点输入（逗号分隔）
└── 告警记录列表
    ├── 告警级别筛选下拉框（全部/INFO/WARNING/CRITICAL）
    ├── 告警类型筛选下拉框（全部/overdue/outsource_overdue/outsource_remind）
    ├── 表格列表（时间/标题/级别/类型/操作列）
    └── 忽略按钮（调用 POST /api/container/alert/<id>/dismiss）
```

### 验收标准

1. **功能验收**：
   - [ ] 超时告警规则可读可写
   - [ ] 外协催单规则可读可写
   - [ ] 告警记录列表可查看
   - [ ] 可按级别/类型筛选告警
   - [ ] 可执行忽略操作
   - [ ] 配置保存后实时生效

2. **UI 验收**：
   - [ ] 页面风格与调度中心现有页面一致
   - [ ] 表单校验（数字范围、逗号分隔格式等）
   - [ ] 保存成功/失败有明确提示
   - [ ] 告警列表分页显示
   - [ ] 移动端适配（如果现有页面支持）

3. **交互验收**：
   - [ ] 配置修改后未保存有离开确认提示
   - [ ] 保存按钮有加载状态
   - [ ] 忽略操作有二次确认弹窗

4. **测试验收**：
   - [ ] 页面渲染正常，无 JS 错误
   - [ ] 表单提交流程正确（API 调用的参数格式正确）
   - [ ] 告警列表加载正常

## 实现约束

### 技术栈
- 复用调度中心前端的现有 UI 框架/模板引擎
- 样式优先复用全局 CSS 变量
- API 调用复用项目中已有的 HTTP 工具函数

### 接口规范
- API 调用使用项目中已有的 HTTP 客户端（如 axios / fetch）
- 请求/响应格式与后端 API 定义一致

### 质量要求
- 不做 UI 自动化测试（按极简结对编程规范）
- 做语法 lint 校验
- 禁止硬编码 API URL，从配置文件中读取
- 禁止硬编码样式值（使用全局 CSS 变量）
