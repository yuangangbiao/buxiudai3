### P0-2.2: N+1 查询优化

#### 优化点 1: 操作员导出统计查询

**文件**: [desktop_web/server.py](file:///d:/yuan/不锈钢网带跟单3.0/desktop_web/server.py#L928-L953)

**问题**: 对每个 operator 单独执行 SQL 统计查询 → N+1 问题

**修复前**:
```python
for o in operators:
    oid = o.get('operator_id') or o.get('id')
    cur.execute("""SELECT ... FROM workreport_records WHERE worker_id=%s...""", (oid,))
```

**修复后** (批量查询):
```python
valid_oids = [o.get('operator_id') or o.get('id') for o in operators if ...]
placeholders = ','.join(['%s'] * len(valid_oids))
cur.execute(f"""SELECT worker_id, ... FROM workreport_records
    WHERE worker_id IN ({placeholders}) GROUP BY worker_id""", tuple(valid_oids))
```

**性能提升**: N 次查询 → 1 次查询 (假设 N=50，性能提升约 50 倍)

**文件**: [desktop_web/server.py](file:///d:/yuan/不锈钢网带跟单3.0/desktop_web/server.py#L2230-L2278)

**问题**: 多个工人同时报工可能导致数据覆盖 (竞态条件)
