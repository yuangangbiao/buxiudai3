**产品经理**：小曦  
**分析日期**：2026-06-23  
**分析对象**：不锈钢网带跟单3.0 Web5001系统 + 桌面端

## 一、工厂核心痛点分析（按角色）

### 1.1 生产主管的痛点

| 痛点编号 | 痛点描述 | 具体场景 | 影响程度 |
|---------|---------|---------|---------|
| PM-001 | **排产信息分散，看不到全局** | 排产时需要从订单列表、工单列表来回切换，无法一眼看清哪些订单已排产、哪些未排产、哪些交期紧迫 | 🔴 高 |
| PM-002 | **交期紧迫时无法快速评估产能** | 临时加单时不知道车间当前产能是否饱和，没有可视化的产能看板 | 🔴 高 |
| PM-003 | **订单变更时排产调整困难** | 客户临时改单，已排产的工单需要手工一个个去改，没有批量调整功能 | 🟠 中 |
| PM-004 | **车间工人操作复杂** | 现有系统功能按钮多、层级深，车间工人（可能只有小学文化）不会用 | 🔴 高 |
| PM-005 | **缺乏生产进度实时追踪** | 生产到哪一步了？哪个工序卡住了？只能靠电话或现场问 | 🔴 高 |
| PM-006 | **排产发布后无法追踪执行状态** | 发布了任务到微信报工，但不知道工人有没有收到、有没有开工 | 🟠 中 |

**现场观察**：
- 车间噪音大，工人不可能盯着电脑屏幕看
- 工人年纪大，拼音打字困难
- 生产节奏快，操作必须3秒内完成
- 工厂经常停电或断网

### 1.2 仓库管理员的痛点

| 痛点编号 | 痛点描述 | 具体场景 | 影响程度 |
|---------|---------|---------|---------|
| WH-001 | **物料齐套判断依赖人工经验** | "这个订单要多少料？"——靠仓管脑子算，容易算错 | 🔴 高 |
| WH-002 | **物料计算规则复杂** | 不同产品类型、不同规格的物料计算公式不一样，系统里的计算器不好用 | 🔴 高 |
| WH-003 | **库存不足时无法及时预警** | 等生产到一半才发现缺料，停工待料损失大 | 🔴 高 |
| WH-004 | **来料批次管理混乱** | 同一种物料有多个批次，入库时没有记录批次信息 | 🟠 中 |
| WH-005 | **物料模板复用性差** | 类似订单的物料配置需要重新录一遍，没有模板库或模板不好用 | 🟠 中 |
| WH-006 | **桌面端和移动端数据不同步** | 仓管在桌面端录了物料，移动端看不到，或者反过来 | 🟠 中 |

**现场观察**：
- 仓管经常在仓库和办公室之间跑，没法一直守在电脑前
- 手机扫码录入比手输快10倍
- 晚上仓库光线暗，屏幕看不清

## 二、Web5001 vs 桌面端体验差距分析

| 功能模块 | 桌面端体验 | Web5001体验 | 差距分析 |
|---------|-----------|------------|---------|
| **订单管理** | ✅ 列表+详情+操作一体化 | ❌ 只能查看，编辑需要跳回桌面端 | web端功能严重阉割，跟单员无法独立完成工作 |
| **生产排单** | ✅ 完整排产流程 | ❌ 无法排产，只能查看 | 生产主管必须用桌面端 |
| **物料备料** | ✅ 物料计算+模板+库存集成 | ❌ 仅查看物料清单 | 仓库管理员无法在web端工作 |
| **质检管理** | ✅ 完整的质检任务编制+发布 | ❌ 仅查看记录 | 质检员无法独立工作 |
| **发货管理** | ✅ 完整发货流程+物流追踪 | ❌ 部分支持 | 物流专员勉强能用 |
| **工序报工** | ✅ 完整报工流程 | ✅ 手机端扫码+快速报工 | 🔴 唯一超越桌面端的模块 |
| **数据统计** | ✅ 实时数据看板 | ❌ 无统计报表 | 管理者无法在web端查看数据 |
| **权限控制** | ✅ 细粒度权限 | ⚠️ 权限粗糙 | web端管理员和普通员工权限差异小 |

### 核心差距根因

1. **架构设计问题**：Web5001是桌面端的"阉割版"，不是独立设计
2. **数据同步问题**：两套系统数据不同步，状态不一致
3. **场景设计问题**：web端没有考虑"移动办公"场景

## 三、工厂特殊场景需求分析

### 3.1 网络不稳定环境

| 需求 | 当前状态 | 问题 | 建议方案 |
|------|---------|------|---------|
| **离线模式** | ❌ 无 | 工厂车间经常断网，操作丢失 | 本地缓存+断点续传 |
| **离线数据队列** | ❌ 无 | 断网时无法保存 | 先存本地，联网后自动上传 |
| **冲突检测** | ❌ 无 | 离线修改后联网可能覆盖他人数据 | 乐观锁/版本号检测 |
| **网络状态提示** | ⚠️ 弱 | 用户不知道当前是否联网 | 顶部状态条显示网络状态 |

**工厂实况**：
- 车间里有大型电机，启动时干扰WiFi
- 梅雨季节雷击多，容易断电
- 郊区基站信号差，4G也不稳定

### 3.2 车间视角特殊需求

| 需求 | 当前状态 | 问题 | 建议方案 |
|------|---------|------|---------|
| **大字/高对比度** | ❌ 无 | 车间屏幕远、光线差看不清 | 提供车间模式（字号放大2倍） |
| **扫码快速录入** | ⚠️ 仅工序报工 | 其他模块还是手工输入 | 全模块支持扫码 |
| **一键操作** | ❌ 无 | 工人不会操作复杂界面 | 大按钮+一步完成 |
| **语音播报** | ❌ 无 | 工人不看屏幕 | 新任务到达时语音播报 |
| **手势操作** | ❌ 无 | 触屏车间平板 | 支持滑动、长按手势 |

### 3.3 批量操作需求

| 需求 | 当前状态 | 问题 | 建议方案 |
|------|---------|------|---------|
| **批量排产** | ❌ 无 | 一次排多个订单要一个个点 | 支持勾选+批量发布 |
| **批量发货** | ❌ 无 | 发货单要一个个录 | Excel导入批量发货 |
| **批量质检** | ❌ 无 | 同批次产品要一个个录 | 扫码+批量确认 |
| **批量归档** | ✅ 有 | - | 已有功能，保持 |
| **批量打印** | ❌ 无 | 工单、标签要一个个打印 | 勾选+批量打印 |

### 3.4 即时消息推送需求

| 需求 | 当前状态 | 问题 | 建议方案 |
|------|---------|------|---------|
| **订单状态变更通知** | ⚠️ 仅微信 | 微信推送容易被忽略 | 增加APP内推送+短信 |
| **逾期预警通知** | ❌ 无 | 订单快逾期没人知道 | 提前3天/1天推送提醒 |
| **物料缺料预警** | ❌ 无 | 物料不够了才告知 | 备料不足提前通知 |
| **质检异常通知** | ❌ 无 | 不良品出了才告知 | 实时推送质检结果 |
| **物流异常通知** | ⚠️ 仅追踪页面 | 物流异常不会主动告知 | 物流异常时自动推送 |

### 3.5 权限分级需求

| 角色 | 桌面端权限 | Web5001权限 | 建议 |
|------|-----------|------------|------|
| **工人** | 报工、工序查看 | 工序报工 | 简化界面，只显示自己任务 |
| **组长** | 本组工序管理 | 本组数据查看 | 增加"我的班组"视图 |
| **主管** | 全模块管理 | 仅查看 | 开放更多管理功能 |
| **厂长** | 全权限 | 仅查看 | 增加数据看板权限 |
| **跟单员** | 订单管理 | 仅查看 | 增加订单编辑权限 |

## 五、用户体验改善建议

### 5.1 信息架构重组

```
跟单员视角（以订单为中心）
├── 订单列表（筛选：状态/客户/交期）
├── 订单详情（基本信息+生产进度时间轴）
├── 快速操作（确认/排产/质检/发货）
└── 订单预警（逾期/即将逾期）

生产主管视角（以工单为中心）
├── 今日任务看板
├── 产能利用率图
├── 工序进度追踪
└── 异常工单告警

仓库视角（以物料为中心）
├── 缺料预警列表
├── 入库扫码
├── 物料追溯
└── 批次管理

质检视角（以质检单为中心）
├── 待检任务列表
├── 质检标准库
├── 不良品记录
└── 质检统计
```

### 5.2 关键交互优化

| 场景 | 现状 | 建议 |
|------|------|------|
| 查看订单进度 | 打开订单详情，再找关联工单，再找报工记录 | 订单详情页顶部加"进度时间轴"，一目了然 |
| 新建发货单 | 选择成品→填物流→填运单 | 扫码运单自动填充+历史记录联想 |
| 批量排产 | 一个个点"发布任务" | 勾选多个→一键发布+进度条 |
| 查看逾期订单 | 筛选状态→找交期列→人工判断 | 加"逾期"标签+红色高亮+顶部预警条 |

## 六、技术架构建议

### 6.1 Web5001独立化改造

```
当前架构（桌面端附属）：
┌─────────────┐
│  Web5001    │ ← 桌面端的阉割版，数据同步问题
└──────┬──────┘
       │ 同步
┌──────▼──────┐
│  桌面端5001 │ ← 数据源
└─────────────┘

建议架构（独立设计）：
┌─────────────┐     ┌─────────────┐
│  Web5001    │     │  桌面端5001 │
└──────┬──────┘     └──────┬──────┘
       │ 共享              │ 共享
       └────────┬─────────┘
                ▼
        ┌───────────────┐
        │   MySQL DB    │ ← 单一数据源
        └───────────────┘
```

## 七、总结

### 7.1 核心发现

1. **Web5001是半成品**：功能严重阉割，无法独立支撑业务
2. **信息孤岛严重**：订单→工单→报工→质检→发货，数据不贯通
3. **工厂特殊场景未考虑**：网络不稳定、大字需求、扫码需求被忽视
4. **角色视角缺失**：没有"工人视角"、"主管看板"等专属视图

### 7.2 行动建议

| 阶段 | 时间 | 行动 |
|------|------|------|
| 短期(1个月) | P0功能 | Web端订单管理+进度追踪+批量发货+运单扫码 |
| 中期(2个月) | P1功能 | 交期预警+产能看板+物料计算优化+质检标准 |
| 长期(3个月) | P2/P3功能 | 离线模式+车间大字+语音播报+完整报表 |

### 7.3 关键成功指标

| 指标 | 当前值 | 目标值 |
|------|-------|-------|
| 跟单员电话追问次数/天 | 30+ | <10 |
