### 🔴 H6: 多实例部署方案未验证

**场景**：
- 部署 2 个 5002 实例（HA）
- outbox worker 会**重复消费**

**问题**：
- 文档提到多实例
- 但 outbox worker **无分布式锁**
- 重复消费会导致镜像表 REPLACE INTO 触发 race condition

**修复**：
- 用 SELECT ... FOR UPDATE SKIP LOCKED
- 或用 Redis 分布式锁
- 或只在一个实例上跑 worker

## 七、关键洞察

> **方案不仅是写代码，更是写文档**：
>
> 58 个修复让代码稳定，但**没人能独立运维**：
>
> - 错误码字典定义了但代码没用
> - 告警规则写了但监控没配
> - 性能基线是模板没采集
> - 备份/恢复/容量完全没规划
>
> **方案成熟度 = 代码质量 + 文档完整度**。
> 当前方案真实成熟度只有 **49%**（不是 80%）。
> **文档/运维/容灾**是下一个阶段的关键。
