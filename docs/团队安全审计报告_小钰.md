## B. JWT Secret 硬编码漏洞

| # | 位置 | 风险等级 | 代码片段 | 修复方案 |
|---|------|---------|---------|---------|
| 1 | server.py:L32 | 🔴严重 | `app.secret_key = os.getenv('JWT_SECRET_KEY', 'desktop-web-dev-secret')` | 必须设置环境变量，禁止默认值 |

### 详细分析

**漏洞代码**:
```python
# server.py:L32
app.secret_key = os.getenv('JWT_SECRET_KEY', 'desktop-web-dev-secret')
```

**风险说明**: 如果生产环境未设置 `JWT_SECRET_KEY` 环境变量，Flask session 将使用默认的 `desktop-web-dev-secret`，攻击者可以伪造任意 session。

✅ **修复方案**:
```python
import secrets

jwt_secret = os.getenv('JWT_SECRET_KEY')
if not jwt_secret:
    raise RuntimeError("JWT_SECRET_KEY 环境变量必须设置！禁止使用默认值。")
app.secret_key = jwt_secret
```

## C. Token生成不安全

| # | 位置 | 风险等级 | 代码片段 | 修复方案 |
|---|------|---------|---------|---------|
| 1 | server.py:L542 | 🔴严重 | `token = base64.b64encode(f"{user['id']}:{user['name']}".encode()).decode()` | 使用 PyJWT 或 secrets.token_hex |

### 详细分析

**漏洞代码**:
```python
# server.py:L534-543
@app.route('/api/login', methods=['POST'])
def api_login():
    ...
    if r.status_code == 200 and body.get('code') == 0:
        user = body['data']
        import base64
        token = base64.b64encode(f"{user['id']}:{user['name']}".encode('utf-8')).decode('ascii')
        session['dispatch_token'] = token
```

**风险说明**: Token 仅使用 base64 编码用户ID和姓名，任何人都可以轻易伪造。攻击者只需知道用户ID即可伪造有效token。

✅ **修复方案**:
```python
import secrets

@app.route('/api/login', methods=['POST'])
def api_login():
    ...
    if r.status_code == 200 and body.get('code') == 0:
        user = body['data']
        # 使用安全的随机token
        token = secrets.token_hex(32)
        session['dispatch_token'] = token
        session['dispatch_user'] = user
```

## E. CSRF保护缺失

| # | 范围 | 风险描述 | 风险等级 | 修复方案 |
|---|------|---------|---------|---------|
| 1 | 所有 POST/PUT/DELETE API | 无CSRF Token验证 | 🟡中 | 使用Flask-WTF或自定义CSRF中间件 |

### 修复方案

```python
from functools import wraps

CSRF_TOKEN_KEY = '_csrf_token'

def generate_csrf_token():
    """生成CSRF Token"""
    if CSRF_TOKEN_KEY not in session:
        session[CSRF_TOKEN_KEY] = secrets.token_hex(16)
    return session[CSRF_TOKEN_KEY]

def csrf_protect(f):
    """CSRF保护装饰器"""
    @wraps(f)
    def decorated_function(*args, **kwargs):
        if request.method in ('POST', 'PUT', 'DELETE', 'PATCH'):
            token = session.get(CSRF_TOKEN_KEY)
            request_token = request.headers.get('X-CSRF-Token') or \
                           request.form.get(CSRF_TOKEN_KEY) or \
                           request.json.get(CSRF_TOKEN_KEY) if request.is_json else None
            if not token or token != request_token:
                return jsonify({'code': 403, 'message': 'CSRF验证失败'}), 403
        return f(*args, **kwargs)
    return decorated_function

# 注册全局CSRF token生成
@app.before_request
def setup_csrf():
    if CSRF_TOKEN_KEY not in session:
        session[CSRF_TOKEN_KEY] = secrets.token_hex(16)

# 在所有敏感路由上使用
@app.route('/api/orders/create', methods=['POST'])
@csrf_protect
def api_orders_create():
    ...
```

## F. XSS漏洞

| # | 页面 | 位置 | 风险等级 | 代码片段 | 修复方案 |
|---|------|------|---------|---------|---------|
| 1 | process_admin.html | L303 | 🟢低 | `data-process='${JSON.stringify(rowData)}'` | 对属性值转义 |
| 2 | 所有模板 | 多处 | 🟢低 | onclick属性中的变量 | 确保变量已转义 |

### 详细分析

**风险点1** (process_admin.html:L303):
```html
<tr data-id="${p.id}" data-process='${JSON.stringify(rowData).replace(/'/g, "&#39;")}' ...>
```

**说明**: 虽然使用了 `replace(/'/g, "&#39;")`，但这不能防止所有XSS攻击。建议使用更安全的JSON序列化方式。

✅ **修复方案**:
```javascript
// 使用 data-* 属性存储时，先进行HTML转义
function escapeAttr(str) {
    return String(str).replace(/"/g, '&quot;').replace(/'/g, '&#39;');
}

// 模板中使用
<tr data-process='${escapeAttr(JSON.stringify(rowData))}' ...>
```
