## 二、回滚条件

### 2.1 立即回滚条件

| 指标 | 阈值 | 动作 |
|------|------|------|
| API 错误率 | > 1% | 立即回滚 |
| 响应时间 | > 3s | 立即回滚 |
| 前端报错 | > 5次/10min | 立即回滚 |
| 数据丢失 | 任意 | 立即回滚 |

### 2.2 监控命令

```bash
# 查看 5008 日志错误
tail -n 100 /path/to/logs/5008.log | grep -i error

# 查看 API 响应时间
curl -o /dev/null -s -w "%{time_total}s\n" http://localhost:5008/api/schedule/list

# 查看数据库连接
SHOW STATUS LIKE 'Threads_connected';
SHOW STATUS LIKE 'Max_used_connections';
```

## 四、异常处理

### 4.1 部分功能异常

如果某个功能异常，但其他功能正常：
1. 保留当前灰度比例
2. 修复问题后继续放量
3. 不同功能独立放量

### 4.2 全部回滚

如果出现严重问题：
1. 执行 `git stash && git checkout HEAD~1`
2. 通知相关人员
3. 分析问题根因
4. 修复后重新灰度
