# 库存数据库统一方案

## 现状
- 嵌入式库存: steel_belt.inventory 表 (MySQL, 1个表)
- 独立WMS: inventory_db 数据库 (MySQL, 独立库, 6+表)
- 两套数据不互通, 存在不一致风险

## 统一方案
1. Phase 1: 分析 inventory_db 表结构, 找出与 steel_belt.inventory 的映射关系
2. Phase 2: 编写数据迁移脚本 `scripts/migrate_inventory.py`
   - 从 inventory_db 读取数据
   - 转换字段映射 (name→material_name, 等)
   - 去重后写入 steel_belt.inventory
3. Phase 3: 更新 InventoryView 只读 steel_belt
4. Phase 4: 独立WMS改为只读模式 (保留历史查询)
5. Phase 5: 停机窗口执行迁移, 验证数据完整性

## 风险
- 数据量未知, 迁移可能耗时
- 字段映射可能不完全
- 独立WMS客户端需更新连接配置
- 建议: 先在测试环境验证完整流程
