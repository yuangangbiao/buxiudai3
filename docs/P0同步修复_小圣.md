## 问题描述

web5001 的工序操作（报工/开始/完成/重置）**不触发 SYNC_BRIDGE 同步**，导致：
- 工人在 web5001 报工后
- 桌面端看不到报工数据
- 桌面端的 SYNC_BRIDGE 接收不到通知

## 修复方案

添加统一的 `_sync_bridge()` 函数，在工序操作成功后发送同步通知到 SYNC_BRIDGE。

---

## 1. 添加的同步函数

**位置**: `desktop_web/server.py` 行 73-84

```python
def _sync_bridge(data):
    """向SYNC_BRIDGE发送同步通知，失败不阻断主流程"""
    try:
        url = os.getenv('SYNC_BRIDGE_URL')
        if not url:
            return
        data['source'] = 'web5001'
        data['timestamp'] = datetime.now().isoformat()
        requests.post(url, json=data, timeout=5)
    except Exception as e:
        logger.warning(f'[SYNC_BRIDGE] 同步失败: {e}')
```

## 3. 同步数据格式

每个工序操作发送的同步数据包含：

| 字段 | 说明 | 来源 |
|------|------|------|
| `type` | 同步类型 | `process_start` / `process_complete` / `process_reset` / `process_report` |
| `process_id` | 工序ID | URL参数 |
| `production_id` | 生产单ID | 数据库 `old['production_id']` |
| `source` | 来源 | 固定值 `web5001` |
| `timestamp` | 时间戳 | `datetime.now().isoformat()` |

报工额外字段：

| 字段 | 说明 |
|------|------|
| `completed_qty` | 累计完成数量 |
| `qualified_qty` | 累计合格数量 |
| `status` | 新状态 |
| `worker` | 报工工人 |

## 4. 环境配置

需要在环境变量中配置 SYNC_BRIDGE_URL：

```bash
SYNC_BRIDGE_URL=http://127.0.0.1:5003/api/sync-bridge/receive
```

**注意**: 同步失败不会阻断主流程（try/except 包裹）
