# 排产流程规范 v1.0

## 一、流程概述

```
┌─────────────────┐
│   桌面生成端     │  生成工单
└────────┬────────┘
         │ 发布消息到容器中心
         ↓
┌─────────────────┐
│   容器中心       │  接收工单，通知生产部门
└────────┬────────┘
         │ 发送排产制定通知
         ↓
┌─────────────────┐
│   生产部门       │  制定排产信息
└────────┬────────┘
         │ 提交排产信息到容器中心
         ↓
┌─────────────────┐
│   容器中心       │  通知桌面生成端
└────────┬────────┘
         │ 发送排产确认通知
         ↓
┌─────────────────┐
│   桌面生成端     │  确认排产
└─────────────────┘
```

## 二、流程阶段定义

### 阶段1：工单发布 (ORDER_PUBLISH)
| 属性 | 说明 |
|------|------|
| 触发者 | 桌面生成端 |
| 动作 | 生成工单，发布到容器中心 |
| 状态 | `published` |
| 数据 | 工单基本信息、产品信息、数量、交期 |

### 阶段2：排产通知 (SCHEDULE_NOTIFY)
| 属性 | 说明 |
|------|------|
| 触发者 | 容器中心 |
| 动作 | 向生产部门发送排产制定通知 |
| 状态 | `waiting_schedule` |
| 数据 | 工单信息、要求完成时间 |

### 阶段3：排产制定 (SCHEDULE_CREATE)
| 属性 | 说明 |
|------|------|
| 触发者 | 生产部门 |
| 动作 | 制定排产信息（工序、人员、时间） |
| 状态 | `scheduled` |
| 数据 | 工序列表、人员分配、预计完成时间 |

### 阶段4：排产确认 (SCHEDULE_CONFIRM)
| 属性 | 说明 |
|------|------|
| 触发者 | 桌面生成端 |
| 动作 | 确认排产信息 |
| 状态 | `confirmed` / `rejected` |
| 数据 | 确认结果、修改意见（可选） |

## 三、状态机定义

```
                    ┌──────────────┐
                    │   CREATED    │  工单创建
                    └──────┬───────┘
                           │ 发布到容器中心
                           ↓
                    ┌──────────────┐
         ┌─────────│  PUBLISHED   │  已发布
         │         └──────┬───────┘
         │                │ 通知生产部门
         │                ↓
         │         ┌──────────────┐
         │  超时   │WAITING_SCHD  │  等待排产
         │  重发   └──────┬───────┘
         │                │ 收到排产信息
         │                ↓
         │         ┌──────────────┐
         │         │  SCHEDULED   │  已排产
         │         └──────┬───────┘
         │                │ 通知桌面端
         │                ↓
         │         ┌──────────────┐
         │         │SCHED_NOTIFIED│  排产已通知
         │         └──────┬───────┘
         │                │ 确认/拒绝
         │                ↓
         │         ┌──────────────┐
         │         │  CONFIRMED   │  已确认
         │         └──────────────┘
         │
         │         ┌──────────────┐
         └────────→│  REJECTED    │  已拒绝（可重新排产）
                   └──────────────┘
```

## 四、消息类型定义

### 4.1 工单发布消息 (ORDER_PUBLISH)
```json
{
  "msg_type": "ORDER_PUBLISH",
  "order_no": "ORD-2024-001",
  "product_name": "不锈钢网带",
  "quantity": 1000,
  "delivery_date": "2024-12-31",
  "priority": "normal",
  "source": "desktop",
  "published_at": "2024-01-15T10:00:00"
}
```

### 4.2 排产通知消息 (SCHEDULE_NOTIFY)
```json
{
  "msg_type": "SCHEDULE_NOTIFY",
  "order_no": "ORD-2024-001",
  "schedule_required_by": "2024-01-16T18:00:00",
  "recipient": "production_dept",
  "notified_at": "2024-01-15T10:05:00"
}
```

### 4.3 排产提交消息 (SCHEDULE_SUBMIT)
```json
{
  "msg_type": "SCHEDULE_SUBMIT",
  "order_no": "ORD-2024-001",
  "schedule": {
    "processes": [
      {"name": "来料", "worker": "张三", "start": "01-20", "end": "01-21"},
      {"name": "编织", "worker": "李四", "start": "01-22", "end": "01-25"},
      {"name": "质检", "worker": "王五", "start": "01-26", "end": "01-26"}
    ],
    "total_days": 7,
    "estimated_complete": "2024-01-27"
  },
  "submitted_by": "production_dept",
  "submitted_at": "2024-01-15T14:00:00"
}
```

### 4.4 排产确认消息 (SCHEDULE_CONFIRM)
```json
{
  "msg_type": "SCHEDULE_CONFIRM",
  "order_no": "ORD-2024-001",
  "result": "confirmed",
  "confirmed_by": "desktop_user",
  "confirmed_at": "2024-01-15T15:00:00",
  "comments": "排产时间可接受"
}
```

### 4.5 排产拒绝消息 (SCHEDULE_REJECT)
```json
{
  "msg_type": "SCHEDULE_REJECT",
  "order_no": "ORD-2024-001",
  "result": "rejected",
  "reason": "编织工序时间过长，需要压缩",
  "rejected_by": "desktop_user",
  "rejected_at": "2024-01-15T15:00:00"
}
```

## 五、通知规则

### 5.1 通知触发条件
| 事件 | 通知对象 | 通知方式 |
|------|----------|----------|
| 工单发布 | 生产部门 | 企业微信/邮件 |
| 排产超时提醒 | 生产部门 | 企业微信/邮件 |
| 排产已提交 | 桌面生成端 | 企业微信/页面提示 |
| 排产已确认 | 生产部门 | 企业微信 |
| 排产被拒绝 | 生产部门 | 企业微信 |

### 5.2 超时规则
| 阶段 | 超时时间 | 动作 |
|------|----------|------|
| 发布→等待排产 | 30分钟 | 首次提醒 |
| 等待排产→已排产 | 24小时 | 超时提醒 |
| 排产→确认 | 2小时 | 提醒确认 |

## 六、数据存储

### 6.1 排产记录表 (schedule_records)
```sql
CREATE TABLE schedule_records (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  order_no VARCHAR(50) NOT NULL,
  status VARCHAR(20) NOT NULL DEFAULT 'published',
  schedule_data TEXT,
  published_at DATETIME,
  notified_at DATETIME,
  submitted_at DATETIME,
  confirmed_at DATETIME,
  confirmed_by VARCHAR(50),
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 6.2 排产流程日志表 (schedule_flow_logs)
```sql
CREATE TABLE schedule_flow_logs (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  order_no VARCHAR(50) NOT NULL,
  event_type VARCHAR(30) NOT NULL,
  event_data TEXT,
  operator VARCHAR(50),
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 七、API接口设计

### 7.1 工单发布
```
POST /api/schedule/publish
```

### 7.2 获取排产状态
```
GET /api/schedule/status/<order_no>
```

### 7.3 提交排产
```
POST /api/schedule/submit
```

### 7.4 确认/拒绝排产
```
POST /api/schedule/confirm
```

### 7.5 获取待排产工单列表
```
GET /api/schedule/pending
```

## 八、流程监控指标

| 指标 | 计算方式 | 目标值 |
|------|----------|--------|
| 排产及时率 | 按时完成排产数 / 总工单数 | ≥95% |
| 平均排产周期 | 从发布到提交的平均小时数 | ≤4小时 |
| 确认响应时间 | 从通知到确认的平均分钟数 | ≤30分钟 |
