# ALIGNMENT_企业微信应用机器人.md

## 项目名称
企业微信应用机器人

---

## 一、项目上下文分析

### 1.1 现有项目结构

| 模块 | 说明 |
|------|------|
| `wechat_work_bot_v2.py` | 企业微信群机器人服务器（Flask） |
| `wechat_app_bot.py` | 企业微信应用机器人核心模块 |
| `container_center_v5.py` | 容器中心（任务池） |
| `integration/wechat_notifier.py` | 企业微信通知集成服务 |

### 1.2 现有实现问题

| 问题 | 说明 |
|------|------|
| **代码分散** | 群机器人和应用机器人分散在多个文件 |
| **功能重复** | `send_text` 等方法在多个地方重复实现 |
| **缺乏统一封装** | 没有统一的 Bot 接口 |
| **配置分散** | 部分硬编码OPERATORS等常量 |

### 1.3 技术栈

| 组件 | 技术 |
|------|------|
| 后端框架 | Flask |
| 企业微信API | 企微应用机器人 Webhook |
| 消息格式 | 文本、Markdown、图文 |
| 存储 | SQLite（复用 storage_layer） |

## 二、需求理解

### 2.1 原始需求

用户请求：**制作企业微信应用机器人**

### 2.2 需求边界确认

| 功能点 | 范围 | 说明 |
|--------|------|------|
| 核心能力 | 发送消息到企业微信 | 支持文本、Markdown、图文 |
| 消息接收 | 接收并处理用户指令 | 如报工、查询任务等 |
| 任务通知 | 从容器中心获取任务并通知 | 任务分配、状态变更通知 |
| 指令解析 | 解析用户发送的指令 | 报工、确认、完成等 |
| 会话管理 | 多用户会话跟踪 | 用户状态机管理 |

### 2.3 不在本次范围

| 功能点 | 原因 |
|--------|------|
| 企业微信小程序 | 已有单独文档 |
| 移动端APP | 已有单独实现 |
| 客服对话功能 | 如需请单独提出 |

---

## 三、需求理解

### 3.1 对现有项目的理解

1. **容器中心** (`container_center_v5.py`) 已实现任务池功能，支持持久化
2. **企业微信机器人** 已有基础实现，但代码分散
3. **报工流程** 已有完整流程设计（参考移动端报工系统设计报告）

### 3.2 架构设计原则

| 原则 | 说明 |
|------|------|
| 模块化 | 清晰的 Bot 基类和具体实现 |
| 可扩展 | 易于添加新的消息类型和指令 |
| 复用 | 复用 storage_layer、container_center 等现有模块 |
| 配置分离 | 所有配置通过 .env 管理 |

## 四、技术方案

### 4.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                    企业微信应用机器人                     │
├─────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐  │
│  │ BotFactory  │  │ MessageHub  │  │ CommandMgr   │  │
│  │ 机器人工厂   │  │ 消息中心    │  │ 指令管理器   │  │
│  └─────────────┘  └─────────────┘  └─────────────┘  │
├─────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────┐  │
│  │              Bot 实现层                          │  │
│  │  ┌────────────┐  ┌────────────┐  ┌──────────┐ │  │
│  │  │GroupBot    │  │AppBot      │  │CorpBot    │ │  │
│  │  │群机器人    │  │应用机器人  │  │企业机器人│ │  │
│  │  └────────────┘  └────────────┘  └──────────┘ │  │
│  └─────────────────────────────────────────────────┘  │
├─────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐  │
│  │ Container   │  │ Storage     │  │ Notifier    │  │
│  │ 容器中心     │  │ 存储层      │  │ 通知服务    │  │
│  └─────────────┘  └─────────────┘  └─────────────┘  │
└─────────────────────────────────────────────────────────┘
```

### 4.2 核心模块

| 模块 | 职责 |
|------|------|
| `BotFactory` | 统一创建和管理不同类型机器人 |
| `BaseBot` | 机器人基类，定义通用接口 |
| `GroupBot` | 群机器人实现（Incoming Webhook） |
| `AppBot` | 应用机器人实现（支持用户识别） |
| `MessageHub` | 消息路由和分发 |
| `CommandManager` | 指令解析和执行 |

## 五、验收标准
