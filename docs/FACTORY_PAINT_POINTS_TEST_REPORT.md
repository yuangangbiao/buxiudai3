## 二、工厂5类角色痛点

### 2.1 工人（报工员）

| 痛点 | 场景 | 影响 |
|------|------|------|
| 网络不稳定 | 工厂车间WiFi差，提交报工失败 | 报工数据丢失 |
| 误操作无撤销 | 点错工单或数量，无法撤回 | 每天约10次误操作 |
| 界面不友好 | 工人文化水平低，看不懂表格 | 拒绝使用系统 |
| 等待时间长 | 高峰期（每月20号）系统卡顿 | 影响生产节奏 |

### 2.2 跟单员

| 痛点 | 场景 | 影响 |
|------|------|------|
| 交期追踪难 | 订单状态不实时 | 客户催单时手忙脚乱 |
| 信息不同步 | 车间报工后，办公室看不到 | 反复核对 |
| 批量处理缺失 | 一次处理多个订单只能逐个 | 每天浪费1-2小时 |

### 2.3 生产主管

| 痛点 | 场景 | 影响 |
|------|------|------|
| 看不懂数据 | 系统输出的图表太专业 | 凭经验判断 |
| 预警不及时 | 订单逾期后才通知 | 已造成损失才知道 |
| 设备状态不透明 | 不知道机器是否空闲 | 排产不合理 |

### 2.4 质检员

| 痛点 | 场景 | 影响 |
|------|------|------|
| 质检结果通知慢 | 纸质记录，流转慢 | 下一道工序等材料 |
| 不合格品处理乱 | 返工/报废流程不清 | 库存数据不准 |
| 质检标准不统一 | 不同质检员判断不同 | 客户投诉 |

### 2.5 仓库管理员

| 痛点 | 场景 | 影响 |
|------|------|------|
| 物料找不到 | 库存数据不准 | 影响生产 |
| 采购无依据 | 不知道何时该补货 | 库存积压或断货 |
| 盘点困难 | 纸质记录，盘点耗时 | 每月浪费2天 |

## 三、技术可行性评估

| 痛点 | 技术方案 | 可行性 | 风险 |
|------|---------|-------|------|
| 断网离线 | 本地SQLite队列 + 恢复后同步 | ✅ | 数据冲突解决 |
| 5分钟撤销 | 状态机 + undo_log表 | ✅ | 并发下撤销冲突 |
| 交期预警 | 定时任务扫描 + 消息通知 | ✅ | 性能（大量订单）|
| 批量操作 | 批量API + 事务保证 | ✅ | 部分成功处理 |
| 高峰期预案 | 连接池扩容 + 限流 | ⚠️ | CI资源限制 |

## 六、验收标准

### 6.1 断网离线

- [ ] 断网时工人能正常操作
- [ ] 断网30分钟内数据完整
- [ ] 恢复网络后10秒内同步
- [ ] 并发修改时提示用户确认

### 6.2 5分钟撤销

- [ ] 报工后5分钟内可撤销
- [ ] 只能撤销自己的报工
- [ ] 撤销后状态正确回滚
- [ ] 撤销操作写入日志

### 6.3 交期预警

- [ ] 提前3天预警
- [ ] 微信/短信/系统通知
- [ ] 订单号+客户+逾期日期
- [ ] 同订单不重复发预警

### 6.4 批量操作

- [ ] 支持1-100个订单
- [ ] 成功的记录成功，失败的提示
- [ ] 全部成功或全部失败
- [ ] 100个订单≤5秒

### 6.5 工人端验收

- [ ] 手机端≤3步完成报工
- [ ] 全中文，无英文/代码
- [ ] 操作后有明确成功/失败提示
- [ ] 断网时显示友好提示
