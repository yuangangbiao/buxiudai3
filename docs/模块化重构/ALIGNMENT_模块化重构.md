## 一、项目和任务特性规范

### 1.1 项目概述

| 项目 | 内容 |
|------|------|
| 项目名称 | 不锈钢网带跟单系统 3.1 模块化重构 |
| 项目类型 | 架构重构 + 模块独立化 |
| 核心目标 | 将改进计划中的功能拆分为独立模块，主系统暂不集成 |

| 组件 | 技术 |
|------|------|

| 现有架构 | Tkinter（GUI）+ MySQL/SQLite |
| 目标架构 | 模块化 + 事件驱动 |
| 依赖模块 | mobile_api_ai（已有熔断器、队列、健康检查等） |

### 1.3 代码规范遵循

**已确认规范（依据 `CODING_STANDARDS.md`）**：

| 规范项 | 要求 |
|--------|------|
| 敏感信息 | 禁止硬编码密码/API密钥，使用 `.env` 管理 |
| 阈值配置 | 禁止硬编码阈值（如连接数、超时），从环境变量读取 |
| 路径管理 | 禁止分散硬编码路径，使用 `config.py` 统一管理 |
| sys.path | 只在入口文件设置一次 |
| 数据库操作 | 必须使用 context manager `get_db_cursor()` |
| 异常处理 | 禁止裸 `except:`，必须记录日志 |
| 日志规范 | 禁止 `print()`，使用 `logger = logging.getLogger(__name__)` |

## 四、需求理解

### 4.1 架构演进方向

```
现状架构：
┌─────────────────────────────────────────┐
│           桌面端主系统 3.1               │
├─────────────────────────────────────────┤
│  主窗口 (main_window.py)                │
│  ├─ 订单视图 (order_view.py)            │
│  ├─ 工序视图 (process_view.py)          │
│  ├─ ...                                 │
└─────────────────────────────────────────┘
        │
        ▼
目标架构（模块化后）：
┌─────────────────────────────────────────┐
│           桌面端主系统 3.1               │
├─────────────────────────────────────────┤
│  核心层 (core/)                          │
│  ├─ event_bus.py（已有）               │
│  ├─ events.py（新建）← 事件类型定义      │
│  └─ exceptions.py（已有）               │
├─────────────────────────────────────────┤
│  模块层（独立）                          │
│  ├─ auto_publish_service.py（新建）      │
│  ├─ material_publish_service.py（新建）  │
│  └─ modular_config.py（新建）           │
├─────────────────────────────────────────┤
│  集成层                                 │
│  ├─ container_event_listener.py（增强） │
│  └─ desktop_container_integration.py（增强）│
└─────────────────────────────────────────┘
```

### 4.2 模块职责划分

| 模块 | 职责 | 依赖 |
|------|------|------|
| `core/events.py` | 定义所有事件类型常量 | 无 |
| `container_event_listener.py` | 监听事件，调用容器发布 | event_bus, container_center |
| `desktop_container_integration.py` | 提供发布任务接口 | container_center_v5 |
| `auto_publish_service.py` | 自动发布逻辑（读取开关状态） | config, event_bus |
| `material_publish_service.py` | 备料发布逻辑 | config, event_bus, container |
| `modular_config.py` | 统一配置管理 | os, json |

### 4.3 数据流设计

```
【自动发布场景】
排产确认事件
  → EventBus.publish(PRODUCTION_CONFIRMED)
    → ContainerEventListener 收到事件
      → 读取 auto_publish 开关
        → 若开启：调用 DesktopContainerIntegration.publish_report_task()
          → 任务发布到容器池

【备料发布场景】
仓管点「发布用料需求」
  → 调用 MaterialPublishService.publish_requirements(order_id, process_id)
    → 收集已勾选备料项
      → 调用 ContainerCenter.publish_material_task()
        → 用料需求推送到移动端
```

## 六、验收标准

### 6.1 模块验收清单

| # | 模块 | 验收标准 |
|---|------|----------|
| 1 | `core/events.py` | 定义所有事件类型常量，可独立导入 |
| 2 | `container_event_listener.py` | 监听事件后可正确触发发布逻辑 |
| 3 | `desktop_container_integration.py` | `publish_report_task()` 方法可单独调用 |
| 4 | `auto_publish_service.py` | 读取开关状态，根据状态决定是否发布 |
| 5 | `material_publish_service.py` | 收集备料项并发布，无UI依赖 |
| 6 | `modular_config.py` | 统一管理模块配置，支持 `.env` 读取 |

### 6.2 技术验收清单

- [ ] 所有模块可通过 `python -c "import 模块名"` 单独导入
- [ ] 无硬编码密码、API密钥、阈值
- [ ] 无裸 `except:`
- [ ] 无 `print()` 调试
- [ ] 配置通过 `modular_config.py` 统一管理
- [ ] 单元测试覆盖核心方法

### 6.3 集成验收清单（后续）

- [ ] 模块可被主系统 views 调用
- [ ] 事件总线可正确触发各监听器
- [ ] 容器池可接收并分发任务
