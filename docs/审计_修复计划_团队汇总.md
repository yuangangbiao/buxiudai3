## 2. 共识问题（4 专家中 ≥3 个共同指出）

### 2.1 🔴 P0-1 修复方向错位（小圣+小贺+小钰 共同指出）
- **计划方向**: 改 `_call_dispatch` 转发层加状态校验
- **代码现状**（desktop_web/server.py:2228-2399）: `api_process_start/complete/reset/report` 4 个函数**全部直连 DB**，不走 `_call_dispatch`
- **真实问题**: 改错地方。真假成功在 `complete`/`reset` 的 `rowcount` 缺失（line 2278-2282, 2313-2316）
- **修复方案**: 改 4 个直连 DB 函数，不改 `_call_dispatch`

### 2.2 🔴 P0-3 修复方案错位（小圣+小贺+小钰 共同指出）
- **计划方向**: 加 `SELECT FOR UPDATE` 行锁
- **代码现状**（desktop_web/server.py:2348）: `api_process_report` **已有 `FOR UPDATE`**
- **真实问题**: 5 线程仍超额 60%，根因是 read-then-write 模式（line 2354: `new_completed = old_completed + qty`）
- **修复方案**: 改用数据库原子加法（`UPDATE ... SET completed_qty = completed_qty + %s`），不走应用层加减

### 2.3 🔴 P0-4 SYNC_BRIDGE 攻击面反而扩大（小圣+小钰 共同指出）
- **计划方向**: 5003 加接收端点 + 5001 真发 POST
- **代码现状**（server.py:73-83）: `_sync_bridge` 用 `requests.post` **无 HMAC、无 nonce、无 timestamp**
- **新风险**: 任意内网主机可伪造 `process_start/complete/report` 事件
- **修复方案**: 必须加 HMAC-SHA256 签名 + nonce 防重放 + 5min timestamp 窗口 + 双向认证

### 2.4 🔴 P2-12 admin 鉴权数量严重低估（小贺+小钰 共同指出）
- **计划数字**: 5 个 admin 路由
- **实测数字**: **20-30+ 写操作路由无 `@require_auth`**
  - L188 下单 / L2228 工序开始 / L2335 报工 / L2737 质检新增 / L2923 质检判定 / L3088 发货新增 等
- **P0-3 报工超额的根因**: 报工接口本身**无鉴权**（不止是行锁缺失）
- **修复方案**: 全部 30+ 写操作补 `@require_auth` + `@verify_csrf_token`

### 2.5 🔴 跨服务 token 协议不一致（小曦+小钰 共同指出）
- **5001 生成**（server.py:627）: `session['dispatch_token'] = _secrets.token_hex(32)`
- **5003 期望**（standalone_dispatch_server.py:115-122）: `X-Dispatch-Token = base64(uid:uname)`
- **后果**: 修复 P0-1 后**所有 `_call_dispatch` API 永远返 401**
- **修复前置**: 必须先统一 token 协议（5001 改生成方式 或 5003 改解析方式）

### 2.6 🔴 role='worker' 写死（小钰指出）
- **5003 登录返**（L152/158）: `role: 'worker'` 硬编码
- **后果**: P2-12 修复后 `@require_role('admin')` **永远 fail**
- **修复前置**: 必须先从 `operators` 表读 role

### 2.7 🟠 审计日志不合规（小钰+小曦 共同指出）
- **现状**: 30+ 处 `logger.info(f"[工序报工] id={process_id}")` 只记 ID
- **合规**: GMP/ISO 9001 要求记操作员 + 时间 + 前后值
- **修复方案**: 统一 `audit_log()` 函数 + 12 个关键操作清单 + DB `audit_logs` 表

1. **统一 token 协议**（5001 改生成 或 5003 改解析）

5. **SYNC_BRIDGE 加 HMAC + nonce + timestamp + 审计**

### Phase 2 - 业务 Bug 修复（13 项 P0/P1，3 天）
1. **P0-1 改 4 个直连 DB 函数**（不是改 _call_dispatch）
2. **P0-3 改数据库原子加法**（不是加 FOR UPDATE）
3. **P0-2 shipment 缺 import random**（5min）
4. **P1-5 IDOR 升 P0**
5. **P2-11 质检 SQL 异常外露 升 P0**
6. **P1-7/8/9/10 SQL 错**
7. **审计日志统一函数 + DB 表**

### Phase 3 - 体验优化（4 项 P2，1 天）
1. P2-13 静态资源 404
2. P2-14 质检前端校验
3. P2-15 空态引导
