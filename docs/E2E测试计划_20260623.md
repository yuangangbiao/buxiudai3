## 阶段 0：ALIGN（对齐）

### 0.1 测试目标

| 目标 | 验收标准 | 测量方式 |
|------|---------|---------|
| **业务流跑通** | 真实下单→发货全流程无中断 | 8 步骤全绿 |
| **跨服务同步** | 5001↔5003↔5008↔8008 数据一致 | 同步 Δ=0 |
| **权限闭环** | 未登录/低权限 100% 拒绝 | 401/403 全覆盖 |
| **并发安全** | 5 线程报工不超额 | quantity ≤ 计划 |
| **数据完整** | 订单状态机不跳级 | 状态机路径完整 |

### 0.2 范围边界

**包含**：
- ✅ HTTP API 端到端（5001/5003/5008/8008）
- ✅ 真实数据库操作（steel_belt + container_center）
- ✅ 跨服务 token 鉴权
- ✅ 业务流联动（订单→工艺→排产→报工→质检→发货）
- ✅ 失败重试 + 死信重放
- ✅ 监控埋点采集

**不包含**（先不测）：
- ⏸️ 桌面端 Tkinter UI（仅验证后端 API，UI 用 Playwright 单独计划）
- ⏸️ 微信企业号推送（云端依赖）
- ⏸️ 性能压测（另出压测计划）
- ⏸️ 跨地域灾备

## 阶段 1：DESIGN（设计）

### 1.1 测试架构

```
┌─────────────────────────────────────────────┐
│  E2E 测试框架（pytest + requests）            │
├─────────────────────────────────────────────┤
│  conftest.py 共享 fixture                     │
│  ├ admin_session()      admin 已登录 session │
│  ├ worker_session()     worker 已登录        │
│  ├ qc_session()         质检员已登录          │
│  ├ cleanup_e2e_data()   测试后清理 fixture   │
│  └ service_urls()       4 个服务 URL         │
├─────────────────────────────────────────────┤
│  tests/e2e/                                │
│  ├ test_01_auth.py        鉴权闭环           │
│  ├ test_02_order.py       订单全流程         │
│  ├ test_03_process.py     工艺/计算          │
│  ├ test_04_schedule.py    排产/调度          │
│  ├ test_05_workreport.py  报工/扫码          │
│  ├ test_06_quality.py     质检/复检          │
│  ├ test_07_shipment.py    发货/物流          │
│  ├ test_08_cost.py        成本/核算          │
│  ├ test_09_sync.py        跨服务同步         │
│  ├ test_10_concurrent.py  并发安全           │
│  └ test_11_metrics.py     监控埋点           │
└─────────────────────────────────────────────┘
```

### 1.2 关键设计原则

| 原则 | 实现 |
|------|------|
| **真实业务** | 不 mock 数据库，不 mock HTTP 客户端 |
| **业务流联动** | 跨 4 个服务连续调用，验证数据流转 |
| **失败注入** | 故意让一个服务失败，验证降级 + 重试 |
| **数据隔离** | 测试用 E2E_ 前缀数据，结束清理 |
| **可重复** | 跑完一次能再跑，幂等清理 |
| **可观测** | 每个用例打印关键状态码 + 响应 |

### 1.3 工具栈

- **pytest 9.0.3** — 测试框架
- **requests 2.34.2** — HTTP 客户端
- **pymysql 1.2.0** — 真实 DB 操作（清理/准备）
- **freezegun** — 时间冻结（如需）
- **allure-pytest** — 测试报告（可选）


```bash

pytest tests/e2e/test_10_concurrent.py -v

```

### 4.3 失败处理

| 失败类型 | 处理 |
|---------|------|
| 鉴权失败 | 暂停后续，先修 token / CSRF / 角色 |
| 单个 API 失败 | 记录 + 继续，结束后汇总 |
| 跨服务同步失败 | 暂停，先修 SYNC_BRIDGE |
| 并发超额 | 暂停，先补 TOCTOU 防护 |
| 数据库约束 | 清理残留 + 重跑 |
