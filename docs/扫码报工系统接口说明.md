# 扫码报工系统接口说明

**对接系统**：容器中心 API（端口 5002）  
**基地址**：`http://localhost:5002`  
**前端文件**：`晨圣报工系统.html`

---

## 通用响应格式

所有接口统一返回 JSON，结构如下：

### 成功响应
```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

### 失败响应
```json
{
  "code": 1,
  "message": "错误描述"
}
```

---

## 一、扫码查询工单信息

一站式查询工单基本信息 + 所有工序完成进度。

### 请求
```
GET /api/scan-info?code={工单号}
```

| 参数 | 位置 | 必填 | 说明 |
|------|------|------|------|
| code | query | 是 | 工单号（如 WO-202605006） |

### 响应 data 结构
```json
{
  "code": "WO-202605006",
  "order_no": "WO-202605006",
  "order_no": "",
  "customer_name": "客户名",
  "product_name": "产品名称",
  "quantity": 1000,
  "unit": "米",
  "delivery_date": "2026-06-01",
  "priority": "normal",
  "total_completed_qty": 350,
  "total_remaining_qty": 650,
  "processes": [
    {
      "process_id": "uuid-xxx",
      "process_name": "拉丝",
      "step_name": "拉丝",
      "required_qty": 1000,
      "completed_qty": 350,
      "remaining_qty": 650,
      "unit": "米",
      "status": "processing",
      "last_report_operator": "张三",
      "last_report_time": "2026-05-17T10:30:00",
      "last_report_qty": 50
    }
  ]
}
```

### 前端对接示例
```javascript
fetch('http://localhost:5002/api/scan-info?code=WO-202605006')
  .then(r => r.json())
  .then(ret => {
    if (ret.code !== 0) { /* 未找到 */ return; }
    const d = ret.data;
    // d.order_no       — 工单号
    // d.product_name   — 产品名
    // d.customer_name  — 客户名
    // d.quantity       — 需求总量
    // d.total_completed_qty — 已完成总量
    // d.processes      — 工序列表，每项包含 completed_qty / required_qty
  });
```

## 二、提交报工

对指定工序提交报工数量。

### 请求
```
POST /api/process_sub_step
Content-Type: application/json
```

### 请求体
```json
{
  "process_id": "工序ID（必填）",
  "order_no": "工单号（必填）",
  "step_name": "工序名称（必填）",
  "quantity": 50,
  "operator": "操作人",
  "remark": "备注（选填）"
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| process_id | 是 | 工序唯一标识（从 scan-info 返回的 processes[].process_id） |
| order_no | 是 | 工单号 |
| step_name | 是 | 工序名称（从 scan-info 返回的 processes[].step_name） |
| quantity | 是 | 本次报工数量（大于0） |
| operator | 否 | 操作人姓名 |
| remark | 否 | 备注 |

### 响应 data 结构
```json
{
  "code": 0,
  "message": "报工成功",
  "data": {
    "record": {
      "id": "uuid-xxx",
      "process_id": "uuid-xxx",
      "order_no": "WO-202605006",
      "step_name": "拉丝",
      "batch_no": "SHP-20260517-XXXXXX",
      "quantity": 50,
      "operator": "张三",
      "remark": "",
      "created_at": "2026-05-17T10:30:00"
    },
    "summary": {
      "process_id": "uuid-xxx",
      "total_qty": 350,
      "completed_qty": 350,
      "shipped_qty": 200,
      "remaining_qty": 650,
      "last_report_time": "2026-05-17T10:30:00"
    }
  }
}
```

> `record` — 本次报工记录详情  
> `summary` — 该工序累计汇总（含本次）

### 前端对接示例
```javascript
fetch('http://localhost:5002/api/process_sub_step', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    process_id: 'uuid-xxx',
    order_no: 'WO-202605006',
    step_name: '拉丝',
    quantity: 50,
    operator: '张三'
  })
})
  .then(r => r.json())
  .then(ret => {
    if (ret.code === 0) {
      // 报工成功，summary 含累计完成量
      const summary = ret.data.summary;
    }
  });
```

## 三、附加查询接口

### 3.1 查询工序子步骤列表
```
GET /api/process_sub_steps/{process_id}
```
返回该工序的所有报工记录列表。

### 3.2 查询工序报工汇总
```
GET /api/process_sub_step_summary/{process_id}
```
返回该工序的累计汇总数据。

---

## 对接流程

```
用户扫码/输入工单号
       │
       ▼
  GET /api/scan-info?code=WO-xxx
       │
       ├─ 返回工单信息 + 工序列表
       │
       ▼
  显示工序进度 → 用户选择工序报工
       │
       ▼
  POST /api/process_sub_step
       │
       ├─ 返回本次记录 + 累计汇总
       │
       ▼
  重新查询 goToOrder() 刷新页面
```

---

## 注意事项

1. **报工后刷新**：提交报工成功后，调用 `goToOrder(工单号)` 重新查询以刷新数据
2. **接口鉴权**：部分接口带 API Key 验证（`X-API-Key` 请求头），当前前端版本使用无需鉴权的接口
