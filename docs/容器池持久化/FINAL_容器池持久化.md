## 二、需求背景

**原始需求**：完成容器池持久化

**问题分析**：
- `TaskPool` 原始实现使用内存存储 (`self.tasks = {}`)
- 服务重启后所有任务数据丢失
- 需要改造为 SQLite 持久化

---

## 三、解决方案

### 3.1 架构改造

```
改造前：                           改造后：
┌─────────────────┐              ┌─────────────────┐
│   TaskPool      │              │   TaskPool      │
│   (内存存储)     │     →        │   (SQLite持久化) │
│                 │              │  ┌────────────┐ │
│  self.tasks = {}│              │  │self.storage│ │
│                 │              │  │self.tasks  │ │
└─────────────────┘              │  └────────────┘ │
                                 └─────────────────┘
```

### 3.2 核心改动

| 改动点 | 说明 |
|--------|------|
| 新增 `self.storage` | SQLiteStorage 实例 |
| 新增 `SimpleMemoryStorage` | 降级用简化内存存储 |
| 新增 `_task_to_package()` | Task → storage package 转换 |
| 新增 `_package_to_task()` | storage package → Task 转换 |
| 新增 `load_from_storage()` | 启动时加载任务 |
| 新增 `_save_task()` | 保存单个任务 |
| 修改公共方法 | 所有状态变更调用 `_save_task()` |
