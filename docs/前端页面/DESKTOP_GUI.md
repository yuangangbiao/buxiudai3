## 一、系统总览

### 1.1 系统定位

桌面 GUI 是不锈钢网带跟单系统的**原生 Windows 应用**,基于 Tkinter 构建,提供完整的离线操作能力,与桌面端 Web(5001)1:1 功能对齐,但运行更稳定、响应更快。

**设计目标**:
- 🖥️ 原生桌面体验
- 📊 大屏信息密度
- ⚡ 本地数据库直连,无网络依赖
- 🔧 高级配置与调试支持

- `AutoRefreshMixin` 每 30 秒自动刷新
- 弹窗打开时暂停刷新
- 窗口切换时重新挂载

## 五、视图懒加载机制

为加快启动速度,所有视图采用**懒加载**:

```python
# main_window.py
def show_module(self, module_id):
    # 销毁当前视图
    for widget in self.content_area.winfo_children():
        widget.destroy()
    
    # 按需导入(首次访问时才加载)
    if module_id == "orders":
        from desktop.views.order_view import OrderListView  # 延迟导入
        self.current_view = OrderListView(self.content_area)
    elif module_id == "production":
        from desktop.views.production_view import ProductionView
        self.current_view = ProductionView(self.content_area)
    # ...
```

#### 懒加载流程

```
启动应用
  ↓
主窗口加载(只加载主窗口 + 标题栏 + 侧边栏)
  ↓
100ms 后异步加载默认模块(订单管理)
  ↓
用户点击其他模块 → 按需加载对应视图
  ↓
切换模块时 → 销毁旧视图,加载新视图
```

#### 启动时间对比

| 模式 | 启动时间 |
|:-----|:--------:|
| 全部加载(优化前) | ~5s |
| 懒加载(当前) | **~1.5s** |

## 六、自动刷新机制

### 6.1 AutoRefreshMixin

所有需要自动刷新的视图继承 `AutoRefreshMixin`:

```python
# utils/auto_refresh_mixin.py
class AutoRefreshMixin:
    """自动刷新混入类"""
    refresh_interval = 30  # 秒
    
    def _start_auto_refresh(self):
        """启动定时刷新"""
        self._refresh_data()
        self._timer = self.after(self.refresh_interval * 1000, self._auto_tick)
    
    def _auto_tick(self):
        """定时器回调"""
        self._refresh_data()
        self._timer = self.after(self.refresh_interval * 1000, self._auto_tick)
    
    def _stop_auto_refresh(self):
        """停止定时器"""
        if self._timer:
            self.after_cancel(self._timer)
```

### 6.2 自动刷新生命周期

```
视图初始化 → _start_auto_refresh()
  ↓
30 秒后 → _refresh_data()
  ↓
弹窗打开时 → _stop_auto_refresh()(防止冲突)
  ↓
弹窗关闭时 → _start_auto_refresh()(恢复)
  ↓
视图销毁时 → 自动清理定时器
```

## 九、设计原则

### 9.1 懒加载优先

- 启动只加载主窗口
- 视图按需导入
- 启动时间从 5s 降到 1.5s

### 9.2 自动刷新

- AutoRefreshMixin 统一管理
- 30 秒间隔
- 弹窗时暂停

### 9.3 模态对话框

- 所有对话框 `grab_set()` 模态
- 阻止操作主窗口
- ESC 关闭

### 9.4 状态机保护

- 订单状态按规则流转
- 跨状态跳转需二次确认
- 归档后禁止修改

### 9.5 离线优先

- 直连本地 MySQL
- 无网络依赖
- 数据本地处理
