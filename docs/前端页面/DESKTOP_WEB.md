## 一、系统总览

### 1.1 系统定位

桌面端 Web 是对桌面 GUI(Tkinter)的**浏览器替代方案**,提供与 GUI 1:1 的业务功能,同时具备以下优势:

- ✅ 跨平台(Windows/Mac/Linux)
- ✅ 远程访问(局域网/公网)
- ✅ 多用户协作
- ✅ 易于维护升级
- ✅ 支持移动端响应式(< 768px)

### 1.2 页面布局三大区域

```
┌──────────────────────────────────────────────────────────────────┐
│ 📋 不锈钢网带跟单系统    👤 管理员  ⚙️ 设置  🚪 退出              │  ← 顶部 Navbar
├──────────────────────────────────────────────────────────────────┤
│ 📝订单 │🔍查询 │🏭排产 │📋备料 │🔧工序 │✅质检 │🚚发货 │📊看板 │  ← 二级 Tab 导航
│ 👤人员 │📊统计 │📋日志 │📦BOM │⚠预警 │📋导入导出 │              │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │

│                  (动态切换 19 个页面)                              │
```

## 三、公共导航与组件

### 3.1 顶部 Navbar

```

│ 🏭 不锈钢网带跟单系统 v3.0            👤 管理员  ⚙️ 设置  🚪 退出   │

```

| 元素 | 行为 |
|:-----|:-----|
| 系统标题 | 点击返回首页 `/orders` |
| 👤 用户名 | 显示当前登录用户(localStorage) |
| ⚙️ 设置 | 跳转 `/inventory/settings`(库存) |
| 🚪 退出 | 清空 token → 跳 `/login` |

### 3.2 二级 Tab 导航

14 个 Tab,根据页面动态高亮当前所在模块。

```javascript
// shared.js 渲染
function renderNav(activeKey, pageTitle) {
  // activeKey: 当前页对应的 key
  // pageTitle: 页面标题(显示在顶部)
}
```

### 3.3 公共工具栏模式

```html
<div class="toolbar">
  <select><option>状态筛选</option></select>
  <input type="text" placeholder="搜索...">
  <input type="date"> 至 <input type="date">
  <button>🔍 搜索</button>
  <button>🔄 刷新</button>
```

### 3.4 公共弹窗模式

- 标题 + 字段 + 取消/确认按钮
- 遮罩点击关闭(`if(event.target===this)closeModal()`)
- ESC 关闭
- 必填字段红星标识

```
│                                                                  │

```

#### 子步骤流程(5 步)

| 步骤 | 操作 | 说明 |
|:----:|:-----|:-----|
| 1 | 打开 `http://localhost:5001/login` | 进入登录页 |
| 2 | 输入用户名 | 必填,3-20 字符 |
| 3 | 输入密码 | 必填,6+ 字符,可点击 👁 切换可见 |
| 4 | 可选勾选"记住登录" | localStorage 存储,默认勾选 |
| 5 | 点击 [登 录 →] | POST `/api/auth/login`,成功后跳 `/orders` |

#### 错误处理

- 401 错误:显示"账号或密码错误"红色提示
- 5 次错误:账号锁定 5 分钟
- 网络错误:显示"请检查网络连接"

## 五、跨页面公共交互模式

### 5.1 通用提交流程

```
1. 填写表单
   ↓
2. 前端校验(必填、格式、范围)
   ↓
3. 点击提交按钮
   ↓
4. 按钮变为 loading 状态
   ↓
5. POST 请求(带 CSRF token)
   ↓
6a. 成功 → Toast 绿色提示 + 跳转/刷新
6b. 失败 → Toast 红色提示 + 保留表单
```

### 5.2 通用删除流程

```
1. 点击行 ··· 菜单的"删除"
   ↓
2. 弹出确认弹窗:"确认删除 XXX?"
   ↓
3. 点击 [确认删除]
   ↓
4. POST 请求
   ↓
5. 成功后刷新列表 + Toast 提示
```

### 5.3 通用导出流程

```
1. 筛选条件(可选)
   ↓
2. 点击 [📤 导出]
   ↓
3. 系统生成 Excel 文件
   ↓
4. 浏览器自动下载
   ↓
5. 文件名: {页面名}_{时间戳}.xlsx
```

### 5.4 通用批量操作流程

```
1. 多选(复选框)
   ↓
2. 工具栏显示 [批量操作] 按钮
   ↓
3. 选择批量操作类型
   ↓
4. 二次确认弹窗(危险操作)
   ↓
5. POST 批量请求
   ↓
6. 进度提示 + 结果反馈
```

## 九、设计原则

### 9.1 1:1 复刻 GUI

- 与桌面 Tkinter 视图功能一致
- 用户切换零成本

### 9.2 浏览器增强

- ✅ 自动刷新(避免手动)
- ✅ 响应式布局(多设备)
- ✅ 导出/导入(批量操作)
- ✅ 拖拽排序(效率提升)

### 9.3 集中导航

- 二级 Tab 固定顶部
- 任意页面快速跳转其他模块

### 9.4 安全防护

- CSRF token 所有 POST 请求
- XSS 转义所有 innerHTML
- JWT token 自动跳转登录
- 二次确认危险操作

### 9.5 错误友好

- 表单字段错误红色高亮
- 网络错误友好提示
- 失败重试一键操作
- 错误报告可下载
