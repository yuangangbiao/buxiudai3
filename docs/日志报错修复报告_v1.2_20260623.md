## 4. P1-R3: 订单号并发冲突（最关键）

**问题**：
```
pymysql.err.IntegrityError: (1062, "Duplicate entry 'ORD-20260623-0019' for key 'orders.order_no'")
```

**根因**：[models/database/utils_db.py:22-33](file:///d:/yuan/%E4%B8%8D%E9%94%88%E9%92%A2%E7%BD%91%E5%B8%A6%E8%B7%9F%E5%8D%953.0/models/database/utils_db.py#L22-L33)

```python
# 原版 (有 bug)
def generate_order_no():
    c.execute("SELECT COUNT(*) as cnt FROM orders WHERE order_no LIKE %s", (f"ORD-{today}-%",))
    row = c.fetchone()
    seq = row['cnt'] + 1  # ← 5 线程都读到 seq=1 → 都生成 ORD-XXX-0001
    return f"ORD-{today}-{seq:04d}"
```

**修复**（v1 microsecond + v2 加 pid）：

```python
# v2 修复 (无锁, 用 microsecond + pid 后缀保证唯一)
def generate_order_no():
    pid = os.getpid()
    for attempt in range(10):
        now = datetime.now()
        seq_part = f"{now.strftime('%H%M%S')}{now.microsecond:06d}"
        order_no = f"ORD-{today}-{seq_part}-{pid:04d}"
        # 校验不重复
        c.execute("SELECT 1 FROM orders WHERE order_no=%s", (order_no,))
        if c.fetchone() is None:
            return order_no
        _t.sleep(0.001)
    # 兜底
    fallback = f"ORD-{...}-{int(_t.time()*1e9) % 1000000000:09d}"
    return fallback
```
