# DESIGN - planned_qty_formula 公式与模板字段对齐修复

## 1. 修复目标

将 planned_qty_formula 公式统一为"带 `{}` 占位符 + 标准单位约定"的标准形式，并让 3 份模板的字段集合对齐到模板 2。

**不在本次范围**：质检规则 `process_name` 错位、`status_key_map` 中"成品入库"、去重逻辑、冗余赋值等。

## 2. 单位约定（业务决策）

| 字段 | 单位 | 录入位置 | 公式中应转为 |
|------|------|----------|--------------|
| 总长度 | 米 (m) | orders.extra_params | → `*1000` 转毫米 |
| 网带节距 | 毫米 (mm) | orders.extra_params | 不变 |
| 物料数量 | 件/项数 | order_materials 表（已存在） | calc_data 注入 key="物料数量" |

## 6. 数据流验证

```
订单录入 (orders.extra_params)
  ├─ 总长度=5 (米)
  ├─ 网带节距=25.4 (毫米)
  └─ 物料数量 = COUNT(order_materials where required_qty>0)

排产触发 ProcessCalcEngine.calculate_planned_qty
  ├─ formula = "{总长度}*1000/{网带节距}"
  ├─ calc_data = {总长度:5, 网带节距:25.4, ..., 物料数量:N}
  ├─ 替换 {总长度}→5, {网带节距}→25.4
  └─ _calc_expr("5*1000/25.4", data) = 196.85 → ceil = 197
```

**预期效果**：
- 修复前：模板 1 的"激光切板"公式 `"总长度/网带节距"` → `_calc_expr("总长度/网带节距", data)` → 字符串解析失败 → 0
- 修复后：`{总长度}*1000/{网带节距}` → 197（按 5m 长 + 25.4mm 节距）
