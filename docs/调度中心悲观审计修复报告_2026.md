## 二、问题详情

### 问题 1: 裸 except 静默忽略

**位置**: `dispatch_center/schedule_routes.py:234-239`

**当前代码**:
```python
    finally:
        if conn:
            try:
                conn.close()
            except Exception:  # ❌ 裸 except，静默忽略
                pass
```

**问题分析**:
- 连接关闭失败时静默忽略
- 可能掩盖连接池问题
- 不符合防御性编程原则

**影响评估**:
- 风险等级：🟡 MEDIUM
- 实际影响：轻微（连接池通常能恢复）
- 但静默失败不符合生产标准

**修复方案**:
```python
    finally:
        if conn:
            try:
                conn.close()
            except Exception as e:
                logger.warning(f"关闭数据库连接失败: {e}")  # ✅ 记录日志
```

### 问题 2: cursor.close() 位置不对

**位置**: `dispatch_center/schedule_routes.py:230`

**当前代码**:
```python
        cursor.close()

    except Exception as e:
        logger.error(f"从主系统数据库获取订单信息失败: {e}")
    finally:
        if conn:
            # ...
```

**问题分析**:
- `cursor.close()` 在 try 块内
- 如果 `cursor.execute()` 抛异常，`cursor.close()` 不会执行
- 虽然 MySQL 连接关闭时会一起关闭 cursor，但不符合规范

**影响评估**:
- 风险等级：🟢 LOW
- 实际影响：轻微（连接关闭时会关闭 cursor）
- 但不符合最佳实践

**修复方案**:
```python
    except Exception as e:
        logger.error(f"从主系统数据库获取订单信息失败: {e}")
    finally:
        if conn:
            try:
                cursor.close()
                conn.close()
            except Exception as close_err:
                logger.warning(f"关闭数据库连接失败: {close_err}")
```
