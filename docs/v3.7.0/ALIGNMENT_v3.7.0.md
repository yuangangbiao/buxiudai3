## 4. 疑问澄清（关键决策点）

### Q1: DLQ 表结构是什么？

**答**: 通过查看 `_reconcile.py` 和现有 `dlq` 表 schema，DLQ retry 需要：
- 失败消息存表（id, payload, retry_count, next_retry_at, error, created_at）
- 调度进程启动时启动 worker
- 指数退避重试
- 成功则删除，失败则增加 retry_count

### Q2: 是否要拆分 _core.py？

**答**: **不拆**。9635 行完整拆分风险过大，可能引发大量回归。本次仅：
- 抽取 1-2 个高频函数到新文件（试点）
- 评估拆分成本/收益

### Q3: L1 冒烟测试跑通条件？

**答**: L1 测试是**离线测试**（不依赖真实服务），使用 mock + 单元测试风格，5 分钟内跑完。
