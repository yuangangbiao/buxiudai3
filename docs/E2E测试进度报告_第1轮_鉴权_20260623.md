### 影响范围

- **生产数据安全**: 任何未登录用户能写入报工/质检/考勤数据

- **监控失效**: 任何未登录用户能重置监控
- **历史 P0-3 报工超额的根因之一**: 5008 扫码报工无鉴权

### 根因分析

- 5008 蓝图 (`mobile_api_ai/api/*.py`) 大部分路由没有 `@require_token` / `@require_auth` 装饰器
- 5001 桌面 Web 已修（46/46 路由），但 **5008 mobile 完全没修**
- 5003 dispatch 之前小钰把 `is_admin` 加进去了，但 5008 没做
