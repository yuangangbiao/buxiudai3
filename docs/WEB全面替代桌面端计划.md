# Web端全面替代桌面端 — 迁移计划 v1.0

> 日期：2026-06-22
> 规模：XL（跨系统，9个新页面+打印+增强）

## 一、目标

用 Web 端（desktop_web）**完全替代**桌面端（desktop），用户无需安装任何软件，浏览器即可完成所有跟单操作。

## 三、功能清单与验收标准

### 核心流程（P0，必做）

| # | 功能 | 验收标准 |
|---|------|---------|
| F1 | 发货管理 | 给定一个已完成的订单 → 可创建发货单 → 填写物流信息 → 状态变为"已发货" |
| F2 | 工序追踪 | 给定一个工单 → 可查看所有工序状态 → 可查看甘特图 → 可统计工时 |
| F3 | 订单查询 | 给定关键词组合 → 只返回匹配的订单 → 可导出Excel |
| F4 | 数据导入 | 上传Excel文件 → 系统自动解析 → 预览并确认导入 → 订单入库 |
| F5 | 数据导出 | 点击"导出" → 生成Excel文件 → 包含所有字段 → 自动下载 |

### 管理功能（P1，重要）

| # | 功能 | 验收标准 |
|---|------|---------|
| F6 | 操作员CRUD | 可新增操作员 → 编辑信息 → 删除（软删除） → 列表实时更新 |
| F7 | 操作员微信同步 | 点击同步 → 从企业微信拉取最新成员 → 自动匹配现有操作员 |
| F8 | 订单删除 | 勾选订单 → 点击删除 → 确认提示 → 订单进入归档状态 |
| F9 | 订单详情弹窗 | 点击订单 → 弹窗显示完整信息 → 含工序/物料/质检Tab |
| F10 | 订单模板管理 | 可保存当前参数为模板 → 可命名 → 可从模板加载参数 |
| F11 | BOM清单 | 可查看BOM列表 → 可新增BOM → 可计算材料需求 |
| F12 | 订单打印 | 点击打印按钮 → 生成PDF → 浏览器打印/保存 |
| F13 | 工续卡打印 | 点击工续卡 → 生成PDF → 显示工序+负责人+时间 |
| F14 | 发货单打印 | 点击发货单 → 生成PDF → 含物流信息+明细 |

### 辅助功能（P2，完整）

| # | 功能 | 验收标准 |
|---|------|---------|

| `/api/statistics/finished` | GET | 成品统计 |

### 4.3 技术选型

| 需求 | 选型 | 理由 |
|------|------|------|
| PDF生成 | jsPDF + html2canvas | 纯前端，无需后端 |
| Excel导入导出 | SheetJS (xlsx.js) | 浏览器端解析，速度快 |
| 甘特图 | SVG手绘 | 轻量，不引入大库 |
| 日期选择 | 原生 `<input type="date">` | 兼容性好 |
| 图表 | ECharts | 仪表板已有经验 |

## 七、风险识别

| # | 风险 | 影响 | 缓解方案 |
|---|------|------|---------|
| R1 | Excel导入数据量大导致浏览器卡死 | 高 | 前端分片处理，后端流式解析 |
| R2 | PDF打印格式不兼容老浏览器 | 中 | 使用 html2canvas 兼容方案 |
| R3 | 企业微信同步 API 变更 | 中 | 预留 mock 接口，支持手动导入 |
| R4 | 多用户同时操作同一订单 | 中 | 乐观锁 + 冲突提示 |
| R5 | 移动端小屏幕布局问题 | 低 | 响应式，优先核心功能 |
| R6 | 数据库 DDL 操作需停机 | 高 | 使用在线 DDL 工具，避免锁表 |
| R7 | 工序甘特图性能 | 低 | 仅渲染可视区域数据 |

## 八、决策日志

| # | 决策点 | 候选方案 | 最终选择 | 选择理由 | 否决方案 | 否决理由 |
|---|--------|---------|---------|---------|---------|---------|
| D1 | PDF生成 | A) 后端wkhtmltopdf B) 前端jsPDF C) 后端PDFKit | B | 纯前端，无需服务依赖，跨平台 | A,C | 需安装二进制/增加后端依赖 |
| D2 | Excel解析 | A) SheetJS纯前端 B) 后端openpyxl | A | 浏览器端预览，无需上传文件 | B | 增加服务器负载，用户无法预览 |
| D3 | 甘特图 | A) ECharts B) SVG手绘 C) D3.js | B | 轻量，不需要引入大库，定制灵活 | A,C | 引入库太大，功能冗余 |
| D4 | 订单删除 | A) 物理删除 B) 软删除+归档 | B | 保留操作历史，符合审计要求 | A | 永久删除无法追溯 |
