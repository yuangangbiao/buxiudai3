# 任务接收系统使用说明

## 📋 系统概述

本系统实现了基于企业微信群的任务分配和接收机制：
- 扫描工单二维码获取任务
- 企业微信消息通知和确认
- 任务状态追踪
- 超时提醒

---

## 🚀 快速开始

### 1. 启动服务

```bash
# 启动容器中心API
cd mobile_api_ai
python container_center_api.py

# 启动企业微信机器人（新终端）
python wechat_work_bot_v2.py
```

### 2. 配置企业微信机器人

在 `mobile_api_ai/.env` 文件中配置：
```
WECHAT_WORK_BOT_URL=https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=你的key
```

## 📖 使用指南

### 企业微信群命令

| 命令 | 说明 | 示例 |
|------|------|------|
| `帮助` | 查看所有可用命令 | `帮助` |
| `扫码 工单号` | 扫码获取任务 | `扫码 WO202604001` |
| `确认` | 确认接收到的任务 | `确认` |
| `确认 任务ID` | 确认指定任务 | `确认 PKG123456` |
| `待确认` | 查看待确认任务 | `待确认` |
| `任务` / `我的任务` | 查看我的任务 | `任务` |
| `查单 订单号` | 查询订单状态 | `查单 ORD202604001` |
| `报工 订单号 工序 数量` | 报工 | `报工 ORD202604001 编织 100` |

## 🔄 完整操作流程

### 流程1：扫码获取任务

```
1. 扫描工单二维码 (WO:WO202604001)
   ↓
2. 在企业微信群发送: "扫码 WO202604001"
   ↓
3. 收到任务分配通知
   ↓
4. 回复: "确认" 来确认接收
   ↓
5. 开始工作
   ↓
6. 完成后回复: "报工 ORD202604001 编织 100"
```

### 流程2：查看和确认任务

```
1. 发送: "待确认" 查看未确认任务
   ↓
2. 发送: "确认" 确认任务
   ↓
3. 发送: "任务" 查看任务列表
```

## 📡 API接口

### 1. 扫码获取任务

**接口**: `POST /api/scan/task`

**请求参数**:
```json
{
    "qr_data": "WO:WO202604001",
    "operator_id": "OP001"
}
```

**响应**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "order_info": {
            "order_no": "ORD202604001",
            "customer_name": "上海机械厂",
            "product_type": "不锈钢编织网",
            "quantity": 100,
            "unit": "米"
        },
        "current_process": {
            "process_name": "编织",
            "planned_qty": 100,
            "status": "待开始"
        },
        "voice_text": "张三你好，订单ORD202604001...",
        "next_steps": ["确认任务", "开始工作", "报工"]
    }
}
```

### 2. 分配任务到企业微信

**接口**: `POST /api/wechat/assign-task`

**请求参数**:
```json
{
    "order_no": "WO202604001",
    "operator_id": "OP001"
}
```

## 🔍 二维码格式

| 类型 | 格式 | 示例 |
|------|------|------|
| 工单 | `WO:工单号` | `WO:WO202604001` |
| 订单 | `ORD:订单号` | `ORD:ORD202604001` |
| 员工 | `WORKER:员工ID` | `WORKER:OP001` |

---

## 📊 任务状态

| 状态 | 说明 | 操作 |
|------|------|------|
| `pending` | 待分发 | - |
| `distributed` | 已分发，待确认 | 发送"确认" |
| `acknowledged` | 已确认 | 执行工作 |
| `completed` | 已完成 | 报工 |
| `expired` | 已过期 | - |

---

## ⏰ 超时提醒

- 任务分发后 **30分钟** 未确认 → 首次提醒
- 之后每 **15分钟** 提醒一次
- 最多提醒 **3次**
