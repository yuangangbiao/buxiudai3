# 主系统事件驱动集成方案

## 📋 概述

本方案将容器池功能正式集成到跟单系统主程序中，通过事件驱动架构实现松耦合：

```
┌─────────────────────────────────────────────────────────┐
│                   桌面端主系统                          │
│  ┌──────────────┐   ┌──────────────┐   ┌──────────────┐│
│  │  订单创建    │→  │ 工序创建     │→  │ 工序完成    ││
│  └──────────────┘   └──────────────┘   └──────────────┘│
│         ↓                   ↓                   ↓       │
│  ┌──────────────────────────────────────────────────┐   │
│  │              EventBus (事件总线)                  │   │
│  └──────────────────────────────────────────────────┘   │
│         ↓                   ↓                   ↓       │
│  ┌──────────────────────────────────────────────────┐   │
│  │         ContainerEventListener (事件监听器)       │   │
│  └──────────────────────────────────────────────────┘   │
│         ↓                                               │
│  ┌──────────────────────────────────────────────────┐   │
│  │      DesktopContainerIntegration (容器集成)        │   │
│  └──────────────────────────────────────────────────┘   │
└────────────────────────┬────────────────────────────────┘
                         ↓
                ┌──────────────┐
                │ 容器池数据库 │
                └──────────────┘
```

## 🚀 集成步骤

### Step 1: 在主程序中初始化监听器

修改 `main.py`，在初始化应用后添加：

```python
def main():
    try:
        # ... 现有代码 ...
        
        # 初始化应用
        initialize_app()
        
        # ====== 新增：初始化容器事件监听器 ======
        try:
            from container_event_listener import init_container_listener
            listener = init_container_listener()
            if listener.is_ready():
                print('[主程序] 容器事件监听器已启动')
            else:
                print('[主程序] 容器事件监听器不可用')
        except ImportError as e:
            print(f'[主程序] 容器监听器模块不可用: {e}')
        
        # 启动备份系统
        # ... 现有代码 ...
        
        # 启动主窗口
        # ... 现有代码 ...
        
    except Exception as e:
        # ... 现有代码 ...
```

### Step 2: 在工序视图中发布事件

修改 `views/process_view.py` 的 `_add_process` 方法：

```python
# 1. 在顶部添加导入
try:
    from container_event_listener import publish_process_created_event
    EVENT_PUBLISHER_AVAILABLE = True
except ImportError:
    EVENT_PUBLISHER_AVAILABLE = False
```

```python
# 2. 在添加工序成功后发布事件
    def _add_process(self):
        """添加工序"""
        if not self.current_order_id:
            alert('请先选择订单！')
            return
        
        # ... 现有表单创建和保存代码 ...
        
        # ====== 新增：发布工序创建事件 ======
        if EVENT_PUBLISHER_AVAILABLE and process_id:
            publish_process_created_event(
                order_id=self.current_order_id,
                production_id=self.current_prod_id,
                process_id=process_id,
                process_data=form_data
            )
        
        # 刷新列表
        self.load_processes()
        self._show_order_info()
```

## 📊 完整流程示例

### 1️⃣ 用户创建订单

```
1. 用户在订单视图创建订单
2. OrderDAO.create() 保存到数据库
3. 发布 ORDER_CREATED 事件
4. EventBus 分发事件
   ↓
5. ContainerEventListener.on_order_created() (可选，不发布任务)
   (通常在工序创建时才发布)
```

### 2️⃣ 用户创建生产工单

```
1. 用户在生产视图创建工单
2. ProductionDAO.create() 保存并创建工序记录
3. 发布 production:created 事件
```

### 3️⃣ 用户添加工序（或使用默认工序）

```
1. 用户在工序视图添加工序
2. ProcessDAO.create() 保存
3. 发布 process:created 事件
   ↓
4. ContainerEventListener.on_process_created() 监听到
   ↓
5. 获取订单、生产工单、工序信息
   ↓
6. publish_report_task() 发布到容器池
   ↓
7. 如果是编织/定型，同时发布质检任务
   ↓
8. 任务存储在 wechat_container.db
```

### 4️⃣ 网页端扫码获取任务

```
1. 用户在网页端扫码工单
2. /api/scan/task 接口被调用
3. 从容器池 (wechat_container.db) 获取任务
4. 显示任务详情
5. 用户确认/报工
```

## 🔧 架构优势

| 特性 | 说明 |
|------|------|
| 松耦合 | 工序视图不需要知道容器池的存在 |
| 可扩展 | 可以添加更多监听器处理不同逻辑 |
| 易维护 | 事件处理逻辑集中管理 |
| 容错性 | 容器集成失败不影响主系统运行 |

## 📝 事件定义

### 现有事件

```python
from core.event_bus import Events

Events.ORDER_CREATED        # 订单创建
Events.ORDER_UPDATED        # 订单更新
Events.ORDER_STATUS_CHANGED # 订单状态变更
Events.PROCESS_COMPLETED    # 工序完成
```

### 新增事件

```python
# 在 views/process_view.py 中发布
'process:created'  # 工序创建

# 在 models/production.py 中发布（可选）
'production:created'  # 生产工单创建
```
