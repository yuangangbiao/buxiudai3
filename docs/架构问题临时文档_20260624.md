### 🔴 A1 循环导入（conftest.py ↔ health.py）

| 属性 | 值 |
|------|-----|
| **严重性** | 🔴 P0 致命 |
| **文件** | tests/conftest.py:23 + tests/core/health.py:9 |
| **类型** | 架构（模块依赖） |
| **修复成本** | 30 min |
| **影响** | pytest 启动即崩，所有测试无法运行 |

**问题描述**:
- conftest.py 第 23 行 `from tests.core.health import health_checker`
- health.py 第 9 行 `from tests.conftest import SERVICES`
- pytest 启动时 conftest 触发 health.py 导入，health.py 反向导入 conftest，**conftest 还未完成 → AttributeError**

**修复方案**:
1. 创建 `tests/core/_config.py` 集中存放 SERVICES/DB_CONFIG
2. conftest.py 和 health.py 都从 _config.py 导入
3. 消除循环依赖

**修复后状态**: ⏳ 待修复

### 🟠 A5 跨 worker 数据竞争

| 属性 | 值 |
|------|-----|
| **严重性** | 🟠 P1 严重 |
| **文件** | tests/conftest.py:155-164 + 218-239 |
| **类型** | 架构（并发隔离） |
| **修复成本** | 1h |
| **影响** | pytest -n 2 时两个 worker 抢着 cleanup → flaky 失败 |

**问题**:
- `setup_test_environment` 是 `scope='session'` + `autouse=True`
- `pytest -n 2` 时 2 个 worker 同时调 cleanup
- 互相 DELETE 对方数据
- 收尾时 2 个 worker 抢着清

**修复方案**:
1. 改用 `scope='worker'`（pytest-xdist 提供）
2. 加 worker 互斥锁
3. 改 cleanup 逻辑为软删除 + 标记

**修复后状态**: ⏳ 待修复

### 🟠 A10 测试间无 worker 隔离

| 属性 | 值 |
|------|-----|
| **严重性** | 🟠 P1 严重 |
| **文件** | tests/conftest.py 全局 |
| **类型** | 架构（隔离机制） |
| **修复成本** | 2h |
| **影响** | 多 worker 并发会：互相 DELETE 对方订单、端口冲突、Browser 共享导致状态污染 |

**问题**:
- 所有 fixture 是 `scope='function'` 或 `scope='session'`
- 无 `scope='worker'` (pytest-xdist) 配置
- 无端口隔离
- 无数据库 schema 隔离

**修复方案**:
1. fixture 改用 `scope='worker'`
2. 端口分配器避免冲突
3. 数据库用 worker_id 前缀隔离
4. Browser 上下文加 worker_id 标记

**修复后状态**: ⏳ 待修复
