## 二、核心模块

### 2.1 容器池 (TaskPool)

```
container/
├── __init__.py
├── task_pool.py      # 任务容器池
└── dispatcher.py     # 任务分配器
```

**任务状态流转：**

```
PENDING → ASSIGNED → IN_PROGRESS → COMPLETED
  ↓          ↓            ↓
CANCELLED   (取消)      FAILED (失败)
```

**任务类型：**

| 类型 | 说明 | 示例 |
|------|------|------|
| report | 报工任务 | 编织100米 |
| quality | 质检任务 | 终检 |
| material | 物料需求 | 领料50kg |
| approval | 审批任务 | 经理审批 |

---

### 2.2 分配器 (Dispatcher)

**核心职责：**

1. **分配任务** - 将容器池中的任务分配给指定员工
2. **接收结果** - 接收移动端执行结果
3. **回写处理** - 根据任务类型调用不同的处理逻辑

**分配流程：**

```
1. 员工登录
       ↓
2. 调用 /api/tasks/dispatch
       ↓
3. 分配器查询容器池
       ↓
4. 符合条件的任务状态改为 ASSIGNED
       ↓
5. 返回任务列表给移动端
```

---

### 2.3 发布器 (TaskPublisher)

**桌面端使用：**

```python
from container import TaskPublisher, task_pool

publisher = TaskPublisher(task_pool)

# 发布报工任务
publisher.publish_report_task(
    order_no='ORD202604001',
    process_name='编织',
    record_id=103,
    operator_id='OP001',
    planned_qty=100,
    priority='high'
)

# 发布质检任务
publisher.publish_quality_task(
    order_no='ORD202604001',
    order_id=1,
    inspector_id='OP004',
    inspection_type='终检'
)
```

```

┌─────────────────────────────────────────────────────────────────┐

│                                                                  │
│ 12. 接收回写指令                                                 │
│ 13. 执行数据库更新                                               │

│ 14. 更新订单状态                                                 │

## 四、API接口

### 认证相关

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/auth/login` | POST | 员工登录，获取Token |
| `/api/auth/verify` | GET | 验证Token有效性 |

### 任务相关

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/tasks` | GET | 获取我的任务列表 |
| `/api/tasks/dispatch` | POST | 批量获取并锁定任务 |
| `/api/tasks/<id>` | GET | 获取任务详情 |
| `/api/tasks/<id>/start` | POST | 开始执行任务 |
| `/api/tasks/<id>/complete` | POST | 提交任务结果 |

### AI相关

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/ai/speech-to-report` | POST | 语音转报工 |
| `/api/ai/chat` | POST | AI智能对话 |

### 管理相关

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/pool/status` | GET | 获取容器池状态（仅管理员） |
| `/publish-tasks` | POST | 发布示例任务（测试用） |

## 五、安全特性

### 数据库保护

| 层级 | 保护措施 |
|------|----------|
| 网络层 | 数据库端口(3306)完全不暴露 |
| 应用层 | 移动端只通过API与容器池交互 |
| 认证层 | JWT Token验证，每次请求校验 |
| 权限层 | 任务只能被分配给的员工执行 |

### 任务隔离

- 每个任务有唯一ID和版本号
- 任务只能被分配一次
- 已完成任务不能重复提交
- 结果回写需要验证任务状态

## 八、与库存管理系统对接

```

    ↓
1. 库存系统检测到物料低于阈值
2. 生成物料需求任务到容器池
