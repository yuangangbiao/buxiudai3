### 2.1 24 个测试文件 BOM 头错误 → pytest 完全无法解析

**症状**: `invalid non-printable character U+FEFF (<unknown>, line 1)`  
**影响**: pytest 静默跳过这些测试，整个 `_complete` 系列实际是**死测试**！


**根因**: 文件开头有 UTF-8 BOM (`\ufeff`)，Python parser 不识别。  
**修复**: 1 行命令即可：

```powershell
# 批量去除 BOM
Get-ChildItem -Path tests -Recurse -Include *.py | ForEach-Object {
    $bytes = [System.IO.File]::ReadAllBytes($_.FullName)
    if ($bytes.Length -ge 3 -and $bytes[0] -eq 0xEF -and $bytes[1] -eq 0xBB -and $bytes[2] -eq 0xBF) {
        [System.IO.File]::WriteAllBytes($_.FullName, $bytes[3..($bytes.Length-1)])
        Write-Host "Fixed: $($_.FullName)"
    }
}
```

**修复后验证**: `pytest tests/unit/models/test_order_complete.py --collect-only` 应能列出用例

## 三、🟠 P1 高优先级漏洞

### 3.1 pytest 配置文件互相打架

**冲突**:
- `pytest.ini`: `testpaths = tests/unit`，包含 unit/integration
- `pyproject.toml`: `testpaths = ["tests"]` + `--ignore=tests/e2e --ignore=tests/integration`

**修复**: 保留一份，建议**只留 `pyproject.toml`**（更现代，覆盖率配置完整）：

```toml
[tool.pytest.ini_options]
minversion = "7.0"
testpaths = ["tests"]
python_files = ["test_*.py"]
pythonpath = ["."]
addopts = ["-v", "--tb=short", "--strict-markers",
           "--ignore=tests/e2e",
           "--ignore=tests/integration",
           "--cov=core", "--cov=models", "--cov=services", "--cov=utils",
           "--cov-report=term-missing",
           "--cov-report=html:coverage_html",
           "--cov-report=xml:coverage.xml",
           "--cov-fail-under=70"]  # 提到 70
```

然后删除 `pytest.ini`。

### 3.2 433 个"other"可疑无断言测试（⚠️ 经悲观审计修正：原报 219，实际 433）

### 3.3 281 组跨文件同名测试（⚠️ 修正：pytest 不冲突，仅命名风格问题）

> ⚠️ **悲观审计修正**：pytest 收集 3511 个测试时，同名函数按文件隔离，不冲突。`-k` 关键字同时命中同名函数是 pytest 设计允许的行为。281 组重名是**代码风格问题**，严重度降为 🟡 中。

| 其他典型重名 (`test_init`, `test_normal`, `test_empty` 等) | 各 10~100+ |

**风险**: pytest 收集不冲突（同函数名按文件隔离）；`-k` 关键字筛选会同时命中同名函数是设计允许行为，严重度降为代码风格问题。
