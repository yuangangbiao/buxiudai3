## 关键发现

1. **数据源**: 订单管理页面读的是 MySQL `orders` 表（不是 SQLite `system.db`），K22 之前的脏数据诊断只看了 system.db，**漏掉了 MySQL**。
2. **MySQL 密码在 .env 文件**（`MYSQL_PASSWORD=88888888`），被 gitignore 不在 Glob 默认扫描范围。
3. **测试订单特征**: 6-14 和 6-15 两天灌入 230 条，客户名 80% 是"张三"或空，规格参数全空，数量 1/10/100 高度规则化，状态全是"待确认" — 明显脚本批量灌入。
4. **GO-AUTO-001 是真实业务订单**: 虽然创建时间 6-15（与测试订单同天），但有完整字段（电话 13800138000、304不锈钢、单价 50、编织网带）— **不能按时间一刀切**，按 order_no 前缀 + 字段完整性综合判断更安全。
