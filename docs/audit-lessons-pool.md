### 2026-06-15 SQL JOIN 优化:列名歧义导致 1052 错误

| 字段 | 内容 |
|------|------|
| **日期** | 2026-06-15 |
| **审计对象** | `mysql_storage.py get_process_records()` 性能优化 |
| **检查项#** | 新增检查项(JION 优化规则) |
| **问题** | 优化 `ROW_NUMBER() OVER (PARTITION BY order_no)` 查询时,使用 `INNER JOIN` 重构 SQL,但 `where_clause` 被子查询和外层查询**共用**,导致 `Column 'order_no' in where clause is ambiguous` 错误(MySQL 不知道指的是子查询还是外层的 `order_no`) |
| **根因** | ① 变量复用时未考虑上下文差异 ② JOIN 操作时未明确列的表别名 ③ `where_clause` 在子查询和外层查询中含义不同但参数相同 |
| **建议** | ① **JOIN 操作中,所有列引用必须明确表别名**(`p.order_no`, `t.order_no` 等) ② 共用变量时检查每个使用位置的上下文 ③ 复杂 SQL 写完后用 `EXPLAIN` 验证 ④ 测试时覆盖所有分支(`if status and not search` 等) |
| **级别** | 🔴 规则级(SQL 优化必检) |
| **状态** | **新增** — 下次 SQL 修改时必检 |
