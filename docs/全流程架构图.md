## 二、主业务流程（含质检生产环节）

```mermaid
flowchart TD
    START([🎯 起点：自动跟单主软件建立订单])

    %% === 阶段一：订单 ===
    subgraph STAGE1["📋 阶段一：订单创建"]
        direction LR
        O1[📝 桌面端建立订单<br/>order_view.py<br/>→ MySQL orders 表]
        O2{订单已确认?}
        O1 --> O2
        O2 -->|否| O3[等待]
        O2 -->|是| O4[✅ 订单已确认<br/>Event: order:confirmed]
    end

    STAGE1 --> STAGE2

    %% === 阶段二：工单发布 ===
    subgraph STAGE2["📋 阶段二：工单发布"]
        direction LR
        W1[🖥️ 生产视图<br/>点击"发布任务"]
        W2[📡 POST /processes<br/>dispatch_center.py]

        W4[🔴 匹配流程模板<br/>不锈钢网带→production<br/>不锈钢丝→production]
        W5[📋 工序任务入容器池<br/>ContainerCenter<br/>dispatch_document]
        W6[🟡 tmpl_task_assigned<br/>📢 新任务通知→微信群]
        W1 --> W2 --> W3 --> W4 --> W5 --> W6
    end

    STAGE2 --> STAGE3

    %% === 阶段三：排产确认 ===

        S3[🟠 更新 process_records<br/>current_step → confirmed<br/>status → scheduled]
        S4[🟡 tmpl_schedule_complete<br/>✅ 排产完成确认→微信群]

        S1 --> S2A
        S1 --> S2B
        S2A --> S3
        S2B --> S3
        S3 --> S4
        S3 --> S5
    end

    STAGE3 --> STAGE4

    %% === 阶段四：物料管理 ===

        M3[🟠 用料需求入容器池<br/>dispatch_document type=material]
        M4[🟡 tmpl_material_shortage<br/>⚠️ 物料短缺→微信群]

        P3[🖥️ 桌面端发布工序<br/>点击"发送"]
        P4[📡 POST /process-tasks/send]
        P5[🟠 工序任务入容器池]
        P6[🟡 tmpl_task_assigned<br/>📢 工序任务通知→微信群]

        P5 --> P6
        P5 --> P7

        direction LR
        R0[🟠 开始生产<br/>status → in_production]
        R0 --> R1

        subgraph REPORT_LOOP["报工循环（可多次报工）"]
            direction LR
            R1[📱 晨圣报工小程序<br/>📷 扫码选择工序]

            R3[🟠 创建 process_sub_steps<br/>step_name / batch_no / quantity]
            R4[🟠 current_step +1]
            R5[🟡 tmpl_sub_step_created<br/>📋 报工通知<br/>→ 微信应用消息<br/>含：工序/数量/报工人]
            R1 --> R2 --> R3 --> R4 --> R5
        end

        R5 --> QC_GATE

        subgraph QC_STEP["质检环节（生产过程中嵌入）"]

            QC_GATE{需要质检?}
            QC_GATE -->|是| Q1[📱 晨圣报工小程序<br/>📷 扫码质检]
            Q1 --> Q2[📡 POST /api/quality]
            Q3{质检结果<br/>passed?}
            Q4[🟡 tmpl_alert_quality<br/>🔍 质量异常告警→微信群]
            Q5[🟠 记录质检结果<br/>quality_records 表]
            Q3 -->|不通过| Q4 --> Q5
            Q3 -->|通过| Q5
        end

        QC_GATE -->|否| SKIP_QC[跳过质检]

        REPORT_LOOP --> QC_GATE
        QC_STEP --> MORE_REPORT

        MORE_REPORT{还有未完成的工序?}
        MORE_REPORT -->|是| R1
        MORE_REPORT -->|否| QC_CHECK
    end

    STAGE6 --> STAGE7

    %% === 阶段七：报工完成 ===
    subgraph STAGE7["📋 阶段七：报工完成"]
        direction LR
        C1{completed_qty<br/>≥ order_qty?}
        C2[🟡 Event: process:completed<br/>报工完成]
        C3[🟡 tmpl_process_advance<br/>📍 工序全部完成→微信群]

        C1 -->|是| C2 --> C3
        C4 -.-> R1
    end

    STAGE7 --> STAGE8

    %% === 阶段八：成品入库 ===
    subgraph STAGE8["📋 阶段八：成品入口/入库"]
        direction LR
        WH1{自动入库<br/>还是手动?}
        WH2[🤖 自动判断<br/>触发自动入库]
        WH3[🖥️ 手动点击<br/>"确认入库"]
        WH4[🟠 current_step → warehousing<br/>成品入库]
        WH5[🟡 tmpl_warehousing_notify<br/>📥 入库通知→微信群]
        WH1 -->|自动| WH2
        WH1 -->|手动| WH3
        WH2 --> WH4
        WH3 --> WH4
        WH4 --> WH5
    end

    STAGE8 --> STAGE9

    %% === 阶段九：发货 ===

        direction LR
        SH1[🖥️ 发货视图<br/>shipment_view.py<br/>点击"发货"]
        SH2[📡 确认发货<br/>orders 表<br/>status → shipped]
        SH3[🟡 tmpl_process_complete<br/>🎉 流程已完成→微信群]
        SH4[🟠 同步到云端<br/>ContainerCenter → CC_CLOUD]
        SH1 --> SH2 --> SH3
        SH2 --> SH4
    end

    STAGE9 --> END([✅ 订单完结])

    %% 样式
    classDef stage fill:#e3f2fd,stroke:#1565c0,stroke-width:2px
```

## 六、微信消息发送链路

```mermaid
sequenceDiagram
    autonumber
    participant USER as 👤 操作员
    participant WW as 📱 企业微信
    participant CC6 as ☁️ 云端5006
    participant CP as 📡 CloudPoller<br/>(本地5003)
    participant DC as 🔴 调度中心<br/>(本地5003)
    participant CC as 🟠 容器中心<br/>(本地5002)

    rect rgb(230, 245, 255)
        Note over USER,WW: 【接收微信消息】用户 → 系统
        USER ->> WW: 发送消息（报工/质检/确认等）
        WW ->> CC6: 消息推送
        CC6 ->> CC6: 存储到 cloud_message_store
    end

    rect rgb(255, 240, 230)
        Note over CP,CC6: 【轮询拉取】CloudPoller 主动拉取
        loop 每5秒轮询
            CP ->> CC6: GET /api/poll?since_id={cursor}
            CC6 -->> CP: 新消息列表
            CP ->> CP: 更新游标 cloud_cursor.txt
            CP ->> CP: 消息去重检查
        end
    end

    rect rgb(255, 245, 230)
        Note over CP,DC: 【消息处理】CloudPoller 分发给调度中心
        CP ->> DC: POST /api/dispatch-center/task-notify
        Note right of DC: 根据消息类型路由处理
    end

    alt 工序报工消息
        DC ->> CC: POST /api/process_sub_step
        CC -->> DC: 创建 process_sub_steps
        DC ->> DC: send_to_wechat(tmpl_sub_step_created)
    end

    alt 质检消息
        DC ->> CC: POST /api/quality
        CC -->> DC: 记录质检结果
        alt 质检不通过
            DC ->> DC: send_to_wechat(tmpl_alert_quality)<br/>⚠️ 质量异常告警→微信群
        end
    end

    alt 排产确认消息
        DC ->> DC: check_and_trigger_auto_confirmation
        Note right of DC: 关键词匹配：确认/收到/好的
        DC ->> CC: POST /processes/confirm-by-reply
        CC -->> DC: 更新流程状态
        DC ->> DC: send_to_wechat(tmpl_schedule_complete)
    end

    rect rgb(240, 255, 240)
        Note over DC,WW: 【发送微信消息】系统 → 用户
        DC ->> DC: send_to_wechat(template, channels)
        alt 发送到微信群
            DC ->> CC6: POST /api/wechat/send (wechat_app_bot)
            CC6 ->> WW: 应用消息推送
        end
        alt 发送到微信应用
            DC ->> CC6: POST /api/send_app_message
            CC6 ->> WW: 应用消息推送
        end
        WW -->> USER: 收到通知
    end
```
