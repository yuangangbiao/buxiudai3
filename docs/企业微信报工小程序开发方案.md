# 企业微信报工小程序 - 开发方案与成果

## 一、项目概述

### 1.1 项目背景
为不锈钢网带跟单系统3.0开发移动端报工功能，支持员工通过企业微信小程序扫描二维码进行考勤打卡、报工提交、质检记录和审批操作。

### 1.2 项目目标
- 实现移动端扫码登录，无需手动输入账号密码
- 支持员工扫码查看工单详情和工序进度
- 移动端提交报工记录，实时同步生产进度
- 移动端进行质检记录，确保质量可追溯
- 移动端审批流程，提高管理效率
- 提供模拟API服务器，支持前端独立开发调试

### 1.3 技术栈
| 类别 | 技术 |
|------|------|
| 后端框架 | Flask |
| 数据库 | MySQL (复用桌面系统数据库) |
| 前端 | HTML/CSS/JavaScript |
| 移动端 | 企业微信小程序 |
| API风格 | RESTful |
| 认证方式 | JWT Token |

## 三、功能模块

### 3.1 认证模块 (Auth)
| 功能 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 员工扫码登录 | /api/auth/login | POST | 扫描员工二维码登录 |
| Token验证 | /api/auth/verify | GET | 验证Token有效性 |

**登录流程：**
1. 员工使用企业微信扫描二维码
2. 小程序获取微信授权码
3. 调用登录接口，传入授权码或员工ID
4. 服务器返回JWT Token
5. 小程序本地存储Token，后续请求携带

### 3.2 扫码模块 (Scan)
| 功能 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 扫码工单 | /api/scan/workorder/{order_no} | GET | 获取工单详情和工序进度 |
| 扫码员工 | /api/scan/worker/{operator_id} | GET | 获取员工信息和待办任务 |

**工单扫码返回数据：**
- 订单基本信息（客户、产品、数量、交期）
- 工单状态和优先级
- 工序列表及完成进度
- 当前进行中的工序详情
- 整体完成百分比

### 3.3 报工模块 (Process)
| 功能 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 我的任务 | /api/process/my-tasks | GET | 获取员工待报工任务列表 |
| 提交报工 | /api/process/{record_id}/report | POST | 提交报工记录 |
| 报工历史 | /api/process/history | GET | 查询报工历史记录 |

**报工状态：**
- `待开始`：工序未开始
- `进行中`：工序进行中，可更新完成数量
- `已完成`：工序完成，进入下一工序

### 3.4 质检模块 (Quality)
| 功能 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 质检列表 | /api/quality/list | GET | 获取质检记录列表 |
| 提交质检 | /api/quality/{order_id}/create | POST | 提交质检记录 |

**质检结果：**
- `合格`：产品通过质检
- `不合格`：产品未通过，需返工或报废
- `待复检`：需要再次检验确认

### 3.5 消息模块 (Message)
| 功能 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 消息列表 | /api/message/list | GET | 获取消息列表 |
| 未读数量 | /api/message/unread-count | GET | 获取未读消息数量 |

**消息类型：**
- `通知`：系统通知、工单完成提醒等
- `告警`：质量异常、延期风险等
- `审批`：审批结果通知

### 3.6 审批模块 (Approval)
| 功能 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 待我审批 | /api/approval/pending | GET | 获取待审批列表 |
| 审批通过 | /api/approval/{id}/approve | POST | 审批通过 |
| 审批拒绝 | /api/approval/{id}/reject | POST | 审批拒绝 |

**审批类型：**
- 生产计划调整申请
- 物料延迟申请
- 紧急订单审批
- 报废审批

### 3.7 统计模块 (Stats)
| 功能 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 看板数据 | /api/stats/dashboard | GET | 获取首页看板统计数据 |
| 产量趋势 | /api/stats/production-trend | GET | 获取产量趋势数据 |
| 工人排名 | /api/stats/worker-ranking | GET | 获取工人产量排名 |

## 五、数据库设计

### 5.1 移动端专用表

#### mobile_process_report (移动端报工记录)
```sql
CREATE TABLE mobile_process_report (
    id INT PRIMARY KEY AUTO_INCREMENT,
    process_record_id INT NOT NULL COMMENT '工序记录ID',
    worker_id VARCHAR(50) NOT NULL COMMENT '工人ID',
    completed_qty DECIMAL(10,2) DEFAULT 0 COMMENT '完成数量',
    status VARCHAR(20) DEFAULT '进行中' COMMENT '状态：进行中/已完成',
    report_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '报工时间',
    report_type VARCHAR(20) DEFAULT 'MOBILE' COMMENT '报工类型：MOBILE移动端/DESKTOP桌面端',
    remarks TEXT COMMENT '备注',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

#### mobile_quality_record (移动端质检记录)
```sql
CREATE TABLE mobile_quality_record (
    id INT PRIMARY KEY AUTO_INCREMENT,
    order_id INT NOT NULL COMMENT '订单ID',
    inspection_type VARCHAR(50) DEFAULT '巡检' COMMENT '质检类型：来料/首检/巡检/终检',
    result VARCHAR(20) NOT NULL COMMENT '结果：合格/不合格/待复检',
    inspector VARCHAR(50) NOT NULL COMMENT '质检员ID',
    inspection_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '质检时间',
    remarks TEXT COMMENT '备注',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### mobile_message (移动端消息)
```sql
CREATE TABLE mobile_message (
    id INT PRIMARY KEY AUTO_INCREMENT,
    receiver_id VARCHAR(50) NOT NULL COMMENT '接收人ID',
    title VARCHAR(200) NOT NULL COMMENT '消息标题',
    content TEXT COMMENT '消息内容',
    type VARCHAR(20) DEFAULT '通知' COMMENT '类型：通知/告警/审批',
    is_read TINYINT DEFAULT 0 COMMENT '是否已读：0未读/1已读',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### mobile_approval (移动端审批)
```sql
CREATE TABLE mobile_approval (
    id INT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(200) NOT NULL COMMENT '审批标题',
    content TEXT COMMENT '审批内容',
    applicant_id VARCHAR(50) NOT NULL COMMENT '申请人ID',
    approver_id VARCHAR(50) NOT NULL COMMENT '审批人ID',
    status VARCHAR(20) DEFAULT '待审批' COMMENT '状态：待审批/已通过/已拒绝',
    result_comment TEXT COMMENT '审批意见',
    approval_time DATETIME COMMENT '审批时间',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 九、注意事项

### 9.1 开发注意事项
1. 所有POST请求必须设置 `Content-Type: application/json`
2. Token在请求头中携带：`Authorization: Bearer <token>`
3. 模拟服务器仅用于开发测试，生产环境需连接真实数据库
4. 移动端需处理网络异常和超时情况

### 9.2 安全注意事项
1. 生产环境必须使用HTTPS
2. Token需设置过期时间
3. 敏感信息不要硬编码
4. 用户密码不得明文传输

### 9.3 性能注意事项
1. 扫码接口需快速响应，建议200ms内
2. 统计接口可做缓存
3. 列表接口需支持分页
4. 图片等资源需压缩优化
