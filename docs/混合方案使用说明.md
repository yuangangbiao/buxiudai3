# 生产任务助手 - 混合方案使用说明

## 📋 方案概述

采用 **企业微信群通知 + 网页操作** 的混合方案：

| 功能 | 实现方式 |
|------|---------|
| 任务通知 | 企业微信群机器人 |
| 扫码获取任务 | 网页界面 |
| 查看任务列表 | 网页界面 |
| 确认任务 | 网页界面 |
| 提交报工 | 网页界面 |

---

## 🚀 快速开始

### 1. 启动服务

```bash
cd mobile_api_ai
python wechat_work_bot_v2.py
```

### 2. 配置企业微信群机器人

在 `.env` 文件中配置：
```
WECHAT_WORK_BOT_URL=https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=你的密钥
```

### 3. 打开网页

在浏览器中访问：
```
http://localhost:5003
```

---

## 📖 功能使用

### 1️⃣ 扫码获取任务

1. 在网页中点击 **"扫码"** 标签
2. 输入工单信息（如：WO202604001）
3. 点击 **"获取任务"**
4. 查看任务详情，确认后点击 **"确认任务"**

### 2️⃣ 查看任务列表

1. 点击 **"任务"** 标签
2. 查看所有任务的状态
3. 对未确认的任务点击 **"确认任务"**

### 3️⃣ 提交报工

1. 点击 **"报工"** 标签
2. 填写以下信息：
   - 订单号
   - 工序
   - 完成数量
   - 操作员
3. 点击 **"提交报工"**

## 📊 任务状态说明

| 状态 | 说明 | 操作 |
|------|------|------|
| 待确认 | 任务已分配，等待操作员确认 | 点击"确认任务" |

| 已完成 | 任务已完成并报工 | - |

## 🔧 API 接口说明

### 获取任务
```
POST /api/scan/task
Content-Type: application/json

{
    "qr_data": "WO:WO202604001",
    "operator_id": "OP001"
}
```

### 确认任务
```
POST /api/tasks/{task_id}/acknowledge
Content-Type: application/json

{
    "operator_id": "OP001"
}
```

### 查询所有任务
```
GET /api/wechat/pool/tasks/all
```

### 提交报工
```
POST /api/wechat/pool/report
Content-Type: application/json

{
    "order_no": "ORD202604001",
    "process": "编织",
    "quantity": 100,
    "operator_id": "OP001"
}
```

## 🔍 故障排查

### 网页无法访问
- 检查服务是否启动：`python wechat_work_bot_v2.py`
- 确认端口 5003 未被占用

### 群通知不发送
- 检查 `.env` 文件中的 `WECHAT_WORK_BOT_URL`
- 确认企业微信群机器人已添加到群中

### 任务无法确认
- 检查 `wechat_container.db` 是否正常生成
- 查看服务日志中的错误信息
