## 二、架构方案

### 2.1 最终形态

```
浏览器访问 → http://localhost:5010

Flask 5002 (容器中心)
Flask 5003 (调度中心)          5010 统合库存 Web 服务
Flask 5008 (报工)         ──►  ├─ Jinja2 模板渲染
                               ├─ 所有库存 CRUD API
                               ├─ WebSocket (预警推送)
                               └─ 集成到统一导航
```

### 2.2 技术选型

| 层 | 选择 | 理由 |
|----|------|------|
| 后端框架 | Flask (已有) | 零迁移成本，与现有体系一致 |
| 模板引擎 | Jinja2 | Flask 原生，已有 dashboard_server.py 模板 |
| 前端样式 | Bootstrap 5 CDN | 快速出页面，无需构建工具 |
| 交互增强 | HTMX 2.0 CDN | 无 JS 框架，AJAX 通过 HTML 属性驱动 |
| 数据源 | `models/inventory.py` | 复用现有 CRUD |
| 端口 | 5010 (合并现有) | 不新增端口 |

### 2.3 文件结构

```
mobile_api_ai/
├── inventory_api_server.py          # 扩展现有，加页面路由
├── inventory_web/                   # 新建 Web 模块
```

## 四、关键设计决策

### 4.1 导航结构

```
┌──────────────────────────────────────────┐
│ 🏭 不锈钢网带跟单系统 v3.0         [⏰]  │
├──────────┬───────────────────────────────┤
│ 仪表盘   │                               │
│ 库存管理 │       主内容区                  │
│ 订单管理 │                               │
│ 物料清单 │                               │
│ 生产管理 │                               │
│ 质量管理 │                               │
│ 报表导出 │                               │
│ 系统设置 │                               │
└──────────┴───────────────────────────────┘
```

### 4.2 数据源统一

```
桌面端 → tkinter → models/inventory.py → inventory_db MySQL
                                         ↓
Web 版 → Flask → models/inventory.py → inventory_db MySQL  ← 完全复用
```

### 4.3 tkinter 对应到 Web 组件

| tkinter 组件 | Web 等价 |
|-------------|---------|
| `ttk.Treeview` | `<table>` + DataTables.js |
| `ttk.Entry` | `<input>` |
| `ttk.Combobox` | `<select>` |
| `ttk.Button` | `<button>` + HTMX hx-post |
| `messagebox` | Bootstrap toast/modal |
| `filedialog` | `<input type="file">` |
| `tk.Toplevel` | Bootstrap modal |

## 五、不迁移的模块

| 模块 | 原因 |

## 六、最终收益

| 指标 | 桌面版 | Web 版 |
|------|--------|--------|
| 部署方式 | 安装 .exe | 浏览器打开 |

| 维护成本 | 高（GUI 调试） | 低（HTML 调试） |
| 与其他服务集成 | ❌ | ✅ 统一导航 |
