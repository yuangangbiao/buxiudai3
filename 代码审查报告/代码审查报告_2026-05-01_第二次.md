## 三、SQL注入风险 (HIGH)

### 3.1 动态表名构造

| 文件 | 行号 | 风险代码 |
|------|------|----------|
| backup_system.py | 41 | `cursor.execute("SELECT * FROM %s", (table,))` |
| backup_system.py | 44 | `cursor.execute("SHOW COLUMNS FROM %s", (table,))` |
| inventory_backup.py | 306 | `cursor.execute(f"SELECT * FROM {table}")` |

**风险说明**: 使用 `%s` 占位符传递表名不可靠，应使用白名单验证。

**修复方案**:
```python
# ✅ 安全方式
ALLOWED_TABLES = {'orders', 'products', 'inventory', ...}
if table not in ALLOWED_TABLES:
    raise ValueError(f"不允许的表名: {table}")
cursor.execute(f"SELECT * FROM `{table}`")
```

---

### 3.2 动态SQL构造

| 文件 | 行号 | 风险代码 |
|------|------|----------|
| models/operator.py | 172 | `cursor.execute(f"""...{field}...{value}...""")` |
| views/material_prep_view.py | 185, 213 | `cursor.execute(f"""..."{table}"...""")` |

**风险说明**: 动态字段名和表名拼接可能导致SQL注入。

**修复方案**:
```python
# ✅ 使用参数化查询 + 白名单验证
VALID_FIELDS = {'field1', 'field2', ...}
if field not in VALID_FIELDS:
    raise ValueError("无效字段")
cursor.execute(f"UPDATE table SET {field} = %s WHERE id = %s", (value, record_id))
```

## 四、规范问题 (MEDIUM)

### 4.1 sys.path.insert 使用不规范 (60+文件)

**问题文件示例**:
- models/process.py
- models/production.py
- models/shipment.py
- mobile_api/models/database.py
- mobile_api/init_database.py
- views/main_window.py
- 部署包/服务端/inventory_server.py

**问题说明**: 每个模块中重复 `sys.path.insert(0, ...)` 不符合项目规范，且使用硬编码路径。

**修复方案**:
```python
# ❌ 错误 - 分散管理
sys.path.insert(0, os.path.dirname(os.path.dirname(os.path.abspath(__file__))))

# ✅ 正确 - 统一入口
# 只在 main.py 或入口文件中设置一次
import sys
sys.path.insert(0, r'd:\yuan\不锈钢网带跟单3.0')
from config import BASE_DIR
```

---

### 4.2 裸except异常处理

| 文件 | 行号 | 问题代码 |
|------|------|----------|
| models/database.py | 68-70 | `except: pass` |
| models/process_calc_rule.py | ~220 | `except: conn.rollback()` |

**问题说明**: 静默吞掉异常会导致错误难以排查，且无法记录日志。

**修复方案**:
```python
# ❌ 错误
except:
    pass

# ✅ 正确
except Exception as e:
    logger.exception("获取连接失败: %s", e)
    raise
```

---

### 4.3 print调试 (100+处)

**问题说明**: 生产代码中使用 `print()` 而非 `logger` 记录日志，不便于日志管理和排查。

**修复方案**:
```python
# ❌ 错误
print("[ERROR] Add product failed: {e}")

# ✅ 正确
logger.error("Add product failed: %s", e)
```
