# 错误处理上下文记录

## 一、常见错误类型

### 1. 模块导入错误 (ModuleNotFoundError)

**错误示例：**
```
No module named 'views.confirm'
```

**原因分析：**
- `views/__init__.py` 文件为空，没有导出子模块
- 其他模块尝试通过 `from views import confirm` 导入，但 `views` 包没有正确配置

**解决方案：**
在 `views/__init__.py` 中添加正确的模块导出：
```python
from .orders import show_order_confirm, OrderListView
from .dialogs import alert, confirm, popup_form
```

**预防措施：**
- 创建包时确保 `__init__.py` 文件包含必要的导出
- 使用相对导入时检查路径是否正确

---

### 2. 枚举属性错误 (AttributeError)

**错误示例：**
```
AttributeError: type object 'OrderStatus' has no attribute 'COMPLETED'
```

**原因分析：**
- 使用了不存在的枚举成员
- 枚举定义与代码中的引用不一致

**解决方案：**
检查枚举定义，使用正确的成员名称：
```python
# 错误
OrderStatus.COMPLETED.value

# 正确（根据实际枚举定义）
OrderStatus.FINISHED.value
```

**预防措施：**
- 使用枚举时先查看定义文件确认成员名称
- 在 IDE 中启用自动补全功能

---

### 3. DAO 方法不存在 (AttributeError)

**错误示例：**
```
AttributeError: type object 'ProductionDAO' has no attribute 'get_by_id'
```

**原因分析：**
- 调用了 DAO 类中未定义的方法
- 方法名称拼写错误

**解决方案：**
在对应的 DAO 类中添加缺失的方法：
```python
@staticmethod
def get_by_id(prod_id: int) -> dict:
    """根据ID获取记录"""
    conn = get_connection()
    try:
        cursor = conn.cursor()
        cursor.execute("SELECT * FROM table_name WHERE id=%s", (prod_id,))
        row = cursor.fetchone()
        cursor.close()
        return dict(row) if row else None
    finally:
        conn.close()
```

**预防措施：**
- 调用 DAO 方法前检查是否存在
- 保持 DAO 接口的一致性

### 4. 国际化格式化错误 (TypeError)

**错误示例：**
```
TypeError: not all arguments converted during string formatting
```

**原因分析：**
- 使用 `%` 格式化但国际化系统使用 `.format()`
- 参数名称不匹配

**解决方案：**
使用正确的格式化方式：
```python
# 错误
t('process.record_count') % "0"

# 正确
t('process.record_count', count="0")
```

**预防措施：**
- 统一使用国际化函数的参数传递方式
- 在 i18n 配置中使用 `{参数名}` 格式

---

### 5. 线程安全问题

**错误示例：**
```
RuntimeError: dictionary changed size during iteration
```

**原因分析：**
- 多线程环境下对全局变量进行读写操作
- 没有使用锁保护共享资源

**解决方案：**
使用线程安全的配置类：
```python
class ConfigManager:
    _lock = threading.Lock()
    
    @classmethod
    def get_config(cls, key):
        with cls._lock:
            return cls._config.get(key)
```

**预防措施：**
- 对全局变量的访问使用锁保护
- 使用线程安全的数据结构

---

### 6. 数据库连接池问题

**错误示例：**
```
TimeoutError: Connection pool exhausted
```

**原因分析：**
- 数据库连接没有正确关闭
- 连接池配置不合理

**解决方案：**
- 使用 `try-finally` 确保连接关闭
- 优化连接池大小配置

**预防措施：**
- 始终使用 `finally` 块关闭连接
- 定期检查连接使用情况

## 二、错误处理最佳实践

### 1. 错误日志记录

所有错误都应该记录详细的日志信息：

```python
try:
    # 业务逻辑
except Exception as e:
    log_error(f"操作失败: {str(e)}", exc_info=True)
    alert("操作失败，请查看日志")
```

### 2. 错误提示

向用户提供友好的错误提示，同时记录详细日志：

```python
try:
    result = some_operation()
except ValueError as e:
    log_error(f"数据验证失败: {str(e)}")
    alert(f"数据错误: {str(e)}")
except Exception as e:
    log_error(f"未知错误: {str(e)}", exc_info=True)
    alert("系统错误，请联系管理员")
```

### 3. 错误恢复

对于可恢复的错误，提供重试机制：

```python
max_retries = 3
for attempt in range(max_retries):
    try:
        result = database_operation()
        break
    except ConnectionError as e:
        if attempt < max_retries - 1:
            time.sleep(1)
            continue
        raise
```
