# API 数据源分析

**审计日期**: 2026-05-30  
**数据库**: `steel_belt` (主库) + `container_center` (容器库)，同一个 MySQL 实例

---

## 数据源总览

```
MySQL 127.0.0.1:3306
├── steel_belt (MYSQL_CFG)
│   ├── orders                — 工单
│   ├── process_records       — 工序记录
│   ├── process_sub_steps     — 子步骤
│   ├── production_orders     — 生产订单
│   ├── workers / attendance  — 工人/考勤
│   └── enterprise_structure  — 企业架构
│
└── container_center (CONTAINER_MYSQL_CFG)
    ├── data_packages         — 任务数据包
    ├── dispatch_commands     — 调度指令
    ├── process_records       — 工序镜像(容器端)
    ├── data_flow_logs        — 数据流日志
    ├── data_collection_records— 采集记录
    ├── sync_logs             — 同步日志
    ├── product_flow_map      — 流程映射
    ├── sub_step_audit_log    — 报工审计
    └── ... (12张容器专有表)
```

## :8008 同步桥 (`sync_bridge_server.py`)

| 端点 | 数据源 | 说明 |
|------|--------|------|
| `POST /api/sync/sub-step-report` | **读** `container_center` → **写** `steel_belt` | 报工数据双写 |
| `POST /api/sync/status-change` | 同上 | 状态变更通知 |
| `POST /api/sync/sync-process` | 同上 | 工序批量同步 |

## 数据源占比

| 数据源 | 端点数 | 占比 |
|--------|--------|------|
| `container_center` MySQL | ~120 | 48% |
| `steel_belt` MySQL | ~70 | 28% |
| 外部 (HTTP/微信/AI API) | ~15 | 6% |
| 本地 (JSON/SQLite/内存) | ~10 | 4% |
| 双库 (读 CC + 写 SB) | ~5 | 2% |
| 无数据 (页面渲染) | ~30 | 12% |
