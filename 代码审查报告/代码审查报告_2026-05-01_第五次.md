## 一、SQL注入防护 - ⚠️ MEDIUM

### 1.1 动态表名/库名构造

| 文件 | 行号 | 风险代码 |
|------|------|----------|
| inventory_db_complete.py | 118-119 | `cursor.execute(f"CREATE DATABASE IF NOT EXISTS `{db_name}`...")` |
| database_migration.py | 119, 135 | `cursor.execute(f"DESCRIBE `{table_name}`")` |
| data_migrator.py | 277-278 | `cursor.execute(f"CREATE TABLE `{backup_name}` LIKE `{table_name}`")` |
| backup_database.py | 33, 39 | `cursor.execute(f"SELECT * FROM `{table_name}`")` |
| schema_comparator.py | 95, 98 | `cursor.execute(f"DESCRIBE `{table_name}`")` |

**风险分析**:
- 使用反引号包裹标识符，可防止基本SQL注入
- 但表名/库名来自配置文件或用户输入时，仍存在风险
- `mysql_tool.py:569-571` 使用参数化查询 ✅

**建议修复**:
```python
# 添加白名单验证
ALLOWED_DATABASES = {'inventory_db', 'test_db'}
ALLOWED_TABLES = {'orders', 'products', 'inventory', 'operators', ...}

if db_name not in ALLOWED_DATABASES:
    raise ValueError(f"不允许的数据库名: {db_name}")
```

## 二、API参数处理 - ⚠️ MEDIUM

### 2.1 mobile_api 参数验证

| 文件 | 行号 | 参数 | 风险 |
|------|------|------|------|
| data_collector_api.py | 660-666 | data_type, status, order_no, limit | 缺乏类型验证 |
| container_api_server.py | 102 | types | 缺乏验证 |
| schedule_flow.py | 475 | status_filter | 缺乏验证 |

**当前代码**:
```python
data_type = request.args.get('data_type')
status = request.args.get('status')
order_no = request.args.get('order_no')
limit = int(request.args.get('limit', 100))
```

**建议修复**:
```python
# 参数白名单验证
ALLOWED_DATA_TYPES = {'report', 'quality', 'material', 'approval'}
ALLOWED_STATUSES = {'pending', 'processing', 'completed', 'failed'}

data_type = request.args.get('data_type')
if data_type and data_type not in ALLOWED_DATA_TYPES:
    return fail(code=400, message='无效的data_type')

limit = min(int(request.args.get('limit', 100)), 1000)  # 限制最大值
```
