## 二、安全问题

### 🔴 P0: 无认证机制

所有 33 个路由均可无登录访问。任何人知道 IP:5010 就能操作出入库、删除数据。

**修复建议**: 加 HTTP Basic Auth 或简单 token：
```python
# inventory_api_server.py
from functools import wraps
def require_auth(f):
    @wraps(f)
    def decorated(*args, **kwargs):
        token = request.headers.get('X-Auth-Token', request.args.get('token', ''))
        if token != os.getenv('INVENTORY_API_TOKEN', 'inv_2026'): 
            return jsonify({'code':403,'msg':'未授权'}), 403
        return f(*args, **kwargs)
    return decorated
```

### 🔴 P0: 无 CSRF 保护

8 个 POST/PATCH/DELETE 路由无 CSRF 校验，可被跨站攻击。

**修复建议**: Flask 内置 `app.secret_key` + 表单加 `{{ csrf_token() }}`

### 🟡 P1: shell 命令注入风险（备份模块）

`backup_create` 和 `backup_restore` 通过 `subprocess.run(shell=True)` 执行 `mysqldump`/`mysql`。虽然没有直接拼接用户输入到命令中，但 `shell=True` 不推荐。

**修复建议**: 改用 `shell=False` + 参数列表

### 🟡 P1: 4 处 f-string 拼 SQL

```python
# L100 - WHERE 子句动态拼接（安全，值走 %s 参数化）
count_sql = f"SELECT COUNT(*) ... WHERE {where}"

# L277, L404, L411 - 表名动态拼接（安全，表名来自白名单字典）
_execute(f"INSERT INTO {tables[entity]} ...")
```

4 处均为低风险（表名来自白名单或条件用参数化），但建议统一为 `%s` 占位符。

## 三、数据层

### ✅ 优点

- 数据库统一: `inventory_db`，不再有双库混乱
- DictCursor 统一使用
- 出入库事务完整 (commit/rollback)
- `GREATEST(0, current_qty - %s)` 防止负库存

### ⚠️ 问题

| 问题 | 详情 |
|------|------|
| 连接未池化 | 每次请求 `pymysql.connect()` 新建连接，无复用 |
| 无连接超时 | `connect_timeout=5` 硬编码 |
| 无重试机制 | 连接失败直接抛异常 |
| 备份用 mysqldump | 依赖系统 PATH 中有 `mysqldump` 命令 |
