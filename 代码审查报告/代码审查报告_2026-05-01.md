### 3.2 【严重Bug】重复关闭Cursor  🔴 CRITICAL

**文件**: models/order.py 第102-143行

**问题描述**: OrderDAO.update() 方法中多次调用 cursor.close()，第二次会抛出异常

**问题代码**:
```python
# 第102行
cursor = conn.cursor()  # 第一次创建 cursor
cursor.execute("SELECT status FROM orders WHERE id=%s", (order_id,))
old = cursor.fetchone()
cursor.close()  # 第一次关闭 ✓

# 第122行
cursor = conn.cursor()  # 第二次创建 cursor
cursor.execute("""UPDATE orders SET ...""", (...))
conn.commit()
cursor.close()  # 第二次关闭 ✓

# 第135-136行
cursor.execute("SELECT order_no FROM orders WHERE id=%s", (order_id,))
order_row = cursor.fetchone()
cursor.close()  # ❌ 重复关闭！cursor已被上方关闭
```

**影响分析**:
- 正常情况下会抛出 `InterfaceError: cursor is closed`
- 资源未正确释放
- 可能导致数据库连接异常

**修复方案**:
```python
# 使用 try-finally 确保资源正确释放
conn = get_connection()
try:
    cursor = conn.cursor()
    # ... 第一次查询 ...
    cursor.close()

    cursor = conn.cursor()  # 重新获取cursor
    # ... 更新操作 ...
    cursor.close()
finally:
    conn.close()
```

**修复优先级**: P0 - 立即修复

```python
except Exception as e:
    conn.rollback()  # 出错时回滚
    raise
finally:
    conn.close()
```

**修复优先级**: P0 - 立即修复

### 3.4 【高风险】connection pool资源泄露  🟠 HIGH

**文件**: models/database.py

**问题描述**: get_connection() 使用共享实例，异常时未正确归还连接到池

**问题代码**:
```python
class MySQLConnectionPool:
    _instance = None
    _lock = threading.Lock()

    def __new__(cls):
        if cls._instance is None:
            with cls._lock:
                if cls._instance is None:
                    cls._instance = super().__new__(cls)
                    cls._instance._connection_pool = []
                    cls._instance._max_connections = 5  # 硬编码
        return cls._instance
```

**风险**:
1. 连接池大小仅5，高并发时迅速耗尽
2. 连接泄漏后无法恢复
3. 无连接健康检查机制

**修复方案**:
```python
class MySQLConnectionPool:
    def get_connection(self):
        if not self._connection_pool:
            if len(self._connections_in_use) < self._max_connections:
                conn = self._create_connection()
            else:
                raise Exception("连接池已满，请等待")
        # ... 使用完成后归还连接 ...
```

**修复优先级**: P0 - 连接池配置必须可配置化

### 6.1 数据库连接模板重复  🟠 HIGH

**位置**: models/ 下所有DAO类

**重复代码模式**:
```python
# 每个DAO方法都包含此模板
conn = get_connection()
cursor = None
try:
    cursor = conn.cursor()
    # ... 操作 ...
finally:
    if cursor:
        cursor.close()
    conn.close()
```

**修复建议**:
```python
# ✅ 使用 context manager
from contextlib import contextmanager

@contextmanager
def get_db_cursor():
    conn = get_connection()
    cursor = None
    try:
        cursor = conn.cursor()
        yield cursor, conn
        conn.commit()
    except Exception as e:
        conn.rollback()
        raise
    finally:
        if cursor:
            cursor.close()
        conn.close()

# 使用方式
with get_db_cursor() as (cursor, conn):
    cursor.execute("SELECT * FROM orders")
```

**修复优先级**: P1 - 1周内修复
