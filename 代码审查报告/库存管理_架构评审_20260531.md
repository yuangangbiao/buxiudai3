# 库存管理系统 — 架构评审报告

**日期**: 2026-05-31  
**评审维度**: 分层架构 / 数据流 / 路由设计 / 模板系统 / 安全架构 / 部署结构

---

## 一、分层架构

```
┌─────────────────────────────────────────────┐
│               浏览器 (Bootstrap)              │  表示层
├─────────────────────────────────────────────┤
│  Jinja2 模板 (17个 HTML)                      │  模板层
├─────────────────────────────────────────────┤
│  Flask Blueprint (36路由)                     │  控制层
│  ├─ 页面路由 (14个 render_template)           │
│  ├─ API 路由 (16个 JSON)                      │
│  └─ CRUD 路由 (6个 PATCH/DELETE/POST)         │
├─────────────────────────────────────────────┤
│  连接池 PooledDB (10连接, 2-5缓存)             │  数据层
│  ├─ inventory_db (12表)                       │
│  └─ 备份: mysqldump → inventory_backups/     │
└─────────────────────────────────────────────┘
```

### 评分: ⭐⭐⭐⭐

**优点**: 清晰的三层分离，蓝图隔离业务逻辑  
**缺点**: `inventory_api_server.py` 仍保留独立的 `_get_conn()` (不用连接池)，与 Web 蓝图数据层不一致

## 二、路由设计

### 2.1 路由分类

| 类别 | 数量 | 前缀 | 示例 |
|------|:--:|------|------|
| 页面路由 | 13 | `/inventory/*` | `/inventory/stock` |
| 业务 API | 10 | `/inventory/*/do` | `/inventory/inbound/do` |
| 通用 CRUD | 6 | `/inventory/api/*` | `/inventory/api/product/1` |
| 管理后台 | 4 | `/inventory/backup/*` | `/inventory/backup/create` |
| 外部入口 | 3 | `/login`, `/`, `/api/health` | |

### 2.2 RESTful 评估

| 标准 | 现状 | 建议 |
|------|------|------|
| GET 读资源 | ✅ `/inventory/stock` | — |
| POST 创建 | ⚠️ `/inventory/inbound/do` | 改为 `/inventory/transactions` |
| PATCH 更新 | ✅ `/inventory/api/product/1` | — |
| DELETE 删除 | ✅ `/inventory/api/product/1` | — |
| 幂等性 | ⚠️ `stock/out` 非幂等 | 加 idempotency key |

### 路由命名: 🟡 混合风格

```
✅ RESTful:  /inventory/api/<entity>/<id>     (PATCH/DELETE)
⚠️ RPC风格:  /inventory/inbound/do             (POST)
⚠️ RPC风格:  /inventory/batch/do               (POST)
```

建议统一为: `POST /inventory/api/transactions/inbound` + `POST /inventory/api/transactions/batch`

## 三、数据流

```
用户操作 → form.js → fetch(/inventory/*/do) → routes.py → PooledDB → MySQL
                                                              ↓
                                                        inventory_db
                                                  ┌─── inventory (库存)
                                                  ├─── products (商品)
                                                  ├─── inventory_transactions (流水)
                                                  ├─── inventory_alerts (预警)
                                                  ├─── suppliers / categories / warehouses
                                                  └─── operation_logs (日志)
```

### 数据一致性

| 场景 | 机制 | 评估 |
|------|------|:--:|
| 入库 | UPDATE inventory + INSERT transaction + COMMIT | ✅ 事务 |
| 出库 | UPDATE (GREATEST防负) + INSERT + COMMIT | ✅ |
| 批量 | 循环内 CUD + 统一 COMMIT | ✅ |
| 备份 | mysqldump 子进程 | ⚠️ 无锁，可能不一致 |
| 恢复 | mysql < 子进程 | ⚠️ 覆盖式，无确认 |

## 四、模板系统

### 结构

```
base.html (布局+导航+认证状态)
├── dashboard.html   ← extends base
├── stock_list.html  ← extends base
├── inbound.html     ← extends base
├── ... (12个子模板)
└── print_preview.html ← extends base (有独立打印样式)
```

### 评分: ⭐⭐⭐⭐

**优点**: 单一继承链，CSS 集中管理  
**缺点**: 14 个模板各自内嵌 JS，公共函数 (editRow/delRow) 重复 3 次，建议抽到 `base.html` 底部

## 五、安全架构

```
┌──────────┐     ┌──────────────┐     ┌─────────────┐
│ 浏览器    │────→│ Flask Session│────→│ before_request│
│          │     │ (secret_key) │     │ 白名单:       │
│ 未登录    │     │              │     │ /login, /api/*│
│ → /login │     │              │     │ 拦截: 其他全部 │
└──────────┘     └──────────────┘     └─────────────┘
```

| 层 | 机制 | 状态 |
|----|------|:--:|
| 认证 | Session + 密码 | ✅ |
| 授权 | 白名单路由 | ✅ |
| CSRF | Session (隐式) | 🟡 无显式 token |
| SQL 注入 | %s 参数化 | ✅ |
| Shell 注入 | subprocess 参数化 | ✅ |
| XSS | Jinja2 自动转义 | 🟡 innerHTML 10 处 |

## 六、与其他服务的关系

```
库存管理 :5010 ─── 独立服务，使用 inventory_db
  │
  ├─ 无直接依赖其他服务
  ├─ 侧边栏链接 → :5008 报工 / :5003 调度 / :5002 容器
  └─ server_launcher.py 管理启停
```

**评估**: 松耦合 ✅。唯一交叉点: `server_launcher.py` 配置。

## 七、部署结构

```
server_launcher.py
  └─ 库存管理: inventory_api_server.py → :5010
       ├─ app = Flask(__name__)
       ├─ app.secret_key (session)
       ├─ before_request (认证)
       ├─ /login, /logout (登录)
       ├─ /api/health (健康检查)
       ├─ /api/* (原始 REST API)
       └─ register_blueprint(inventory_web)
            ├─ routes.py (36 条路由)
            ├─ PooledDB (连接池)
            └─ templates/ (17 个 HTML)
```

### 部署问题

| 问题 | 影响 | 建议 |
|------|------|------|
| `inventory_api_server.py` 仍用裸连接 | 旧 API 无池化 | 统一到连接池 |
| 备份路径硬编码 `inventory_backups/` | 不可配置 | 加环境变量 |
| 无进程守护 | 崩溃不重启 | server_launcher 已管理 |
| 无 HTTPS | 密码明文传输 | 仅内网，可接受 |

## 九、建议

| 优先级 | 建议 | 收益 |
|:--:|------|------|
| 🟡 | 合并 `inventory_api_server.py` 的连接层到连接池 | 统一数据层 |
