## 错误详细记录

### 🔴 ERR-SYS-001 - Non-UTF-8编码错误

**错误信息**:
```
SyntaxError: Non-UTF-8 code starting with '\xe5' on line 3, but no encoding declared
```

**发生时间**:
- 2026-04-29 17:13:18 (首次)
- 2026-04-29 17:15:47
- 2026-04-29 17:18:21
- 2026-04-29 17:23:06
- 共计: 8次

**根本原因**:
- 文件保存时编码非UTF-8
- 缺少 `# -*- coding: utf-8 -*-` 声明

**影响范围**:
- database.py
- 其他模块文件

**解决方案**:
1. 在所有.py文件首行添加: `# -*- coding: utf-8 -*-`
2. 使用VSCode/IDE将文件另存为UTF-8编码
3. 统一IDE的默认编码设置为UTF-8

**整改状态**: 待整改 (关联P2-13)

### 🔴 ERR-SYS-003 - 未终止字符串

**错误信息**:
```
SyntaxError: unterminated string literal (detected at line 355)
status VARCHAR(20) DEFAULT '待开?,
                               ^
```

**发生时间**: 2026-04-29 17:16:18, 17:17:04

**根本原因**:
- 中文引号 `'` 被误用作英文引号
- 字符串跨越多行未正确闭合

**影响文件**: database.py 第355行, 第374行

**解决方案**:
```python
# 错误写法
status VARCHAR(20) DEFAULT '待开?',  # 中文引号

# 正确写法
status VARCHAR(20) DEFAULT '待开?',  # 使用英文单引号或转义
```

### 🔴 ERR-SYS-004 - 缩进错误

**错误信息**:
```
IndentationError: unexpected indent
IndentationError: unindent does not match any outer indentation level
```

**发生时间**: 2026-04-29 17:25:44, 17:27:14, 17:29:52

**根本原因**:
- 混合使用Tab和空格
- if/else/for块缩进不一致

**解决方案**:
1. 统一使用4空格缩进
2. 在IDE中设置"显示空白字符"
3. 使用 `python -m py_compile file.py` 检查语法

### 🔴 ERR-DB-009 - 字段默认值错误

**错误信息**:
```
OperationalError: (1101, "BLOB, TEXT, GEOMETRY or JSON column 'product_types_json' can't have a default value")
```

**发生时间**: 2026-04-30 20:45:05

**根本原因**:
- MySQL模式下TEXT类型不能有默认值
- 迁移脚本中的默认值配置错误

**解决方案**:
```sql
-- 错误
product_types_json TEXT DEFAULT '[]'

-- 正确
product_types_json TEXT NULL
```
