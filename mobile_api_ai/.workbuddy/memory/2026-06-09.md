## 修复报工后工序合计不更新

### 问题
报工提交后，再次打开报工页面，工序下拉菜单的合计数值还是旧的。

### 根因
`submitReport()` 成功后虽然调 `goToOrder()` 刷新了 `selectedOrder`，但刷新后只调 `showDetail()`（详情页）。如果用户再次打开报工页面，`showReport()` 在打开时渲染工序下拉，数据源是之前缓存的 `selectedOrder`，里面的 `processes[].completed_qty` 还是旧值。

### 修复
`submitReport()` 成功后先 `await goToOrder()` 刷新最新数据，然后检测当前是否在报工页面（`report-page.active`），如果在的话立刻重新调 `showReport()` 渲染。

### 文件变更
| 文件 | 改动 |
|------|------|
| `templates/mobile_unified.html` | `submitReport()` 中 await goToOrder → 检测 report-page 状态 → 自动刷新工序下拉 |
