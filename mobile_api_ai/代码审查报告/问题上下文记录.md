## 一、项目常见错误模式与正确写法对照

### 1.1 异常处理

| 模式 | ❌ 错误写法 | ✅ 正确写法 |
|------|-----------|-----------|
| 裸 except | `except:` | `except Exception as e:` |
| 静默吞异常 | `except: pass` | `except Exception as e: logger.error(...)` |
| 吞异常返回默认值 | `except: return None` | `except Exception as e: logger.error(...); return None` |

### 1.2 日志使用

| 模式 | ❌ 错误写法 | ✅ 正确写法 |
|------|-----------|-----------|
| 生产代码打印 | `print(f"错误: {e}")` | `logger.error(f"错误: {e}")` |
| 状态提示 | `print("服务启动...")` | `logger.info("服务启动...")` |
| 警告信息 | `print("[警告] 连接失败")` | `logger.warning("连接失败")` |

### 1.3 配置管理

| 模式 | ❌ 错误写法 | ✅ 正确写法 |
|------|-----------|-----------|
| 端口硬编码 | `port=5003` | `config.Config.WECHAT_BOT_PORT` |
| 阈值硬编码 | `MAX_TEXT_LENGTH = 2048` | `Config.MAX_TEXT_LENGTH` |
| 超时硬编码 | `DEFAULT_TIMEOUT = 300` | `Config.SESSION_TIMEOUT` |

### 1.4 路径管理

| 模式 | ❌ 错误写法 | ✅ 正确写法 |
|------|-----------|-----------|
| sys.path 分散 | 每个模块 `sys.path.insert(0, ...)` | 仅在入口设置，其他模块 `from config import BASE_DIR` |
| 相对路径 | `os.path.join(__file__, '..', '..')` | `os.path.join(BASE_DIR, 'subdir')` |

### 1.5 Flask启动配置

| 模式 | ❌ 错误写法 | ✅ 正确写法 |
|------|-----------|-----------|
| 调试模式 | `app.run(debug=True)` | `app.run(debug=False)` |
| 端口硬编码 | `app.run(port=5002)` | `app.run(port=config.Config.CONTAINER_CENTER_PORT)` |

## 四、项目规范要点速查

### 4.1 文件结构
```
mobile_api_ai/
├── config.py           # 统一配置（端口、阈值、路径）
├── app.py              # Flask 入口
├── wechat_server.py    # 微信服务
├── storage_layer.py    # 存储抽象层
├── bots/               # 机器人模块
├── api/                # API 模块
├── services/           # 服务模块
├── commands/           # 命令模块
├── models/             # 数据模型
├── container/          # 容器中心
└── modules/            # 工具模块
```

### 4.2 配置引用规范
```python
# config.py 统一管理
from config import Config

# 端口使用
Config.WECHAT_BOT_PORT
Config.CONTAINER_CENTER_PORT
Config.HTTP_TEST_PORT

# 阈值使用
Config.MAX_TEXT_LENGTH
Config.SESSION_TIMEOUT
Config.JWT_EXPIRE_HOURS

# 路径使用
from config import BASE_DIR
os.path.join(BASE_DIR, 'data', 'file.db')
```

### 4.3 异常处理规范
```python
import logging
logger = logging.getLogger(__name__)

try:
    result = some_function()
except Exception as e:
    logger.error(f"操作失败: {e}")
    # 可选择：return default_value / raise
```

### 4.4 日志规范
```python
logger.info("服务启动完成")           # 正常信息
logger.warning("连接失败，重试中...")  # 可恢复问题
logger.error("操作失败")              # 不可恢复错误
logger.debug("函数入口: x=%s", x)     # 调试信息（避免 f-string）
logger.exception("数据库错误")         # 自动包含traceback
```
