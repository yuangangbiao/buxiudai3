## 2. 速率限制缺失 (HIGH)

### 问题描述

### 受影响范围
- `mobile_api_ai` 全部 5 个 Flask 入口文件
- `不锈钢网带跟单系统3.01` 中 2 个 Flask 入口文件 (`wechat_callback.py`, `dashboard_server.py`)

**搜索验证结果**: 搜索 `Limiter`、`limiter`、`ratelimit`、`rate_limit`、`throttle` 等关键词，两项目均 **零匹配**。

**风险等级**: HIGH  
**影响**: 登录接口可被暴力破解，API 可被大量请求耗尽服务器资源。

## 4. 请求验证跳过 - get_json(force=True, silent=True) (HIGH)

### 问题描述
大量 API 端点使用 `request.get_json(force=True, silent=True)`：
- `force=True`：即使 `Content-Type` 不是 `application/json` 也解析请求体
- `silent=True`：解析失败时静默返回 `None`，不会抛异常

组合使用时，攻击者可发送任意 `Content-Type` 的请求（如表单、纯文本），导致请求验证逻辑被绕过。

## 5. SQL 查询参数化 - 存在危险模式 (MEDIUM)

### 问题描述
虽然大部分查询使用了参数化查询（`%s` 或 `?` 占位符），但 **系统3.01** 中存在多处使用 Python f-string 拼接 SQL 语句的模式，部分存在注入风险。具体分类如下：

### 5a. 安全但易出错的模式 - f-string + %s 占位符 (MEDIUM)

`order.py`、`production.py`、`material_prep_view.py`、`dashboard_server.py` 等文件中大量使用以下模式构建 IN 子句：

```python
placeholders = ",".join(["%s"] * len(order_ids))
cursor.execute(f"""
    UPDATE orders SET ... WHERE id IN ({placeholders})
""", order_ids)
```

虽然 `%s` 是 SQL 占位符而非用户输入，此模式在正确使用时是安全的。但混合 f-string 和参数化查询容易引入人为错误。

### 5b. 危险的 Python 字符串格式化 - 绕过参数化 (MEDIUM)

**文件**: [order.py](file:///d:/yuan/不锈钢网带跟单系统3.01/models/order.py#L909-L913)  
**行号**: L913  
**代码**:
```python
cursor.execute("""
    SELECT COUNT(*) as cnt FROM orders
    WHERE id IN (%s)
    AND COALESCE(is_archived, 0) = 0
""" % (",".join(["%s"] * len(order_ids))), order_ids)
```

此代码使用 Python 的 `%` 运算符对 SQL 字符串先行格式化（替换 `%s` 为 `%s,%s,...`），然后将参数传入 cursor.execute()。虽然当前未直接导致注入，但 `%` 格式化与 SQL 参数化的混用极易出错，若后续修改不小心传入用户数据则直接变成 SQL 注入。

**文件**: [order.py](file:///d:/yuan/不锈钢网带跟单系统3.01/models/order.py#L648)  
**模式复用**: 查询批量订单时同样使用 `f"""...IN ({placeholders})"""` 模式，placeholders 由固定 `%s` 拼接，参数通过 tuple 传入，当前可接受但风险较高。

**风险等级**: MEDIUM  
**影响**: 当前未导致直接 SQL 注入，但混合 f-string 和参数化查询的模式容易在后续维护中引入注入漏洞。

## 修复优先级清单

| 优先级 | 问题 | 建议修复方案 |
|--------|------|-------------|
| **P0 - 立即修复** | JWT 空密钥回退 | 移除 `''` 默认值，改为启动时检查环境变量是否存在，不存在则报错退出 |
| **P0 - 立即修复** | CORS 允许所有来源 | 改为 `CORS(app, origins=['http://localhost:8080', ...])` 明确指定允许的来源 |
| **P1 - 紧急修复** | 缺少速率限制 | 引入 Flask-Limiter，对 `/api/auth/login` 等敏感端点限制 5次/分钟 |
| **P1 - 紧急修复** | get_json(force=True, silent=True) | 移除 `force=True`，使用标准 `request.get_json()` 并处理 415 错误 |
| **P2 - 尽快修复** | f-string SQL 拼接 | 使用 `executemany` 或标准参数化查询替代 IN 子句的 f-string 拼接 |
| **P3 - 常规修复** | 绑定 0.0.0.0 | 配置中明确监听地址，生产环境建议绑定 127.0.0.1 + nginx 反向代理 |
