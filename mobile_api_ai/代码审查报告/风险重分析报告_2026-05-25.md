### R1. CORS 全开放

#### 问题描述
两个核心服务使用默认 CORS 配置，允许任意来源跨域访问：

```
dispatch_center.py:L8328     →  CORS(app)                     # 无任何参数
container_center_api.py:L150 →  CORS(app, origins='*', ...)   # origins 默认值 '*'
```

#### 影响评估
- 攻击者可在任意第三方页面发起跨域请求
- 结合内部 API（如无需认证的端点），可导致数据泄露或操作越权
- 内网横向移动时，攻击者可从任意内部服务发起 CSRF 攻击

#### 根因分析
- 开发阶段为了调试方便使用了 `CORS(app)` 默认配置
- 缺乏安全上线检查清单
- `container_center_api.py` 虽然使用了环境变量，但默认值设置为 `'*'`，形同虚设

#### 修复建议
```python
# dispatch_center.py 修复方案
# 方式1：从环境变量读取白名单（推荐）
CORS(app, origins=os.getenv('CORS_ALLOWED_ORIGINS', 'http://localhost:5000,http://localhost:3000').split(','),
      supports_credentials=True)

# 方式2：如果仅为本地开发，明确限制
CORS(app, origins=['http://localhost:5000', 'http://127.0.0.1:5000'],
      supports_credentials=True)
```

```python
# container_center_api.py 修复方案
# 将默认值从 '*' 改为明确的本地地址
CORS_ALLOWED = os.getenv('CORS_ALLOWED_ORIGINS', 'http://localhost:5000,http://localhost:3000')
CORS(app, origins=CORS_ALLOWED.split(','), supports_credentials=True)

# L199 的手动 header 同样需要修改
resp.headers['Access-Control-Allow-Origin'] = CORS_ALLOWED.split(',')[0]
```

### R2. get_json(force=True, silent=True) 泛滥使用

#### 问题描述
共 **94 处**分布在 14 个文件中，全部使用 `request.get_json(force=True, silent=True)`：

| 文件 | 数量 | 占比 |
|------|:----:|:----:|
| dispatch_center.py | **44** | 47% |
| container_center_api.py | **19** | 20% |
| face_checkin/__init__.py | **12** | 13% |
| container_api_server.py | **8** | 9% |
| 其余10个文件各1-2处 | **11** | 11% |

#### 影响评估
- `force=True`：允许非 `application/json` 的 Content-Type 传入，可绕过 CSRF 保护（表单提交不再受同源策略限制）
- `silent=True`：解析失败静默返回 `None`，后续 `or {}` 兜底掩盖了请求格式错误
- 组合使用时，攻击者可发送 `Content-Type: text/plain` 或 `multipart/form-data` 的请求，而 Flask 仍会尝试解析 JSON 体
- 有两处无 `or {}` 兜底（`dispatch_center.py:7300`、`api/decorators.py:46`），可能导致 `None` 传递到后续逻辑引发 TypeError

#### 根因分析
- 历史代码积累，早期为了兼容不同客户端请求格式而使用
- 44处集中在 `dispatch_center.py`，说明该文件是历史遗留的重灾区
- 缺乏统一的请求解析中间件或装饰器

#### 修复建议
```python
# 方案1：全局替换为标准模式（推荐给高安全要求端点）
data = request.get_json()
if data is None:
    return jsonify({'code': 415, 'message': '请求体必须为有效的 JSON 格式'}), 415

# 方案2：如果想兼容多种 Content-Type 但保留校验
try:
    data = request.get_json(force=False, silent=False)
except Exception:
    return jsonify({'code': 400, 'message': '无效的 JSON 请求体'}), 400

# 方案3：创建统一装饰器（批量修复44处的最佳方案）
from functools import wraps

def require_json(f):
    @wraps(f)
    def wrapper(*args, **kwargs):
        data = request.get_json()
        if data is None:
            return jsonify({'code': 415, 'message': 'Content-Type 必须为 application/json'}), 415
        kwargs['body'] = data
        return f(*args, **kwargs)
    return wrapper

# 使用方式
@app.route('/api/xxx', methods=['POST'])
@require_json
def some_api(body):
    # body 已确保是有效的 dict
    ...
```

### R3. 缺少 API 速率限制

#### 问题描述
全项目 **无任何标准速率限制机制**。搜索 `Limiter`、`ratelimit`、`rate_limit`、`throttle` 等关键词，仅发现一个与 API 限流无关的 `_throttled_persist`（缓存写入节流）。

#### 影响评估
- 登录接口可被暴力破解（如 `/api/auth/login`）
- 全站 API 可被大量请求耗尽服务器资源
- 企业微信回调接口可被重放攻击
- 无 DDoS 基础防护能力

#### 根因分析
- 项目早期为内部使用，未考虑外部攻击面
- 后期接入企业微信后暴露到外网，未补充安全防护

#### 修复建议
```python
# 安装 flask-limiter
# pip install flask-limiter

# 在 app.py 中统一配置
from flask_limiter import Limiter
from flask_limiter.util import get_remote_address

limiter = Limiter(
    app=app,
    key_func=get_remote_address,
    default_limits=["200 per day", "50 per hour"],
    storage_uri=os.getenv('LIMITER_STORAGE', 'memory://'),
)

# 敏感端点单独限流
@app.route('/api/auth/login', methods=['POST'])
@limiter.limit("5 per minute")
def login():
    ...

# 或使用装饰器方式
from flask_limiter import Limiter
limiter = Limiter(key_func=get_remote_address)

def create_app():
    app = Flask(__name__)
    limiter.init_app(app)
    # 全局默认限制
    app.config['RATELIMIT_DEFAULT'] = '200/day;50/hour'
    # 登录接口严格限制
    app.config['RATELIMIT_STORAGE_URL'] = os.getenv('LIMITER_STORAGE', 'memory://')
    return app

# 批量应用到 dispatch_center.py
limiter = Limiter(app, key_func=get_remote_address, default_limits=["200 per day", "50 per hour"])
```

### R4. JWT 密钥空字符串回退（无校验）

#### 问题描述
[settings.py:L96](file:///d:/yuan/不锈钢网带跟单3.0/mobile_api_ai/settings.py#L96)
```python
secret_key=os.getenv('FLASK_SECRET_KEY', os.getenv('JWT_SECRET_KEY', '')),
#                                                            ↑↑ 空字符串回退，无后续检查
```

对比 `config.py:L26` 和 `settings.py:L61` 都有 `if not secret: raise ValueError(...)` 保护，但 **L96 没有**。

#### 影响评估
- 如果 `FLASK_SECRET_KEY` 和 `JWT_SECRET_KEY` **均未设置**，Flask 的 `secret_key` 为空字符串
- Flask session 签名使用空密钥，攻击者可伪造任意 session cookie
- 可能导致会话劫持、越权访问、CSRF token 绕过

#### 根因分析
- 嵌套 `os.getenv` 导致安全意识减弱
- 缺少统一的安全配置校验入口

#### 修复建议
```python
# settings.py 修复方案

# 方式1：单独提取并校验（推荐）
_jwt_secret = os.getenv('JWT_SECRET_KEY')
_flask_secret = os.getenv('FLASK_SECRET_KEY')

if not _flask_secret and not _jwt_secret:
    raise ValueError("必须设置 FLASK_SECRET_KEY 或 JWT_SECRET_KEY 环境变量")

secret_key = _flask_secret or _jwt_secret

# 方式2：直接从 config.py 引用（如果 config.py 已校验）
from config import SECRET_KEY
# 在 settings.py 中直接使用 config.SECRET_KEY
secret_key = SECRET_KEY
```

### R10. Batch1 DDL 与设计文档严重偏差

#### 问题描述
3 张新表的 DDL 实现与 DESIGN 文档存在严重偏差：

| 表 | 问题数量 | 关键问题 |
|---|:--------:|---------|
| customer_contacts | 5项 | **缺少外键约束**、字段长度偏差、缺少默认值 |
| customer_groups | 3项 | **缺少 UNIQUE 约束**、字段名不一致 |
| order_items | **10项** | **缺少外键约束**、**缺少6个核心字段**、字段命名不一致、默认值不一致 |

#### 影响评估
- **缺少外键约束**：数据完整性无法保障，可出现 `customer_id` 引用不存在的客户，`order_id` 引用不存在的订单
- **缺少 UNIQUE 约束**：`group_name` 允许重复，业务逻辑依赖的唯一性无法保证
- **缺少6个核心字段**：`item_no`、`material`、`length`、`width`、`delivery_date` 等业务必需字段缺失，功能不完整
- 表结构歪斜导致后期迁移成本极高

#### 根因分析
- DDL 由 AI 生成，未严格按 DESIGN 文档校验
- 缺少 DDL 评审环节
- 3 张新表未放入 `_migrate_tables()` 函数，而是挂在 `init_db()` 末尾

#### 修复建议
```python
# 1. 将 3 张新表 DDL 移入 _migrate_tables()

# 2. customer_contacts 表修复示例
def _migrate_customer_contacts(cursor):
    cursor.execute("""
        CREATE TABLE IF NOT EXISTS customer_contacts (
            id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
            customer_id INT UNSIGNED NOT NULL,
            name VARCHAR(64) NOT NULL COMMENT '联系人姓名',
            phone VARCHAR(32) DEFAULT '' COMMENT '联系电话',
            email VARCHAR(128) DEFAULT '' COMMENT '邮箱',
            position VARCHAR(64) DEFAULT '' COMMENT '职位',
            is_primary TINYINT(1) DEFAULT 0 COMMENT '是否主要联系人',
            remark TEXT COMMENT '备注',
            created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
            updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
            created_by INT DEFAULT 0,
            updated_by INT DEFAULT 0,
            is_deleted TINYINT(1) DEFAULT 0,
            deleted_at DATETIME DEFAULT NULL,
            deleted_by INT DEFAULT 0,
            version INT DEFAULT 1,
            FOREIGN KEY (customer_id) REFERENCES customers(id) ON DELETE CASCADE,
            INDEX idx_customer_id (customer_id),
            INDEX idx_is_deleted (is_deleted)
        ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='客户联系人'
    """)

# 3. customer_groups 添加 UNIQUE
def _migrate_customer_groups(cursor):
    cursor.execute("""
        CREATE TABLE IF NOT EXISTS customer_groups (
            id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
            group_name VARCHAR(64) NOT NULL UNIQUE COMMENT '分组名称',
            group_desc VARCHAR(256) DEFAULT '' COMMENT '分组描述',
            sort_order INT DEFAULT 0 COMMENT '排序',
            created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
            updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
            created_by INT DEFAULT 0,
            updated_by INT DEFAULT 0,
            is_deleted TINYINT(1) DEFAULT 0,
            deleted_at DATETIME DEFAULT NULL,
            deleted_by INT DEFAULT 0,
            version INT DEFAULT 1,
            INDEX idx_group_name (group_name),
            INDEX idx_is_deleted (is_deleted)
        ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='客户分组'
    """)

# 4. order_items 完整重构
def _migrate_order_items(cursor):
    cursor.execute("""
        CREATE TABLE IF NOT EXISTS order_items (
            id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
            order_id INT UNSIGNED NOT NULL,
            item_no INT DEFAULT 0 COMMENT '行号',
            product_name VARCHAR(256) NOT NULL COMMENT '产品名称',
            specification VARCHAR(256) DEFAULT '' COMMENT '规格型号',
            material VARCHAR(128) DEFAULT '' COMMENT '材质',
            length DECIMAL(10,2) DEFAULT 0 COMMENT '长度(m)',
```
