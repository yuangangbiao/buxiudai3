## 2. 🔴 严重安全问题（Critical）

### 2.1 [CRITICAL] SQL 注入漏洞（routes_data.py:80）

**位置**：[routes_data.py:80](file:///D:/yuan/不锈钢网带跟单3.0/mobile_api_ai/inventory_web/routes_data.py#L75-L82)

**漏洞代码**：
```python
@bp.route('/inventory/base/<entity>/add', methods=['POST'])
def base_add(entity):
    data = request.get_json() or {}
    tables = {'warehouse': 'warehouses', 'category': 'categories', 'supplier': 'suppliers'}
    if entity not in tables: return jsonify({'ok': False}), 400
    execute("INSERT INTO " + tables[entity] + " (code, name, contact) VALUES (%s,%s,%s)",
            (data.get('code',''), data.get('name',''), data.get('contact','')))
```

**风险等级**：🔴 **严重**（虽然此处 entity 已经在白名单中，但字段名是用户输入）

**攻击示例**：
```json
{"name": "test', (SELECT password FROM mysql.user LIMIT 1), 'contact": "x'"}
```

**修复方案**：
```python
ALLOWED_FIELDS = ['code', 'name', 'contact']
fields = [k for k in data.keys() if k in ALLOWED_FIELDS]
# ...完整白名单
```

### 2.2 [CRITICAL] 路径遍历漏洞（routes_system.py:36-50）

**位置**：[routes_system.py:36-50](file:///D:/yuan/不锈钢网带跟单3.0/mobile_api_ai/inventory_web/routes_system.py#L36-L50)

**漏洞代码**：
```python
@bp.route('/inventory/backup/restore/<filename>', methods=['POST'])
def backup_restore(filename):
    backup_dir = os.path.join(PROJECT_ROOT, 'inventory_backups')
    filepath = os.path.join(backup_dir, filename)
    if not os.path.exists(filepath): return jsonify({'ok': False, 'msg': '文件不存在'}), 404
    ...
    subprocess.run(cmd, stdin=f, ...)
```

**风险等级**：🔴 **严重**
- `filename` 参数可注入 `../../etc/passwd` 等路径
- 攻击者可读取/恢复任意文件

**修复方案**：
```python

safe_filename = os.path.basename(filename)  # 剥离路径
if safe_filename != filename:
    return jsonify({'ok': False, 'msg': '非法文件名'}), 400
# 或使用 werkzeug.utils.secure_filename
```

### 2.3 [CRITICAL] 子进程命令注入（routes_system.py:28）

**位置**：[routes_system.py:28](file:///D:/yuan/不锈钢网带跟单3.0/mobile_api_ai/inventory_web/routes_system.py#L28)

**漏洞代码**：
```python
host = os.getenv("MYSQL_HOST","localhost")
```

**位置**：[routes_api.py:41](file:///D:/yuan/不锈钢网带跟单3.0/mobile_api_ai/inventory_web/routes_api.py#L41)

### 2.5 [CRITICAL] SQL 注入（routes_api.py:12-13）

**位置**：[routes_api.py:12-13](file:///D:/yuan/不锈钢网带跟单3.0/mobile_api_ai/inventory_web/routes_api.py#L8-L14)

**漏洞代码**：
```python
@bp.route('/inventory/api/<entity>/<int:eid>', methods=['PATCH'])
def api_update(entity, eid):
    if entity not in tables: return jsonify({'ok':False}),400
    data = request.get_json() or {}
    set_clause = ', '.join('`' + k + '`=%s' for k in data.keys())
    execute("UPDATE %s SET %s WHERE id=%%s" % (tables[entity], set_clause), list(data.values()) + [eid])
```

**风险等级**：🔴 **严重**
- `data.keys()` 是用户输入，可构造任意列名
- 攻击者可更新任意字段（如 `role='admin'`）

**攻击示例**：
```json
{"unit_price": 9999, "category_id=1, name=(SELECT password FROM users) WHERE 1=1 --": "x"}
```

**修复方案**：
```python
ALLOWED_UPDATE_FIELDS = {
    'product': ['name', 'spec', 'unit', 'category_id', 'price'],
    'supplier': ['name', 'contact', 'phone', 'address'],
    # ...
}
fields = [k for k in data.keys() if k in ALLOWED_UPDATE_FIELDS[entity]]
```

## 3. 🟠 高风险问题

### 3.1 [HIGH] 缺少输入验证

**统计**：35 个路由中 **30+ 个没有参数验证**

**示例**：
- [routes_core.py:60](file:///D:/yuan/不锈钢网带跟单3.0/mobile_api_ai/inventory_web/routes_core.py#L57-L70) `stock_in` 直接读取 `data.get('id')`，未验证是否为整数
- `batch_do` (line 157) 接受任意 items，无长度限制

**风险**：类型错误、注入攻击、数据污染

**修复**：使用 `pydantic` 或 `marshmallow` 进行 schema 验证

### 3.2 [HIGH] 缺少并发控制

**问题**：库存台账、库存出入库存在并发竞争条件

**位置**：[routes_core.py:64](file:///D:/yuan/不锈钢网带跟单3.0/mobile_api_ai/inventory_web/routes_core.py#L57-L70)

**漏洞代码**：
```python
c.execute("UPDATE inventory SET current_qty = current_qty + %s WHERE id = %s", (qty, inv_id))
```

**风险**：
- 同时出库时可能出现库存负数（虽然 `outbound_do` 用 `GREATEST(0,...)` 兜底）
- `batch_do` 中循环执行 N 次 UPDATE，无事务隔离
- 多个并发请求可能导致数据不一致

**修复方案**：
```python
# 使用 SELECT ... FOR UPDATE 加锁
c.execute("SELECT current_qty FROM inventory WHERE id=%s FOR UPDATE", (inv_id,))
# 或使用乐观锁（version 字段）
```

### 3.3 [HIGH] 事务不一致

**位置**：[routes_core.py:62-69](file:///D:/yuan/不锈钢网带跟单3.0/mobile_api_ai/inventory_web/routes_core.py#L57-L70)

**问题**：
```python
c.execute("UPDATE inventory ...")
c.execute("INSERT INTO inventory_transactions ...")
conn.commit()
```

**风险**：
- `stock_in` 和 `stock_out` 没有 try-except 包裹，异常时部分写入不会回滚
- `batch_do` 在循环中执行多个事务，全部或部分失败时不回滚

**修复**：
```python
try:
    c.execute(...)
    c.execute(...)
    conn.commit()
except Exception as e:
    conn.rollback()
    return jsonify({'ok': False, 'msg': str(e)}), 500
```

### 3.4 [HIGH] 缺少权限控制

**问题**：35 个路由中**0 个**有权限验证

**影响**：
- 任何用户都能 `POST /inventory/backup/restore/<filename>` 恢复数据库
- 任何用户都能 `DELETE /inventory/api/product/1` 删除数据
- 没有管理员/普通用户区分

**修复**：添加登录认证和 RBAC

### 3.5 [HIGH] SQL 错误暴露

**位置**：多处

**问题**：所有异常处理都直接 `str(e)` 返回给客户端
```python
except Exception as e:
    return jsonify({'ok': False, 'msg': str(e)}), 500
```

**风险**：
- 暴露 SQL 语句、表名、列名
- 暴露文件路径
- 暴露数据库版本

**修复**：
```python
except Exception as e:
    logger.exception(f"[{endpoint}] error")
    return jsonify({'ok': False, 'msg': '内部错误'}), 500
```
