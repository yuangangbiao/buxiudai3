### 2.2 主应用集成点

#### app.py 集成建议
```python
# 可选集成（推荐在配置开启时启用）
from request_tracing import init_tracing
from cache_warmup import warmup_cache

# 在 create_app() 末尾添加
if os.getenv('ENABLE_TRACING', 'false') == 'true':
    init_tracing(app)

# 在应用启动时（main函数）
if os.getenv('ENABLE_CACHE_WARMUP', 'false') == 'true':
    warmup_cache()
```

#### requirements.txt 更新

## 三、性能影响评估

### 3.1 缓存预热模块
| 指标 | 影响值 | 说明 |
|------|--------|------|
| 首次启动延迟 | +1-3秒 | 取决于数据库查询速度 |
| 内存占用 | +5-20MB | 预热数据量决定 |
| 运行时影响 | 无 | 后台异步执行 |

**优化建议**:
- 首次部署时可关闭预热 (`ENABLE_CACHE_WARMUP=false`)
- 生产环境建议开启，减少冷启动延迟

### 3.2 请求追踪中间件
| 指标 | 影响值 | 说明 |
|------|--------|------|
| 请求延迟 | +0.1-0.5ms | 极小 |
| 内存占用 | +1-2KB/请求 | 追踪日志缓存 |
| 磁盘写入 | 按需 | 仅记录慢请求 |

**优化建议**:
- 生产环境可设置 `TRACE_SLOW_ONLY=true`
- 追踪日志自动清理（最大1000条）

### 3.3 数据导出模块
| 指标 | 影响值 | 说明 |
|------|--------|------|
| 导出时间 | +0.5-5秒 | 取决于数据量 |
| 内存占用 | +10-100MB | 大数据导出 |

**优化建议**:
- 超过10000行数据建议分批导出
- 使用CSV格式减少内存占用
