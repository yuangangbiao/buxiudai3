#### C-2.2 `process_records` 字段误用

**位置**：方案 §三-1️⃣ 生产日报 SQL

```sql
-- 方案中：
SUM(pr.completed_task_count) AS 完成数
SUM(pr.task_count) AS 计划数
```

**问题**：`container_center.process_records` 表中 `task_count` 和 `completed_task_count` 字段实际是"任务数"，不是"生产数"。生产数量应该用 `quantity` 字段（quantity 是订单数量）。

```sql
SUM(pr.quantity) AS 计划数,            -- 订单数量

```

#### C-2.3 `process_sub_steps` `record_date` 字段错误

**位置**：方案 §三-3️⃣ 车间产能分析 SQL

```sql

-- 实际字段：record_date DATE DEFAULT NULL
-- 问题：如果 record_date 为 NULL，DATE(NULL) 返回 NULL，会丢失记录
```

#### C-2.4 `process_records.steps` JSON 解析风险

**位置**：方案 §三-4️⃣ 工单进度跟踪 SQL

```sql
JSON_UNQUOTE(JSON_EXTRACT(pr.steps, CONCAT('$[', pr.current_step, '].name'))) AS 当前工序
```

**问题**：
- `current_step` 可能是 0，数组下标 0 合法
- 但 `steps` 可能是 NULL，`JSON_EXTRACT(NULL, ...)` 返回 NULL（OK）
- **如果 current_step >= JSON_LENGTH(steps)，会返回 NULL**
- 需要加越界保护

**修复**：
```sql
CASE 
  WHEN pr.steps IS NOT NULL 
    AND pr.current_step < JSON_LENGTH(pr.steps)
  THEN JSON_UNQUOTE(JSON_EXTRACT(pr.steps, CONCAT('$[', pr.current_step, '].name')))
  ELSE NULL
END AS 当前工序
```

## 三、HIGH 级问题

### H-1：跨库事务一致性未处理

**位置**：整体方案

**问题**：
- stats_exporter 需要从 3 个 MySQL 库读数据
- 但 5005 → 5004 转发写入智能表格是异步操作
- 如果智能表格写入失败，数据已经在本地读取过，**没有事务回滚**

**风险**：
- 失败后智能表格少数据，下次重试会重复写入（无幂等性）
- 没有 sync 记录，无法追踪哪些数据已写入

**修复方案**：
- 在 `inventory_db` 中新增 `stats_sync_log` 表
- 记录每批写入的 hash（基于表类型+日期+记录数）
- 写入前先查 sync_log，避免重复

```sql
CREATE TABLE stats_sync_log (
    id INT AUTO_INCREMENT PRIMARY KEY,
    table_type VARCHAR(64) NOT NULL,
    period_key VARCHAR(64) NOT NULL,  -- 日期/周次/月份
    record_count INT NOT NULL,
    record_hash VARCHAR(64) NOT NULL,  -- SHA256
    sync_status ENUM('success','failed') DEFAULT 'success',
    synced_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_table_period (table_type, period_key)
);
```

### H-2：缺少幂等性保护

**位置**：5005/5004 端点

**问题**：
- Webhook 智能表格 API "仅支持追加写入"
- 同一批数据重复推送会重复写入
- 没有去重机制

**修复方案**：
- 本地 stats_exporter 生成每批数据的 `batch_id` (UUID)
- 每次推送携带 `batch_id`
- 云端 5004 记录已处理的 `batch_id`，避免重复

```python
# 5004 端
processed_batches = set()  # 实际用 Redis 持久化
def smartsheet_write():
    batch_id = data.get('batch_id')
    if batch_id in processed_batches:
        return jsonify({'code': 0, 'message': '已处理，跳过'})
    processed_batches.add(batch_id)
```

### H-4：失败重试未设计

**位置**：触发器设计

**问题**：
- 方案说"三触发器统一入口"
- 但失败时没有重试机制
- 网络抖动或 5004 临时不可用，会丢失一批数据

**修复方案**：
```python
# stats_smart_sheet/smart_sheet_exporter.py
def push_with_retry(table_type, records, max_retries=3):
    for attempt in range(max_retries):
        try:
            return push_to_5005(table_type, records)
        except Exception as e:
            if attempt == max_retries - 1:
                # 写入失败队列，后续人工处理
                save_to_failure_queue(table_type, records, str(e))
                raise
            time.sleep(2 ** attempt)  # 指数退避
```

### H-5：缺少并发控制

**位置**：本地 stats_exporter

**问题**：
