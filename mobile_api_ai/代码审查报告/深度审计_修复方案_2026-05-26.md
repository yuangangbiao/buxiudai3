## 二、架构层缺陷（CRITICAL）

### D1. 双 Config 系统 — 修复目标错误

**严重度**: CRITICAL
**描述**: DESIGN 文档全篇假设 `mobile_api_ai/config.py` 是统一配置入口，但实际运行时：

| 文件 | 角色 | 位置 |
|------|------|------|
| `core/config.py` | **实际运行时配置入口** | `d:\yuan\不锈钢网带跟单3.0\core\config.py` |
| `mobile_api_ai/config.py` | 仅 Flask 应用配置 + 路径设置 | `d:\yuan\不锈钢网带跟单3.0\mobile_api_ai\config.py` |

**证据**:
- dispatch_center.py L73: `from core.config import ...`（导入的是 `core/config.py` 而非 `mobile_api_ai/config.py`）
- dispatch_center.py L88: MYSQL_CFG 直接使用 `int(os.environ.get('MYSQL_PORT', 3306))` 而非从 config 导入

**影响**: 按当前 DESIGN 实施会在错误位置添加配置，导致运行时不生效

### D2. sys.path 集中方案存在循环依赖风险

**严重度**: HIGH
**描述**: DESIGN 建议将所有 `sys.path` 集中到 `config.py`，但：
- `mobile_api_ai/config.py` L15: 已有 `sys.path.insert(0, BASE_DIR)`（模块级执行）
- 其他模块在 import `config` 时就会触发此插入，**本已生效**
- `core/config.py` 不需要 `sys.path`（0处，已验证）
- 生产代码中 60 处 `sys.path` 调用中有许多是**冗余但不危险**的

**影响**: 大规模移除可能导致 import 顺序变化，需逐处验证
