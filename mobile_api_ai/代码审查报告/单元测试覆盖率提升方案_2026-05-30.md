## 5. 测试规范

### 命名规范
```
test_<module>.py
class Test<ClassName>:
    def test_<method>_<scenario>(self, fixture):
        ...
```

### 通用 fixture (conftest.py 扩展)

需要新增以下 fixtures：
- `mock_storage` (已存在)
- `mock_db` (已存在)
- `mock_api_client` (已存在)
- `mock_redis` (新增)
- `mock_wechat_api` (新增)
- `mock_http_response` (新增)

### 测试模板

```python
# -*- coding: utf-8 -*-
"""
<ModuleName> 单元测试
覆盖：<功能描述>
"""
import pytest
from unittest.mock import MagicMock, patch


class TestClassName:
    """<类名> 单元测试"""

    def test_method_success(self, mock_db):
        """测试正常流程"""
        # Arrange
        ...

        # Act
        result = target.method()

        # Assert
        assert result is not None

    def test_method_exception(self, mock_db):
        """测试异常情况"""
        ...
```

## 6. 验收标准

### 自动化验证

1. **pytest.ini 配置更新**：
   ```ini
   addopts = [
       "--cov=api",
       "--cov=models",
       "--cov=storage_mysql",
       "--cov=utils",
       "--cov=services",
       "--cov=container",
       "--cov=container_center_v5",
       "--cov-report=term-missing",
       "--cov-fail-under=60",  # 从 40 提升到 60
   ]
   ```

2. **运行命令**：
   ```bash
   pytest --cov --cov-fail-under=60
   ```

3. **CI 集成**：
   - `.github/workflows/ci.yml` 已配置测试
   - 合并请求必须通过 60% 门槛

### 手工验证

- 每个新测试至少包含 1 个正向、1 个反向用例
- 异常路径必须被测试覆盖
- 边界值必须有专门测试
