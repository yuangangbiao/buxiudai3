## 测试 3: `test_json_field_safety.py`

### 设计

```python
class TestJsonFieldSafety:
    def test_content_string_parse_before_dict_access(self):
        """模拟 MySQL TEXT 列读出的 JSON 字符串，验证 parse → dict 流程"""
        import json
        raw = '{"plan_start": "2026-06-01", "plan_end": "2026-06-05"}'
        
        # 危险写法: 字符串当 dict
        with pytest.raises((TypeError, AttributeError)):
            raw['plan_start'] = "2026-06-01"  # str object does not support item assignment
        
        # 安全写法: 先解析
        parsed = json.loads(raw) if isinstance(raw, str) else (raw or {})
        parsed['plan_start'] = "2026-06-01"
        assert parsed['plan_start'] == "2026-06-01"

    def test_content_or_empty_dict_trap(self):
        """字符串是 truthy 的, .get('content') or {} 不触发"""
        record = {'content': '{"plan_start": "2026-06-01"}'}
        val = record.get('content') or {}
        assert isinstance(val, str), f"OR {} 未触发, content 仍是 str 不是 dict"
        # 这说明需要 json.loads
        import json
        safe = json.loads(val) if isinstance(val, str) else val
        assert isinstance(safe, dict)

    def test_schedule_routes_content_access(self):
        """回归测试: schedule_routes 中已修复的 content 解析逻辑"""
        import json
        # 模拟 MySQL 返回的 record
        record = {
            'order_no': 'ORD-202604210004',
            'content': '{"plan_start": "2026-06-01", "plan_end": "2026-06-05", "schedule_days": 4}'
        }
        # 已修复逻辑
        _content = record.get('content')
        if isinstance(_content, str):
            _content = json.loads(_content)
        if not isinstance(_content, dict):
            _content = {}
        _content['plan_start'] = '2026-06-01'
        assert _content['plan_start'] == '2026-06-01'
```

### 审计

| 维度 | 评估 | 说明 |
|------|------|------|
| 可行性 | ✅ | 纯逻辑测试，无需外部依赖 |
| 假阳性 | 🟢 无 | 测试固定的输入输出 |
| 假阴性 | 🟡 中 | 只验证已知模式，新写法防不住 |
| 运行前提 | 无外部依赖 | 不需数据库 |
| 运行速度 | <0.5秒 | 纯 Python |
| 投入产出 | 🟡 | 更像"文档"而不是"防护"，preflight 模式扫描已覆盖此风险 |

**结论: 价值有限（preflight 已做静态扫描）。可选做，但优先级低于前两个。**
