# 容器中心服务器稳定性深度诊断报告

> 诊断日期: 2026-05-18
> 目标服务: container_center_api.py (容器中心API服务器)
> 数据文件: storage_layer.py (SQLite 存储层)
> 诊断方法: 代码审查 + 日志分析

---

## 一、执行摘要

容器中心服务器（端口 5002）存在 **10 个关键问题**，其中 **2 个 P0 级致命问题**直接导致进程无法长期稳定运行。根因链如下：

```
Flask开发服务器单线程(Werkzeug默认)
    → 请求串行处理
        → SQLite缺少WAL模式(共享连接+多线程)
            → 写操作锁表导致请求阻塞
                → 进程无响应(假死)
                    → 无自动重启机制
                        → 服务永久停摆
```

服务器典型崩溃模式：运行数小时至数天后，SQLite 写锁定导致请求线程阻塞，Flask 单线程无法处理新请求，客户端看到"连接超时/服务不可用"，进程被系统判定为假死后无任何恢复机制。

#### P0-1: SQLite 缺少 WAL 模式

**现状代码：**
```python
# storage_layer.py L438-L448
def connect(self):
    if self._conn is None:
        self._conn = sqlite3.connect(self.db_path, check_same_thread=False)
        self._conn.row_factory = sqlite3.Row
```

**问题分析：**
- 使用 `check_same_thread=False` 允许跨线程访问，但未设置 `PRAGMA journal_mode=WAL`
- SQLite 默认使用 `delete` 日记模式，读操作不阻塞但写操作锁定整个数据库文件
- 写操作期间所有其他线程（包括 HTTP 请求线程）的读写全部阻塞
- 多线程并发（主请求线程 + 定时校对线程 + 启动推送线程 + 桌面端回调线程）显著增加写竞争概率
- 长时间写阻塞被操作系统判定为进程假死

**修复方案（必须同时实施）：**
```python
def connect(self):
    if self._conn is None:
        self._conn = sqlite3.connect(self.db_path, check_same_thread=False)
        self._conn.row_factory = sqlite3.Row
        self._conn.execute('PRAGMA journal_mode=WAL')        # ← WAL模式
        self._conn.execute('PRAGMA busy_timeout=5000')       # ← 忙等待5秒
        self._conn.execute('PRAGMA synchronous=NORMAL')      # ← 平衡性能与安全
```

**现状代码：**

```python
def get_all_process_records(self, limit: int = None, offset: int = 0) -> List[Dict]:
    try:
        cursor = self._conn.cursor()
        if limit:
            cursor.execute('SELECT * FROM process_records LIMIT ? OFFSET ?', (limit, offset))
        else:
            cursor.execute('SELECT * FROM process_records')
        rows = cursor.fetchall()
        return [self._process_row_to_dict(row) for row in rows]
```

#### P2-1: 共享 SQLite 连接跨线程无锁保护

**问题分析：**
- `self._conn = sqlite3.connect(self.db_path, check_same_thread=False)` 禁用线程安全检查
- 定时校对线程、启动推送线程、桌面端回调线程与主请求线程共用同一连接
- 虽 SQLite 本身有文件级锁，但 Python sqlite3 模块的 Connection 对象不是线程安全的
- 多线程同时调用 `cursor()` 可能导致 `sqlite3.ProgrammingError: SQLite objects created in a thread can only be used in that thread`

**修复方案：**
```python
import threading

class SQLiteStorage(BaseStorage):
    def __init__(self, config: Dict):
        self._lock = threading.Lock()    # ← 添加线程锁
    
    def _execute(self, sql: str, params: tuple = None):
        """线程安全的数据库执行"""
        with self._lock:
            cursor = self._conn.cursor()
            if params:
                cursor.execute(sql, params)
            else:
                cursor.execute(sql)
            return cursor
```

#### P2-2: 无优雅关闭机制

**当前状态：**
- 整个 `if __name__ == '__main__':` 块无任何 signal handler 注册
- `disconnect()` 只执行 `self._conn.close()`，无 WAL checkpoint
- `desktop_callback_manager.stop()` 永远不会被调用（因为无人调用它）
- 强制关闭进程（Ctrl+C / taskkill）时，正在进行的写操作可能中断，数据库文件损坏

**修复方案：**
```python
import signal
import sys

def graceful_shutdown(signum, frame):
    logger.info('接收到关闭信号，正在优雅关闭...')
    desktop_callback_manager.stop()
    container_center.storage.disconnect()
    logger.info('服务已关闭')
    sys.exit(0)

signal.signal(signal.SIGINT, graceful_shutdown)
signal.signal(signal.SIGTERM, graceful_shutdown)
```

同时增强 `disconnect()`：
```python
def disconnect(self):
    try:
        if self._conn:
            self._conn.execute('PRAGMA wal_checkpoint(TRUNCATE)')  # ← WAL checkpoint
            self._conn.close()
    except Exception as e:
        logger.error(f'断开数据库连接失败: {e}')
    finally:
        self._conn = None
```

#### P2-3: 后台 Daemon 线程脆弱

**问题分析：**
- `_sync_check_sender` 是一个无限 `while True` 循环
- 如果循环体内抛出未被捕获的异常（即使有 `try/except`，但 `get_all_process_records` 外的异常仍可能逃逸），线程直接死亡
- `daemon=True` 意味着主线程不会等待它，线程静默死亡后无人知晓
- 没有任何线程健康检查或自动重启机制

**线程列表：**

| 线程 | 启动位置 | 类型 | 风险 |
|------|----------|------|------|
| 定时校对线程 | container_center_api.py:L577 | daemon=True | 异常退出后永不恢复 |
| 桌面端回调分发 | desktop_callback.py:L33 | daemon=True | 同上 |
| 启动推送线程 | container_center_api.py:L568 (临时) | daemon=True | 同上 |

**修复方案：**
```python
class ResilientThread(threading.Thread):
    """带自动重启的弹性线程"""
    def __init__(self, target, name=None, restart_delay=10):
        super().__init__(daemon=True)
        self._target_fn = target
        self._restart_delay = restart_delay
        self.name = name or target.__name__

    def run(self):
        while True:
            try:
                self._target_fn()
            except Exception as e:
                logger.error(f'线程 [{self.name}] 异常退出，{self._restart_delay}s后重启: {e}')
                time.sleep(self._restart_delay)
```
