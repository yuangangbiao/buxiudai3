| 维度 | 内容 |

| **是否可恢复** | ❌ 否（已物理删除） |
| **业务影响** | 🟡 低（仅辅助调试材料） |
| **追溯影响** | 🟡 低（v3.6.4.1 治理 E2E 测试记录缺失） |
| **修复建议** | 立即重跑 E2E 测试可恢复数据 |

### 4.2 事故根因

1. **dry-run 覆盖不完整**：仅核查 mobile_api_ai/，漏看项目根 logs/ 子目录
2. **清理策略过激**：对项目根 logs/ 全删，未做子目录白名单
3. **缺乏分级执行流程**：75k+ 文件一次性清理无核验点

### 4.3 防范措施

1. 未来清理前必须**全项目 dry-run**（包括所有子目录）
2. 实施**子目录白名单机制**（如 e2e_* / audit_* / *_shots / ux_test_* 目录自动白名单）
3. 实施**分级清理流程**：小批量 dry-run → 中等范围验证 → 大范围执行
4. 添加**运行中服务日志自动保护**：识别正在写入的 .log 文件并自动排除
