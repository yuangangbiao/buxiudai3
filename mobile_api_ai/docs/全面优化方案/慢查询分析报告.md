## Top 5 候选慢查询

### Q1: 生产工单主列表 JOIN 查询

**位置**: [dispatch_center.py:L5154-L5162](file:///d:/yuan/不锈钢网带跟单3.0/mobile_api_ai/dispatch_center.py#L5154-L5162)

```sql
SELECT po.*, o.customer_name, o.product_type, o.quantity,
       o.unit, o.delivery_date, o.order_no
FROM production_orders po
JOIN orders o ON po.order_id = o.id
WHERE (po.is_deleted = 0 OR po.is_deleted IS NULL)
  AND po.status NOT IN ('已完成', '已发货', '订单完成')
ORDER BY po.updated_at DESC
LIMIT 200
```

| 维度 | 分析 |
|------|------|
| **执行频率** | 每次加载主列表视图 |
| **数据量** | production_orders 和 orders 随业务增长 |
| **慢路径** | `NOT IN` 子句阻止索引优化；`ORDER BY updated_at DESC` 需 filesort |
| **建议** | 在 `production_orders.updated_at` 建索引；用 `status NOT IN ('已完成','已发货','订单完成')` 改为 `status IN (...活跃状态...)` |

### Q2: 流程记录列表分页查询

**位置**: [dispatch_center.py:L7485-L7495](file:///d:/yuan/不锈钢网带跟单3.0/mobile_api_ai/dispatch_center.py#L7485-L7495)

```sql
SELECT * FROM process_records WHERE 1=1 [AND status = %s]
ORDER BY created_at DESC LIMIT %s OFFSET %s
```

| 维度 | 分析 |
|------|------|
| **执行频率** | 每次流程 Tab 翻页 |
| **数据量** | 随工单增长，可能数万至数十万行 |
| **慢路径** | `WHERE 1=1` 通用查询无强制过滤条件，深分页 `OFFSET N` 扫描大量行 |
| **建议** | `process_records.created_at` 建索引；`OFFSET` 深分页改为**游标分页**（`WHERE created_at < ?`）；强制至少一个过滤条件 |

### Q4: 数据流日志聚合查询
