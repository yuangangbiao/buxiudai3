### 5 个需要修复的问题

#### 问题1（CRITICAL）：check_annotation_coverage.py 脚本不存在

**影响范围**：Q1.2-Q1、Q1.2-Q2、Q1.2-Q3

**详情**：`scripts/check_annotation_coverage.py` 在项目中不存在。scripts 目录有 50+ 个脚本，但没有类型注解覆盖率统计工具。

**影响**：这三个量子无法验证其验收标准（覆盖率 ≥ 50%），严重阻塞执行。

**建议修复**：在执行 Q1.2 之前先创建 `scripts/check_annotation_coverage.py`，或在 Q1.3-Q1(pyproject.toml 创建)之后创建该脚本作为配套工具。

#### 问题2（HIGH）：grep/head/ls/&&/; 等 Unix 命令在 Windows 不可用

**影响范围**：14 个量子

| 量子 | 使用的 Unix 命令 | Windows 替代方案 |
|------|----------------|----------------|
| Q1.1-Q1~Q3 | `grep` | `Select-String` 或 `findstr` |
| Q1.3-Q2 | `ls` | `Test-Path` 或 `dir` |
| Q1.3-Q3 | `flake8 . ; black .` 分号 | 分行执行或用 `;` (PowerShell 中 OK) |
| Q1.5-Q1~Q2 | `grep -rn` | `Select-String -Recurse` |
| T1.2-Q1~Q3 | `grep` | `Select-String` |
| M1.4-Q1 | `head` | `Get-Content -TotalCount 5` |
| P1.3-Q2 | `grep -c` | `Select-String -Pattern ... \| Measure-Object` |
| V1.2-Q1 | `&&` | `;` 或 `try/catch` |

**建议修复**：在 Q1.3 工具链量子中增加 `scripts/run_verify.ps1` PowerShell 封装脚本，统一处理跨平台验证命令兼容性。或者在 `.trae/rules/` 项目中记录验证命令需在 Git Bash 环境执行。

#### 问题3（MEDIUM）：flask-limiter 依赖未声明

**影响范围**：P1.6-Q1、P1.6-Q2

**详情**：`enhanced_requirements.txt` 和项目根 `requirements.txt` 均未包含 `flask-limiter`。

**影响**：P1.6 启动时会因 import 失败而阻塞。

**建议修复**：在 P1.6 的输入契约中将 `flask-limiter>=3.0` 添加到 `enhanced_requirements.txt`。

#### 问题4（LOW）：Q1.4-Q2 验证命令不明确

**详情**：文档写"运行扫描脚本确认硬编码清零"，但未指定具体脚本名或命令。

**建议修复**：明确化为 `python scripts/scan_hardcoded.py --check` 或类似命令。该脚本也需在 Q1.4-Q1 中创建。
