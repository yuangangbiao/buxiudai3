# 单用户部署指南

> 适用版本: v3.0
> 最后更新: 2026-05-08
> 适用场景: 单个工厂/企业使用，云端负责消息转发，本地负责业务处理

---

## 一、架构总览

```
┌─────────────────────────────────────────┐
│           ☁️ 腾讯云服务器                  │
│           124.223.57.82                   │
│                                           │
│  wechat_server.py  ← 微信回调入口(5003)    │
│  app.py            ← REST API (5000)       │
│  (其他云端服务暂不启动)                     │
└──────────────┬──────────────────────────┘
               │ 报工/查询/审批等业务请求
               ▼
┌─────────────────────────────────────────┐
│        🏭 本地车间电脑（Windows）          │
│                                           │
│  container_center_v5.py  ← 核心数据枢纽    │
│  container_center_api.py ← 容器API(5002)  │
│  主软件(跟单3.1)         ← 本地桌面端      │
└─────────────────────────────────────────┘
```

### 核心分工

| 位置 | 职责 | 关键约束 |
|------|------|---------|
| **云服务器** | 微信消息接收、签名校验、消息转发 | 必须有公网IP，企业微信回调URL指向这里 |
| **本地电脑** | 报工处理、数据存储、与主软件交互 | 必须能访问主软件数据库，需要固定内网IP |

### 数据流

```
企业微信消息
    ↓ HTTPS
云服务器 wechat_server.py（端口5003）
    ↓ 校验签名 → 解析消息 → HTTP转发
本地 container_center_api.py（端口5002）
    ↓ 业务处理
主软件(跟单3.1) 或 本地数据库
    ↓ 结果返回
云服务器 → 企业微信（通知用户）
```

## 二、☁️ 云服务器部署（腾讯云 Windows Server）

### 2.1 前置条件

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows Server 2019/2022 |
| Python | 3.10+（安装时勾选 Add Python to PATH） |
| 公网IP | 124.223.57.82（已分配） |
| 域名 | api.ningjinchenshengshusongshebeiyouxiangongsi.com（可选） |
| 安全组 | 入站规则放行 TCP 5003 端口 |

### 2.2 部署步骤

```powershell
# 1. 创建目录
mkdir D:\wechat_bot
mkdir D:\wechat_bot\mobile_api_ai

# 2. 上传代码
# 将以下文件上传到 D:\wechat_bot\mobile_api_ai\
#   wechat_server.py      ← 主服务
#   app.py                ← REST API（可选）
#   bots/                 ← 机器人模块
#   commands/             ← 指令模块
#   services/             ← 服务模块
#   api/                  ← API路由
#   integration/          ← 集成模块
#   modules/              ← 工具模块
#   config.py             ← 配置
#   .env                  ← 环境配置
#   requirements.txt      ← 依赖清单
#   WW_verify_*.txt       ← 企业微信校验文件

# 3. 安装依赖
cd D:\wechat_bot\mobile_api_ai
python -m pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
python -m pip install pycryptodome -i https://pypi.tuna.tsinghua.edu.cn/simple

# 4. 创建DAT目录和配置文件
mkdir DAT
```

WECHAT_CORP_ID=ww2a8dcc32f0c57889
WECHAT_AGENT_ID=1000002
WECHAT_SECRET=aM4kGTZiqfswfk4CEv5bmAo7EpMOujoDsjMZyhPWPPA
WECHAT_TOKEN=zxbQCK6nPgAX
WECHAT_AES_KEY=JHVrWePGvQHU4ZcoOkWurKqEsHjzaM58TQl9GCOrJWY

# ─── 本地转发地址（指向车间电脑内网IP） ───
CONTAINER_CENTER_API_URL=http://192.168.1.100:5002

> ⚠️ **关键检查**：`WECHAT_TOKEN` 和 `WECHAT_AES_KEY` 必须与企业微信管理后台 → 应用 → 接收消息 → API接收 中的值**完全一致**，否则回调验证失败。

### 2.4 启动服务

```powershell

cd D:\wechat_bot\mobile_api_ai
python wechat_server.py --port 5003
```

**验证启动成功**：
- 控制台显示 `[Server] 服务初始化完成` 和 `启动服务器: 0.0.0.0:5003`
- 浏览器访问 `http://124.223.57.82:5003/health` 返回 `{"status":"healthy"}`

### 2.5 云服务器安全组配置

登录腾讯云控制台，确认安全组入站规则：

| 方向 | 类型 | 协议 | 端口 | 来源 | 用途 |
|------|------|------|:----:|------|------|
| 入站 | 自定义 | TCP | 5003 | 0.0.0.0/0 | 微信回调入口（必须开放） |
| 入站 | 自定义 | TCP | 5000 | 0.0.0.0/0 | REST API（按需开放） |
| 入站 | 自定义 | TCP | 3389 | 指定IP | 远程桌面（建议限制来源） |

## 七、常见问题排查

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| **企业微信后台保存提示"验证URL失败"** | Token/AESKey 不匹配 | 确认 .env 中的值与企业微信后台**完全一致**（复制粘贴，不要手打） |
| | 安全组未放行 5003 端口 | 腾讯云控制台检查安全组入站规则 |
| | 服务未启动 | 登录云服务器确认 wechat_server.py 在运行 |
| **微信发了消息但没收到回复** | 本地服务未启动 | 检查 container_center_api.py 是否在运行 |
| | 内网地址配置错误 | 确认 .env 中 `CONTAINER_CENTER_API_URL` 指向正确的本地内网IP |
| | 网络不通 | 在云服务器上 `ping 192.168.1.100` 测试连通性 |
| **报工失败** | 主软件未打开 | 确认本地主软件(跟单3.1)正常运行 |
| | 数据库连接失败 | 检查 MySQL 服务是否运行 |
| **端口被占用** | 其他程序占用了端口 | `netstat -ano \| findstr 5003` 查看占用进程，然后停止该进程或使用 `--port` 参数指定其他端口 |
| **服务启动报错** | 依赖未安装 | 重新执行 `pip install -r requirements.txt` |
| | Python 版本过低 | 确认 Python 版本 >= 3.8 |
| **日志中频繁出现重试/超时** | 本地网络不稳定 | 检查网络连接，确认云服务器能稳定访问本地内网地址 |

## 八、维护与更新

### 8.1 更新代码

```powershell
# 1. 停止服务（Ctrl+C）
# 2. 备份旧代码
copy D:\wechat_bot\mobile_api_ai D:\wechat_bot\mobile_api_ai_backup

# 3. 覆盖新代码（保留 .env 和 DAT/ 目录）
# 4. 重新启动服务
```

### 8.2 日志管理

日志文件在 `logs/` 目录下，按日期分文件存储，自动轮转。默认保留 30 天，通过 `LOG_RETENTION_DAYS` 环境变量配置。

### 8.3 备份策略

| 备份内容 | 频率 | 方法 |
|---------|------|------|
| .env 配置 | 每次修改后 | 手动备份 |
| 本地数据库 | 每日 | 主软件自动备份 |
| 云端日志 | 每周 | 手动归档 |

## 九、安全注意事项

| 注意项 | 说明 |
|--------|------|
| 企业微信 Secret 保护 | WECHAT_SECRET 是敏感凭证，不要提交到 git，不要分享给无关人员 |
| 安全组最小权限 | 只开放必要的端口（如 5003），3389(RDP) 尽量限制来源IP |
| API 密钥 | API_SECRET_KEY 用于内部通信认证，定期更换 |
| 网络隔离 | 云端只转发消息，不存储业务数据。敏感数据留在本地 |
| 日志脱敏 | 日志中不要记录完整的 Secret/Token，仅记录前几位用于校验 |
